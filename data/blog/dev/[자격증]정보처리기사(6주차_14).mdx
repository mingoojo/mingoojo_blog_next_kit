### 1. 서비스 공격 유형

1. **서비스 거부 공격 (DoS)이란?**

   > **한 줄 정의:** 서버의 자원을 **고갈**시킬 목적으로 **대량의 데이터를 집중 전송**하여 정상적인 기능을 **방해**하는 공격

2. **DoS 공격 유형**

   |     공격 유형     | 한 줄 정의                                  |
   | :---------------: | ------------------------------------------- |
   | **Ping of Death** | **패킷 크기 초과** (65,536 바이트 이상)     |
   |   **SMURFING**    | **브로드캐스트** 주소로 응답 집중           |
   | **SYN Flooding**  | **3-way-handshake 중단**으로 대기 상태      |
   |   **TearDrop**    | **Fragment Offset 값 변경**으로 재조립 오류 |
   |     **Land**      | **송신=수신 IP**로 무한 응답                |
   |     **DDoS**      | **분산된 공격 지점**에서 동시 공격          |

3. **Ping of Death (죽음의 핑)**

   > **한 줄 정의:** 패킷 크기를 **허용 범위(65,536 바이트) 이상**으로 전송하여 네트워크를 마비시키는 공격
   - Ping of Death 특징

     | 특징      | 설명                                                    |
     | --------- | ------------------------------------------------------- |
     | 패킷 크기 | **65,536 바이트** 이상                                  |
     | 결과      | 패킷 **재조립 부담** + **ICMP 응답** 처리 → 시스템 다운 |
     | 변종      | jolt, sPING, ICMP bug, IceNewk                          |
     | 대비      | **ICMP Ping 메시지 차단** (방화벽)                      |

4. **SMURFING (스머핑)**

   > **한 줄 정의:** **송신 주소를 공격 대상 IP로 위장**하고 **브로드캐스트 주소**로 패킷 전송하여 응답을 집중시키는 공격
   - SMURFING 과정

     | 순서 | 과정                                           |
     | :--: | ---------------------------------------------- |
     |  1   | 송신 주소를 **공격 대상 IP로 위장**            |
     |  2   | **브로드캐스트 주소**로 패킷 전송              |
     |  3   | 네트워크 내 모든 컴퓨터가 **공격 대상에 응답** |
     |  4   | **네트워크 과부하** → 서비스 불가              |

   - SMURFING 대비

     | 대비 방법                                  |
     | ------------------------------------------ |
     | 라우터에서 **브로드캐스트 주소 사용 금지** |

5. **SYN Flooding**

   > **한 줄 정의:** **3-way-handshake 과정을 의도적으로 중단**시켜 서버가 **대기 상태**에 놓이게 하는 공격
   - SYN Flooding 과정

     | 순서 | 과정                                           |
     | :--: | ---------------------------------------------- |
     |  1   | 공격자가 **가상 IP**로 **SYN** 신호 전송       |
     |  2   | 서버가 **SYN+ACK** 응답 + **메모리 공간 확보** |
     |  3   | 가상 IP이므로 **ACK 응답 없음**                |
     |  4   | 서버가 **대기 상태** 유지 → 자원 고갈          |

   - SYN Flooding 대비

     | 대비 방법                   |
     | --------------------------- |
     | **SYN 수신 대기 시간** 단축 |
     | **침입 차단 시스템** 활용   |

6. **TearDrop**

   > **한 줄 정의:** **Fragment Offset 값을 변경**시켜 패킷 재조립 시 오류를 발생시키는 공격
   - TearDrop 대비

     | 대비 방법                             |
     | ------------------------------------- |
     | Fragment Offset 오류 시 **패킷 폐기** |

7. **Land**

   > **한 줄 정의:** **송신 IP = 수신 IP = 공격 대상 IP**로 설정하여 **무한 응답**하게 하는 공격
   - Land 대비

     | 대비 방법                           |
     | ----------------------------------- |
     | 송신/수신 IP 주소의 **적절성 검사** |

8. **DDoS (분산 서비스 거부)**

   > **한 줄 정의:** **분산된 공격 지점**에서 한 곳의 서버에 대해 **동시에 공격**하는 방식
   - DDoS 구성 요소

     |      구성 요소       | 설명                        |
     | :------------------: | --------------------------- |
     | **에이전트 (Agent)** | 공격용 툴 설치된 **호스트** |
     | **핸들러 (Handler)** | 다수 에이전트 관리 프로그램 |
     | **마스터 (Master)**  | 핸들러 설치된 호스트        |

9. **DoS 공격 비교표**

   |       공격        | 핵심 키워드               | 대비              |
   | :---------------: | ------------------------- | ----------------- |
   | **Ping of Death** | 65,536 바이트 초과        | ICMP 차단         |
   |   **SMURFING**    | **브로드캐스트**, IP 위장 | 브로드캐스트 금지 |
   | **SYN Flooding**  | **3-way-handshake** 중단  | 대기 시간 단축    |
   |   **TearDrop**    | **Offset** 변경           | 패킷 폐기         |
   |     **Land**      | **송신=수신** IP          | 적절성 검사       |
   |     **DDoS**      | **분산** 공격             | -                 |

10. **네트워크 침해 공격 용어**

    |             용어              | 한 줄 정의                                         |
    | :---------------------------: | -------------------------------------------------- |
    |     **스미싱 (Smishing)**     | **SMS**를 이용한 개인정보 탈취                     |
    |        **스피어 피싱**        | **특정 대상**에게 이메일로 개인정보 탈취           |
    |            **APT**            | 조직적으로 **지속적**으로 침투하는 **지능형 위협** |
    | **무작위 대입 (Brute Force)** | **모든 값 대입**하여 암호 해독                     |
    |       **큐싱 (Qshing)**       | **QR코드**를 통한 악성앱 유도                      |
    |      **피싱 (Phishing)**      | 공기관/금융기관 **사칭**하여 개인정보 탈취         |
    |        **Ping Flood**         | 많은 **ICMP 메시지** 전송                          |
    |         **Evil Twin**         | 동일 이름 **WiFi** 신호로 정보 탈취                |
    |        **스위치 재밍**        | **위조 MAC 주소**로 스위치 혼란                    |

11. **블루투스 관련 공격**

    |             공격              | 한 줄 정의                          |
    | :---------------------------: | ----------------------------------- |
    |    **블루버그 (BlueBug)**     | 휴대폰 **원격 조정**, 통화 **감청** |
    |  **블루스나프 (BlueSnarf)**   | 장비 **파일 접근**, OPP 이용        |
    | **블루프린팅 (BluePrinting)** | 공격 대상 **블루투스 장비 검색**    |
    |  **블루재킹 (BlueJacking)**   | 블루투스로 **스팸 메시지** 전송     |

12. **정보보안 침해 공격 용어**

    |        용어         | 한 줄 정의                                      |
    | :-----------------: | ----------------------------------------------- |
    |     **좀비 PC**     | 악성코드 감염되어 **DDoS**에 이용되는 PC        |
    |    **C&C 서버**     | **좀비 PC**에 **명령** 내리는 서버              |
    |  **봇넷 (Botnet)**  | 악성코드 감염 PC들이 **네트워크로 연결**된 형태 |
    |    **웜 (Worm)**    | 네트워크로 **자기 복제**하여 시스템 다운        |
    |  **제로데이 공격**  | 취약점 **공표 전**에 이루어지는 공격            |
    |     **키로거**      | **키보드 움직임** 탐지하여 정보 탈취            |
    |    **랜섬웨어**     | 파일 **암호화** 후 **돈 요구**                  |
    |     **백도어**      | 시스템 **비밀 통로**                            |
    |   **트로이 목마**   | **정상 프로그램 위장**, 자기 복제 **없음**      |
    | **파밍 (Pharming)** | **도메인 탈취**하여 **가짜 사이트**로 유도      |

13. **피싱 관련 공격 비교**

    |         공격          | 특징                          |
    | :-------------------: | ----------------------------- |
    |  **피싱 (Phishing)**  | 이메일/메신저로 **사칭**      |
    | **스미싱 (Smishing)** | **SMS** 이용                  |
    |    **스피어 피싱**    | **특정 대상** 선정            |
    |   **큐싱 (Qshing)**   | **QR코드** 이용               |
    |  **파밍 (Pharming)**  | **도메인 탈취** → 가짜 사이트 |

14. **악성코드 비교**

    |    악성코드     | 특징                                 |
    | :-------------: | ------------------------------------ |
    |  **웜 (Worm)**  | **자기 복제** ⭕                     |
    | **트로이 목마** | **자기 복제** ❌, 정상 프로그램 위장 |
    |  **랜섬웨어**   | 파일 **암호화** + **돈 요구**        |

### 2. 서버 인증

1. **보안 서버란?**

   > **한 줄 정의:** 인터넷을 통해 **개인정보를 암호화**하여 송·수신할 수 있는 기능을 갖춘 서버

2. **보안 서버 개요**
   - 보안 서버 필수 기능

     | 기능                     | 설명                                 |
     | ------------------------ | ------------------------------------ |
     | **SSL 인증서**           | 서버에 설치하여 **전송 정보 암호화** |
     | **암호화 응용 프로그램** | 전송 정보를 암호화하여 송·수신       |

   - 보안 서버 구축 필요성

     | 필요성                                 |
     | -------------------------------------- |
     | **스니핑 (Sniffing)** 방지 - 정보 유출 |
     | **피싱 (Phishing)** 방지 - 위조 사이트 |

3. **인증 (Authentication)이란?**

   > **한 줄 정의:** 다중 사용자 시스템에서 로그인을 요청한 **사용자의 정보를 확인**하는 보안 절차
   - 인증 vs 인가

     |   구분   |      영문      | 설명                       |
     | :------: | :------------: | -------------------------- |
     | **인증** | Authentication | 사용자 **정보 확인**       |
     | **인가** | Authorization  | **권한 여부 확인** 후 부여 |

   - 인증의 종류

     | 종류               | 설명                                  |
     | ------------------ | ------------------------------------- |
     | **등록 여부 확인** | 사용자의 **등록 여부** 확인           |
     | **메시지 확인**    | 전송된 메시지의 **위·변조 여부** 확인 |

4. **인증의 주요 유형**

   |     유형      |          영문          | 한 줄 정의                |
   | :-----------: | :--------------------: | ------------------------- |
   | **지식 기반** | Something You **Know** | **알고 있는 정보** 기반   |
   | **소유 기반** | Something You **Have** | **소유하고 있는 것** 기반 |
   | **생체 기반** | Something You **Are**  | **고유한 생체 정보** 기반 |
   | **위치 기반** | Somewhere You **Are**  | **인증 위치** 확인        |

5. **지식 기반 인증 (Something You Know)**

   > **한 줄 정의:** 사용자가 **기억하고 있는 정보**를 기반으로 인증 수행
   - 지식 기반 인증 특징

     | 특징 | 설명                       |
     | ---- | -------------------------- |
     | 장점 | 관리 비용 **저렴**         |
     | 단점 | 기억 못 하면 **인증 불가** |

   - 지식 기반 인증 종류

     |        종류         | 설명                                               |
     | :-----------------: | -------------------------------------------------- |
     | **고정된 패스워드** | 접속할 때마다 **반복 입력**                        |
     |  **패스 프레이즈**  | **긴 문장** 활용 (예: iloveyou)                    |
     | **아이핀 (I-PIN)**  | **주민등록번호 대신** 사용하는 사이버 주민등록번호 |

6. **소유 기반 인증 (Something You Have)**

   > **한 줄 정의:** 사용자가 **소유하고 있는 것**을 기반으로 인증 수행
   - 소유 기반 인증 특징

     | 특징 | 설명                           |
     | ---- | ------------------------------ |
     | 단점 | **도용** 쉬움                  |
     | 해결 | **다른 인증 방식과 함께** 사용 |

   - 소유 기반 인증 종류

     |          종류          | 설명                                                |
     | :--------------------: | --------------------------------------------------- |
     |       **신분증**       | 주민등록증, 운전면허증, 여권 등                     |
     | **메모리 카드 (토큰)** | **마그네틱 선**에 보안 코드 저장 (은행 입출금 카드) |
     |    **스마트 카드**     | **IC칩** 내장, 마이크로프로세서 포함                |
     |        **OTP**         | **일회용 패스워드**, 사용 후 폐기                   |

   - 메모리 카드 vs 스마트 카드

     | 구분 |   메모리 카드    |     스마트 카드     |
     | :--: | :--------------: | :-----------------: |
     | 저장 | **마그네틱 선**  |      **IC칩**       |
     | 기능 |   코드 저장만    | 업무 처리 기능 내장 |
     | 예시 | 은행 입출금 카드 |    IC칩 신용카드    |

7. **OTP (One Time Password)**

   > **한 줄 정의:** 요청할 때마다 **새롭게 생성**되는 **일회용 패스워드**, 한 번 사용 후 **폐기**
   - OTP 특징

     | 특징 | 설명                                   |
     | ---- | -------------------------------------- |
     | 생성 | **암호 알고리즘**으로 매번 새롭게 생성 |
     | 사용 | **한 번만** 사용 가능                  |
     | 폐기 | 사용 후 **폐기**                       |

8. **생체 기반 인증 (Something You Are)**

   > **한 줄 정의:** 사용자의 **고유한 생체 정보**를 기반으로 인증 수행
   - 생체 기반 인증 특징

     | 특징 | 설명                                               |
     | ---- | -------------------------------------------------- |
     | 장점 | 사용 **쉬움**, 도난 **위험 적음**, **위조 어려움** |

   - 생체 인증 대상

     | 인증 대상       |
     | --------------- |
     | **지문**        |
     | **홍채 / 망막** |
     | **얼굴**        |
     | **정맥**        |

9. **기타 인증 기법**
   - 행위 기반 인증 (Something You Do)
     - 사용자의 **행동 정보**를 이용해 인증 수행

       | 인증 대상 |
       | --------- |
       | **서명**  |
       | **동작**  |
       | **음성**  |

   - 위치 기반 인증 (Somewhere You Are)
     - 인증을 시도하는 **위치의 적절성** 확인

       | 인증 방법   |
       | ----------- |
       | **콜백**    |
       | **GPS**     |
       | **IP 주소** |

### 3.보안 아키텍처 / 보안 프레임워크 (정보처리기사)

1. **보안 아키텍처란?**

   > **한 줄 정의:** 정보 시스템의 **무결성, 가용성, 기밀성(CIA)**을 확보하기 위해 보안 요소 및 체계를 식별하고 **관계를 정의한 구조**

2. **보안 아키텍처 개요**
   - 보안 아키텍처 목적

     | 목적                              |
     | --------------------------------- |
     | **무결성 (Integrity)** 확보       |
     | **가용성 (Availability)** 확보    |
     | **기밀성 (Confidentiality)** 확보 |

   - 보안 3대 요소 = CIA
   - 보안 아키텍처 기대 효과

     | 기대 효과                               |
     | --------------------------------------- |
     | **관리적/물리적/기술적** 보안 개념 수립 |
     | 보안 관리 능력 **향상**                 |
     | **일관된 보안 수준** 유지               |

3. **보안 아키텍처 특징**

   | 특징       | 설명                                            |
   | ---------- | ----------------------------------------------- |
   | **유연성** | 보안 수준 변화에도 **기본 구조 수정 없이** 지원 |
   | **확장성** | 보안 요구사항 **변화/추가 수용** 가능           |
   - 보안 아키텍처 표준

     | 표준            |
     | --------------- |
     | **ITU-T X.805** |

4. **관리적 / 물리적 / 기술적 보안**

   |      구분       | 내용                                           |

### 4. 로그 분석 (정보처리기사)

1. **로그(Log)란?**

   > **한 줄 정의:** 시스템 사용에 대한 **모든 내역을 기록**해 놓은 것

2. **로그 개요**
   - 로그의 용도

     | 용도                                            |
     | ----------------------------------------------- |
     | 시스템 **침해 사고** 발생 시 **해킹 흔적** 파악 |
     | **공격 기법** 파악                              |
     | **침입 흔적** 확인                              |
     | **취약점** 확인                                 |

   - 로그 분석 필요성
     - **정기적** 분석으로 침입 흔적/취약점 확인

3. **리눅스(LINUX) 로그**
   - 시스템 모든 로그를 **/var/log** 디렉터리에서 관리
   - 리눅스 로그 관리

     | 항목          | 경로/설명            |
     | ------------- | -------------------- |
     | 로그 디렉터리 | **/var/log**         |
     | 로그 데몬     | **syslogd**          |
     | 설정 파일     | **/etc/syslog.conf** |

   - 로그 관리 프로세스
     1. **syslogd** 데몬 실행
     2. **/etc/syslog.conf** 파일 읽기
     3. 로그 파일 위치 파악
     4. 로그 작업 시작

4. **리눅스 주요 로그 파일**

   |    로그    |  데몬   | 파일명                | 내용                                         |
   | :--------: | :-----: | --------------------- | -------------------------------------------- |
   |  **커널**  | kernel  | /dev/console          | 커널 내용 **장치에 표시**                    |
   |  **커널**  | kernel  | /var/log/**wtmp**     | **성공한 로그인/로그아웃**, 시스템 시작/종료 |
   |  **커널**  | kernel  | /var/run/**utmp**     | **현재 로그인** 사용자 상태                  |
   |  **커널**  | kernel  | /var/log/**btmp**     | **실패한 로그인**                            |
   |  **커널**  | kernel  | /var/log/**lastlog**  | **마지막 성공 로그인**                       |
   |  **부팅**  |  boot   | /var/log/**boot**     | **부팅 메시지**                              |
   |  **크론**  |  crond  | /var/log/**cron**     | **작업 스케줄러** 내역                       |
   | **시스템** | syslogd | /var/log/**messages** | 커널 **실시간 메시지**                       |
   |  **보안**  | xinetd  | /var/log/**secure**   | 시스템 **접속** 로그                         |
   |  **FTP**   |  ftpd   | /var/log/**xferlog**  | **FTP 접속** 사용자                          |

5. **리눅스 로그 파일 핵심**
   1. 로그인 관련 로그

      |   파일명    | 내용                       |
      | :---------: | -------------------------- |
      |  **wtmp**   | **성공한** 로그인/로그아웃 |
      |  **utmp**   | **현재** 로그인 상태       |
      |  **btmp**   | **실패한** 로그인          |
      | **lastlog** | **마지막** 성공 로그인     |

   2. 로그인 로그 암기
      - **wtmp** = **W**rite = **성공** 기록
      - **utmp** = **U**ser = **현재** 사용자
      - **btmp** = **B**ad = **실패** 로그인
      - **lastlog** = **Last** = **마지막** 로그인
   3. 기타 중요 로그

      |    파일명    | 내용                   |
      | :----------: | ---------------------- |
      | **messages** | 커널 **실시간** 메시지 |
      |  **secure**  | 시스템 **접속** 로그   |
      | **xferlog**  | **FTP** 접속           |
      |   **cron**   | 작업 **스케줄러**      |
      |   **boot**   | **부팅** 메시지        |

6. **윈도우(Windows) 로그**
   - **이벤트 로그** 형식으로 시스템 로그 관리
   - Windows 이벤트 뷰어 실행 경로

     | 경로                                       |
     | ------------------------------------------ |
     | **[제어판] → [관리 도구] → [이벤트 뷰어]** |

7. **Windows 이벤트 뷰어 로그**

   |         로그         | 내용                                      |
   | :------------------: | ----------------------------------------- |
   |  **응용 프로그램**   | **응용 프로그램**에서 발생하는 이벤트     |
   |       **보안**       | **로그온 시도**, 파일/객체 생성/조회/제거 |
   |      **시스템**      | Windows **시스템 구성 요소** 이벤트       |
   |      **Setup**       | 프로그램 **설치** 관련 이벤트             |
   | **Forwarded Events** | **다른 컴퓨터**와 상호 작용 이벤트        |

### 5. 보안 솔루션

1. **보안 솔루션이란?**

   > **한 줄 정의:** **접근 통제, 침입 차단 및 탐지** 등을 수행하여 외부로부터의 **불법적인 침입을 막는** 기술 및 시스템

2. **주요 보안 솔루션**

   |    솔루션     | 한 줄 정의                        |
   | :-----------: | --------------------------------- |
   |  **방화벽**   | **침입 차단** 시스템              |
   |    **IDS**    | **침입 탐지** 시스템              |
   |    **IPS**    | **침입 방지** 시스템 (방화벽+IDS) |
   |    **DLP**    | **데이터 유출 방지**              |
   | **웹 방화벽** | **웹 기반 공격** 방어             |
   |    **VPN**    | **가상 사설** 네트워크            |
   |    **NAC**    | **네트워크 접근 통제**            |
   |    **SSH**    | **안전한 원격 접속** 프로토콜     |

3. **방화벽 (Firewall)**

   > **한 줄 정의:** 내부 네트워크와 인터넷 간 전송 정보를 **선별하여 수용·거부·수정**하는 **침입 차단** 시스템
   - 방화벽 특징

     | 특징            | 설명                                     |
     | --------------- | ---------------------------------------- |
     | **내부 → 외부** | 패킷 **그대로 통과**                     |
     | **외부 → 내부** | 내용 **엄밀히 체크**, 인증된 패킷만 통과 |
     | 목적            | **정보 유출 방지**                       |

4. **침입 탐지 시스템 (IDS)**

   > **한 줄 정의:** 컴퓨터 시스템의 **비정상적인 사용, 오용, 남용**을 **실시간으로 탐지**하는 시스템
   - IDS 탐지 방법

     |          방법           | 설명                             |
     | :---------------------: | -------------------------------- |
     | **오용 탐지** (Misuse)  | 미리 입력한 **공격 패턴** 감지   |
     | **이상 탐지** (Anomaly) | **비정상적 행위/자원 사용** 감지 |

   - IDS 종류

     |   종류   |     영문      | 중점                 | 예시                 |
     | :------: | :-----------: | -------------------- | -------------------- |
     | **HIDS** |  Host-Based   | **내부** 시스템 감시 | OSSEC, md5deep, AIDE |
     | **NIDS** | Network-Based | **외부** 침입 감시   | **Snort**, Zeek      |

   - IDS 위치별 탐지 대상

     | 위치              | 탐지 대상                |
     | ----------------- | ------------------------ |
     | **라우터 앞**     | **모든 공격** 탐지       |
     | **라우터 뒤**     | 필터링 **통과한** 공격   |
     | **방화벽 뒤**     | **내부→외부** 공격       |
     | **내부 네트워크** | **내부→내부** 공격       |
     | **DMZ**           | 외부 공격, **서버 보호** |

5. **침입 방지 시스템 (IPS)**

   > **한 줄 정의:** **방화벽 + IDS** 결합, 비정상 트래픽을 **능동적으로 차단/격리**하는 보안 솔루션
   - IPS 특징

     | 특징 | 설명                               |
     | ---- | ---------------------------------- |
     | 구성 | **방화벽** + **IDS** 결합          |
     | 동작 | 패킷 검사 → 비정상 탐지 → **차단** |
     | 방식 | **능동적** 방어                    |

   - IDS vs IPS
     - **IDS** = **탐지**만 (Detection)
     - **IPS** = **탐지 + 차단** (Prevention)

6. **데이터 유출 방지 (DLP)**

   > **한 줄 정의:** **내부 정보**의 **외부 유출을 방지**하는 보안 솔루션
   - DLP 특징

     | 특징 | 설명                             |
     | ---- | -------------------------------- |
     | 대상 | 사내 **PC**, **네트워크** 정보   |
     | 감시 | 메일, 메신저, 웹 하드, 프린터 등 |
     | 목적 | 유출 **사전 방지**               |

7. **웹 방화벽 (Web Firewall)**

   > **한 줄 정의:** **SQL 삽입, XSS** 등 **웹 기반 공격**을 방어하는 **웹 서버 특화** 방화벽
   - 웹 방화벽 특징

     | 특징 | 설명                         |
     | ---- | ---------------------------- |
     | 목적 | **웹 기반 공격** 방어        |
     | 대상 | **SQL 삽입**, **XSS** 등     |
     | 동작 | 공격이 서버 도달 **전** 차단 |

8. **VPN (Virtual Private Network)**

   > **한 줄 정의:** **공중 네트워크**와 **암호화 기술**을 이용하여 **전용 회선처럼** 사용하는 **가상 사설** 네트워크
   - VPN 특징

     | 특징 | 설명                          |
     | ---- | ----------------------------- |
     | 기술 | **암호화** 기술               |
     | 장점 | **비용 절감**, 지역 제한 없음 |
     | 용도 | 원격 근무, 지사 연결          |

9. **NAC (Network Access Control)**

   > **한 줄 정의:** 내부 PC의 **MAC 주소**를 등록하여 **네트워크 접근을 통제**하는 보안 솔루션
   - NAC 기능

     | 기능                          |
     | ----------------------------- |
     | **MAC 주소** 등록 및 관리     |
     | **불법 소프트웨어** 설치 방지 |
     | 백신/**보안 패치** 일괄 배포  |
     | **비인가 시스템** 자동 검출   |

10. **SSH (Secure SHell)**

    > **한 줄 정의:** **원격 로그인, 명령 실행, 파일 복사** 등을 **안전하게** 수행하는 프로토콜
    - SSH 특징

      | 특징 | 설명                           |
      | ---- | ------------------------------ |
      | 포트 | **22번**                       |
      | 기능 | 로그인, 원격 명령, 파일 복사   |
      | 보안 | **데이터 암호화**, 강력한 인증 |
      | 인증 | **키(key)** 기반 인증          |

    - SSH 키 인증

      | 인증 방법                               |
      | --------------------------------------- |
      | 클라이언트 **공개키**를 서버에 **등록** |
