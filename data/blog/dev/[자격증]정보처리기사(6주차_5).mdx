### 1. 데이터 전환

1. **데이터 전환이란?**

   > **한 줄 정의:** 기존 시스템 데이터를 `추출(E) → 변환(T) → 적재(L)`하여 새 시스템에 이관하는 과정

2. **데이터 전환 핵심 개념**

   | 항목      | 내용                                         |
   | --------- | -------------------------------------------- |
   | 정식 명칭 | **ETL** (Extraction, Transformation, Load)   |
   | 다른 명칭 | 데이터 **이행**, 데이터 **이관** (Migration) |
   | 목적      | 기존 시스템 → 새 시스템으로 데이터 이동      |

3. **ETL 과정**

   ```
   [기존 시스템]                [새 시스템]
      ↓                          ↑
   ┌───────┐    ┌───────┐    ┌───────┐
   │   E   │ →  │   T   │ →  │   L   │
   │  추출  │    │  변환  │    │  적재  │
   └───────┘    └───────┘    └───────┘
   Extraction  Transformation   Load
   ```

   | 단계  |      영문      | 한 줄 정의                      |
   | :---: | :------------: | ------------------------------- |
   | **E** |   Extraction   | 기존 시스템에서 데이터 **추출** |
   | **T** | Transformation | 새 시스템 형식에 맞게 **변환**  |
   | **L** |      Load      | 새 시스템에 데이터 **적재**     |

4. 데이터 전환 계획서
   - 데이터 전환 작업에 필요한 **모든 계획을 기록**하는 문서
   1. 주요 항목

      | 항목             | 세부 내용                                           |
      | ---------------- | --------------------------------------------------- |
      | **전환 개요**    | 목표, 성공 요인, 전제/제약조건                      |
      | **대상 및 범위** | 전환 대상, 범위 정의                                |
      | **환경 구성**    | 원천/목적 시스템 구성도, DISK 사용량                |
      | **조직 및 역할** | 전환 조직도, 역할 정의                              |
      | **전환 일정**    | 일정 계획                                           |
      | **전환 방안**    | 규칙, 절차, 방법, 설계, 개발/테스트 계획, 검증 방안 |
      | **정비 방안**    | 정비 대상/방법, 일정, 조직                          |
      | **비상 계획**    | 상황실, 의사소통 체계, **복구 대책**                |

   2. 데이터 전환 계획서 구조

      ```
      ├── 3. 전환 환경 구성
      │   ├── 원천 시스템 구성도
      │   ├── 목적 시스템 구성도
      │   └── DISK 사용량
      ├── 4. 전환 조직 및 역할
      ├── 5. 전환 일정
      ├── 6. 전환 방안
      │   ├── 전환 규칙
      │   ├── 전환 절차
      │   ├── 전환 방법
      │   ├── 전환 설계
      │   ├── 개발/테스트 계획

      ```

### 2. 데이터 전환 방안

1. **데이터 전환 방안이란?**

   > **한 줄 정의:** 데이터 전환을 위한 **규칙, 절차, 방법, 설계, 계획, 검증** 등을 정의한 것

2. **데이터 전환 방안 7가지 항목**

   | 항목                     | 한 줄 정의              |
   | ------------------------ | ----------------------- |
   | **전환 규칙**            | 공통 적용 규칙 기술     |
   | **전환 절차**            | 단계별 절차 기술        |
   | **전환 방법**            | 단위 업무별 방법 기술   |
   | **전환 설계**            | 테이블 매핑 정의서 작성 |
   | **프로그램 개발/테스트** | 전환 프로그램 계획      |
   | **전환 계획**            | 선/본/후 전환 계획      |
   | **검증 방안**            | 정합성 검증 방안        |
   1. 데이터 전환 규칙
      - 전환 과정에서 **공통 적용할 규칙** 기술
   2. 데이터 전환 절차
      - 전환 단계별 **상세 절차** 기술
      - 전환 절차 5단계

        ```
        전환 준비 → 전환 설계/개발 → 전환 테스트 → 실데이터 전환 → 최종 전환/검증
        ```

        | 순서 | 단계                  |
        | :--: | --------------------- |
        |  1   | 전환 **준비**         |
        |  2   | 전환 **설계/개발**    |
        |  3   | 전환 **테스트**       |
        |  4   | **실데이터** 전환     |
        |  5   | **최종** 전환 및 검증 |

        💡 **데이터 흐름도** 작성하여 작업 이해 돕기

   3. 데이터 전환 방법
      - **단위 업무별** 전환 방법과 전제조건 기술
   4. 데이터 전환 설계
      - 원천↔목적 시스템 테이블 **매핑 정의서** 작성
      - 포함 내용
        - 전환 **대상** 기술
        - 전환 **제외 대상** 기술
        - 테이블 **매핑 정의서** 작성
   5. 전환 프로그램 개발 및 테스트 계획
      - 전환 **프로그램 개발/테스트 계획** 수립
      - 프로그램 목록 포함 내용
        - 프로그램 **입력 정보**
        - **중간 생성** 정보
        - **출력** 정보
        - 프로그램 **위치**
        - **담당자**
      - 테스트
        - **체크리스트** 기반 실시
        - 결과를 **전환 시나리오**에 반영
   6. 데이터 전환 계획 (시험 필수!)
      - **선/본/후 전환**으로 분리하여 계획 수립
      - 선/본/후 전환

        |    구분     | 한 줄 정의                 | 대상                                    |
        | :---------: | -------------------------- | --------------------------------------- |
        | **선 전환** | 본 전환 **전에** 미리 전환 | **대량 데이터** (일자별 거래/근태 내역) |
        | **본 전환** | **핵심** 전환 작업         | 주요 데이터                             |
        | **후 전환** | 본 전환 **후에** 전환      | 나중에 해도 되는 대상                   |

      - 전환 시간 단축 전략

        ```
        대량 데이터 → 선 전환 (사전에 미리!)
        핵심 데이터 → 본 전환 (시간대별 상세 작성)
        덜 급한 것 → 후 전환 (나중에)
        ```

      - 전환 시나리오 포함 내용
        - 전환 **시간**
        - 전환 **담당자**
        - **관리자**

   7. 데이터 검증 방안
      - 전환 데이터의 **정합성 검증** 방안 수립
      - 포함 내용
        - 전환 데이터 **정합성** 검증
        - 문제 발생 시 **대응 방안**
        - **단계별** 검증 방안
   8. 데이터 전환 방안 전체 구조

      ```
      데이터 전환 방안
      ├── 1. 전환 규칙 (공통 규칙)
      ├── 2. 전환 절차 (준비→설계/개발→테스트→실전환→검증)
      ├── 3. 전환 방법 (단위 업무별)
      ├── 4. 전환 설계 (매핑 정의서)
      ├── 5. 프로그램 개발/테스트 (체크리스트)
      ├── 6. 전환 계획 (선/본/후 전환)
      └── 7. 검증 방안 (정합성 검증)
      ```

### 3. 데이터 검증

1. **데이터 검증이란?**

   > **한 줄 정의:** 원천→목적 시스템으로 데이터 전환이 **정상 수행되었는지 확인**하는 과정

2. **데이터 검증 분류**

   | 분류 기준     | 종류                                            |
   | ------------- | ----------------------------------------------- |
   | **검증 방법** | 로그, 기본 항목, 응용 프로그램, 응용 데이터, 값 |
   | **검증 단계** | 추출, 전환, DB 적재, DB 적재 후, 전환 완료 후   |

3. 검증 방법에 따른 분류 (5가지)

   | 검증 방법              | 한 줄 정의                         |
   | ---------------------- | ---------------------------------- |
   | **로그 검증**          | 추출/전환/적재 **로그** 검증       |
   | **기본 항목 검증**     | 별도 요청된 **검증 항목** 검증     |
   | **응용 프로그램 검증** | **응용 프로그램** 통한 정합성 검증 |
   | **응용 데이터 검증**   | **업무 규칙** 기준 정합성 검증     |
   | **값 검증**            | 합계, 범위, 속성 변경 **값** 검증  |
   - 값 검증 세부 내용
     - 숫자 항목 **합계** 검증
     - 코드 데이터 **범위** 검증
     - 속성 변경에 따른 **값** 검증

4. **검증 단계에 따른 분류**

   ```
   추출 → 전환 → DB 적재 → DB 적재 후 → 전환 완료 후
   ```

   |    검증 단계     | 목적                               |         검증 방법         |
   | :--------------: | ---------------------------------- | :-----------------------: |
   |     **추출**     | 원천 시스템 데이터 **정합성** 확인 |         로그 검증         |
   |     **전환**     | **매핑 정의서** 반영/오류 확인     |         로그 검증         |
   |   **DB 적재**    | SAM 파일 적재 **오류/누락** 확인   |         로그 검증         |
   |  **DB 적재 후**  | 적재 완료 후 **정합성** 확인       |      기본 항목 검증       |
   | **전환 완료 후** | **추가 검증** 통한 정합성 확인     | 응용 프로그램/데이터 검증 |
   - 검증 단계별 정리
     1. 추출 단계
        - 원천 시스템 데이터 정합성 확인
        - **검증 방법:** 로그 검증
     2. 전환 단계
        - 매핑 정의서 정확히 반영되었는지 확인
        - **검증 방법:** 로그 검증
        - 매핑 정의서 오류 여부 확인
     3. DB 적재 단계
        - SAM 파일 적재 과정 오류/누락 확인
        - **검증 방법:** 로그 검증
     4. DB 적재 후 단계
        - 적재 완료 후 정합성 확인
        - **검증 방법:** 기본 항목 검증
     5. 전환 완료 후 단계
        - 추가 검증 과정으로 최종 정합성 확인
        - **검증 방법:** 응용 프로그램 검증, 응용 데이터 검증

5. **검증 방법 vs 검증 단계 매핑**

   |  검증 단계   |      사용되는 검증 방법       |
   | :----------: | :---------------------------: |
   |     추출     |         **로그** 검증         |
   |     전환     |         **로그** 검증         |
   |   DB 적재    |         **로그** 검증         |
   |  DB 적재 후  |      **기본 항목** 검증       |
   | 전환 완료 후 | **응용 프로그램/데이터** 검증 |

   💡 **포인트:** 초반(추출~적재)은 **로그 검증**, 후반은 **기본 항목/응용 검증**

### 4. 오류 데이터 측정 및 정제

1. **오류 데이터 측정 및 정제란?**

   > **한 줄 정의:** **고품질 데이터** 운영/관리를 위해 오류를 찾아 **측정하고 정제**하는 과정

2. **진행 순서**

   ```
   데이터 품질 분석 → 오류 데이터 측정 → 오류 데이터 정제
   ```

   | 순서 | 단계          | 한 줄 정의                          |
   | :--: | ------------- | ----------------------------------- |
   |  1   | **품질 분석** | 정합성 확인하여 오류 찾기           |
   |  2   | **오류 측정** | 오류 수량 측정, 목록 작성           |
   |  3   | **오류 정제** | 원천 데이터 정제 또는 프로그램 수정 |
   1. 데이터 품질 분석
      - 원천/목적 시스템 데이터의 **정합성 여부 확인**하여 오류 찾기
      - 원천 시스템 정합성 항목

        | 검사 항목         | 확인 내용                        |
        | ----------------- | -------------------------------- |
        | **필수 항목**     | 데이터가 모두 있는가?            |
        | **데이터 유형**   | 문자형, 숫자형, 날짜형 올바른가? |
        | **날짜**          | 유효한 날짜 형태인가?            |
        | **금액**          | 유효한 값 범위인가?              |
        | **계좌번호/코드** | 정해진 규칙에 맞는가?            |
        | **영업점**        | 실제 존재하는가?                 |
        | **업무 규칙**     | 위배되는 정보 없는가?            |
        | **보고서 값**     | 실제 데이터와 일치하는가?        |
        | **잔액**          | 회계 정보와 일치하는가?          |

      - 목적 시스템 정합성 항목
        - 과목별 **좌수 및 잔액**
        - 특정 기준 분류된 좌수 및 잔액
        - **보고서** 항목 또는 **통계** 수치
        - 계좌/고객 **샘플링** 확인
        - 특수 관계 고객 데이터 확인

   2. 오류 데이터 측정
      - 정상/오류 데이터 수를 측정하여 **오류 관리 목록** 작성
      - 측정 내용

        |      구분       | 측정 방법                       |
        | :-------------: | ------------------------------- |
        | **정상 데이터** | 업무 영역별, 테이블별 수량 측정 |
        | **오류 데이터** | 오류 위치, 유형별 수량 측정     |

   3. 오류 데이터 정제
      - 오류 분석 후 **원천 데이터 정제** 또는 **전환 프로그램 수정**
      - 오류 데이터 분석 항목
        - 오류 **상태**
        - 오류 **심각도**
        - **해결 방안**

3. **오류 상태 6가지**

   |      상태      | 한 줄 정의                     |
   | :------------: | ------------------------------ |
   |    **Open**    | 오류 **보고만** 됨 (분석 전)   |
   |  **Assigned**  | 개발자에게 오류 **전달**됨     |
   |   **Fixed**    | 개발자가 오류 **수정**함       |
   |   **Closed**   | 재테스트 후 오류 **없음** 확인 |
   |  **Deferred**  | 오류 수정 **연기**             |
   | **Classified** | 오류가 **아님**으로 확인       |

4. **오류 상태 흐름**

   ```
   Open → Assigned → Fixed → Closed
   ↓        ↓
   Classified  Deferred
   (오류 아님)  (연기)
   ```

5. **오류 심각도 3단계**

   | 심각도 | 한 줄 정의                     |
   | :----: | ------------------------------ |
   | **상** | 데이터 전환 **수행 불가**      |
   | **중** | 데이터 전환 **전반에 영향**    |
   | **하** | 직접 영향 없음, 용도/배치 오류 |

6. 오류 데이터 정제 전체 흐름

   ```
   ┌──────────────────────────────────────────────────┐
   │ 1. 데이터 품질 분석                                  │
   │    - 원천/목적 시스템 정합성 확인                       │
   │    - 오류 데이터 발견                                │
   ├──────────────────────────────────────────────────┤
   │ 2. 오류 데이터 측정                                  │
   │    - 정상/오류 데이터 수량 측정                        │
   │    - 오류 관리 목록 작성                              │
   ├──────────────────────────────────────────────────┤
   │ 3. 오류 데이터 정제                                  │
   │    - 상태/심각도/해결방안 분석                         │
   │    - 원천 데이터 정제 OR 전환 프로그램 수정               │
   └──────────────────────────────────────────────────┘
   ```
