문서를 작성할 때 가장 많이 쓰이는 포맷 중 하나가 `마크다운(Markdown)`이다.  
단순한 텍스트 기반이지만, 최소한의 기호만으로 문서 서식을 표현할 수 있어 가볍고 직관적이다.  
개발자라면 깃허브(GitHub) README, 기술 블로그, 노션(Notion) 등에서 이미 수없이 접했을 것이다.

이번 글에서는 **Markdown의 핵심 문법**을 정리하고, 더 나아가 **`.md`와 `.mdx` 파일의 차이**까지 알아본다.

---

## ✍️ 텍스트 서식

```md
**굵게**
_이탤릭_
**_굵고 이탤릭_**
~~취소선~~
`인라인 코드`
```

#### 결과

**굵게**<br/>
_이탤릭_<br/>
**_굵고 이탤릭_**<br/>
~~취소선~~<br/>
`인라인 코드`<br/>

## 📂 .md 파일과 .mdx 파일의 차이

### `.md` (Markdown)

- 순수 마크다운 문서
- 텍스트, 링크, 표, 코드 블록 등 정적 콘텐츠 작성
- 깃허브 README, 기술 문서, 블로그 포스트에 자주 사용

### `.mdx` (Markdown + JSX)

- 마크다운 안에서 **React 컴포넌트** 사용 가능
- 정적인 문서뿐만 아니라 동적인 UI 삽입 가능
- 예시:

```mdx
# MDX 예제

여기는 일반 마크다운 문법

<MyComponent title="props 전달 가능" />
```

Next.js, Gatsby 같은 프레임워크에서는 `.mdx`를 활용해 문서와 컴포넌트를 한 곳에서 다룰 수 있다.

---

## 📝 정리

- 마크다운은 간단한 기호만으로 강력한 문서를 작성할 수 있는 도구다.
- `.md`는 순수 마크다운, `.mdx`는 마크다운 + React 컴포넌트 확장 버전으로 이해하면 된다.
- 상황에 따라 `.md`로 간단히 문서를 쓰거나, `.mdx`로 동적 요소를 추가하면 된다.
