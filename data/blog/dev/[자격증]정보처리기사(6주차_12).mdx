### 1. 네트워크 관련 신기술

1. **네트워크 신기술 분류**

|        분류         | 기술                               |
| :-----------------: | ---------------------------------- |
| **소프트웨어 정의** | SDN, SDDC, SDS                     |
|    **IoT 관련**     | IoT, NFC, BLE, UWB, 메시 네트워크  |
|    **클라우드**     | 클라우드 컴퓨팅, MCC, 인터클라우드 |
| **차세대 네트워크** | NGN, 5G, 네트워크 슬라이싱         |

2. **지능형 초연결망**

   > **한 줄 정의:** 4차 산업혁명 시대의 데이터 트래픽을 효과적으로 수용하기 위한 **정부 주관 네트워크 사업**

   | 특징 | 설명                                                                 |
   | ---- | -------------------------------------------------------------------- |
   | 주관 | **과학기술정보통신부**                                               |
   | 기술 | **소프트웨어 정의 기술 (SDE)** 적용                                  |
   | 수용 | IoT, 클라우드, 빅데이터, **5G**                                      |
   | 발전 | 초고속정보통신망 → 광대역통합망 → 광대역융합망 → **지능형 초연결망** |

3. **소프트웨어 정의 기술 (SDE/SDx)**

   > **한 줄 정의:** 자원을 **가상화**하여 개별 사용자에게 제공하고 **중앙에서 통합 제어**하는 기술

   |   기술   | 한 줄 정의                                       |
   | :------: | ------------------------------------------------ |
   | **SDN**  | **네트워크**를 가상화하여 소프트웨어로 제어/관리 |
   | **SDDC** | **데이터 센터** 전체를 가상화, 소프트웨어로 관리 |
   | **SDS**  | **스토리지**를 가상화하여 통합/분리 관리         |
   - 용어 설명
     - **SDN** = Software Defined **N**etworking = **네트워크**
     - **SDDC** = Software Defined **D**ata **C**enter = **데이터 센터**
     - **SDS** = Software Defined **S**torage = **스토리지**

4. **IoT (Internet of Things, 사물 인터넷)**

   > **한 줄 정의:** 실세계와 가상 세계의 다양한 사물들을 **인터넷으로 연결**하여 서비스를 제공하는 기술

   | 특징 | 설명                                    |
   | ---- | --------------------------------------- |
   | 발전 | **M2M** → **IoT**로 진화                |
   | 요구 | **개방형 아키텍처**, **정보 보안** 중요 |
   - IoT 주요 기술
     - 스마트 센싱 기술
     - 유무선 통신 및 네트워크 인프라 기술
     - 사물 인터넷 인터페이스 기술
     - 사물 인터넷 서비스 기술
   - IoT 관련 용어
     | 용어 | 한 줄 정의 |
     | :------------------: | ------------------------------------------------- |
     | **올조인 (AllJoyn)** | **오픈소스** 기반 IoT 플랫폼, 다른 OS/HW 간 통신 |
     | **메시 네트워크** | 디바이스를 **그물망**처럼 연결하는 네트워크 |
     | **Wi-SUN** | **저전력 장거리(LPWA)** IoT 통신, 스마트 그리드용 |
     | **UWB** | **초광대역**, 짧은 거리 + 대용량 + 저전력 |
     | **피코넷** | **블루투스/UWB**로 좁은 공간 내 네트워크 형성 |
     | **USN** | **RFID 태그**로 센서 정보 수집 네트워크 |
     | **SON** | **자동 구성** 네트워크, 스스로 망 구성 |
     | **BLE** | **저전력 블루투스**, 절전 모드 유지 |
     | **NFC** | **근거리 무선 통신**, 10cm 내, 결제/교통 |
   - 주요 IoT 기술 상세
     - **NFC (Near Field Communication)**
       - **10cm 이내** 근거리에서 양방향 통신하는 **RFID** 기술

         | 특징   | 설명                  |
         | ------ | --------------------- |
         | 주파수 | **13.56MHz**          |
         | 거리   | **10cm** 이내         |
         | 속도   | 최대 **424kbps**      |
         | 활용   | 결제, 교통, 출입 통제 |

     - BLE (Bluetooth Low Energy)
       - 대기 시 **절전 모드**를 유지하는 **저전력 블루투스**

       | 특징   | 설명                              |
       | ------ | --------------------------------- |
       | 주파수 | **2.4GHz** (일반 블루투스와 동일) |
       | 특징   | 대기 시 **절전 모드**             |
       | 활용   | 시계, 비컨, **웨어러블**          |

     - UWB (Ultra WideBand)
       - **짧은 거리**에서 **대용량 데이터**를 **저전력**으로 전송하는 기술

       | 특징      | 설명                 |
       | --------- | -------------------- |
       | 다른 이름 | **무선 디지털 펄스** |
       | 비교 대상 | **블루투스**         |
       | 장점      | 땅속/벽 뒤 전송 가능 |

5. **클라우드 컴퓨팅 (Cloud Computing)**

   > **한 줄 정의:** 컴퓨팅 자원을 **중앙 컴퓨터**에 두고 **인터넷**으로 언제 어디서나 작업 수행하는 환경
   - 클라우드 vs 그리드 비교

     | 구분 |       클라우드       |            그리드            |
     | :--: | :------------------: | :--------------------------: |
     | 방식 | 중앙 자원 **빌려줌** | 수많은 컴퓨터 **분산 처리**  |
     | 용도 |     일반 서비스      | **대규모 연산** (기상, 우주) |

   - 클라우드 관련 기술

     |         기술          | 한 줄 정의                                  |
     | :-------------------: | ------------------------------------------- |
     |        **MCC**        | **모바일** 기기로 클라우드 인프라 구성      |
     |   **인터클라우드**    | 여러 클라우드 서비스를 **연결**             |
     |   **클라우드 HSM**    | 클라우드 기반 **암호화 키** 관리 보안기기   |
     | **파스-타 (PaaS-TA)** | **개방형** 클라우드 플랫폼, 과기정통부 지원 |

   - 인터클라우드 서비스 형태

     |     형태      | 설명                                  |
     | :-----------: | ------------------------------------- |
     | **대등 접속** | 제공자 간 **직접 연계**               |
     |   **연합**    | **자원 공유**, 동적 할당, 단일 서비스 |
     |   **중개**    | **중개 서비스** 제공                  |

6. **차세대 네트워크 기술**

   |         기술          | 한 줄 정의                                                           |
   | :-------------------: | -------------------------------------------------------------------- |
   |        **NGN**        | ITU-T의 **유선망 기반** 차세대 통신망                                |
   |      **All-IP**       | 모든 통신망을 **IP 기반**으로 통합                                   |
   |        **NDN**        | **콘텐츠** 정보와 라우터 기능으로 데이터 전송                        |
   | **네트워크 슬라이싱** | 하나의 물리 네트워크를 **다수의 가상 네트워크**로 분리 (**5G 핵심**) |
   |        **WDM**        | **파장 분할** 다중화, 광섬유 통신                                    |

7. **기타 네트워크 신기술**

   |        기술         | 한 줄 정의                                    |
   | :-----------------: | --------------------------------------------- |
   |    **징 (Zing)**    | **3.5Gbps** 초고속 NFC, 10cm 이내             |
   |      **WBAN**       | **웨어러블/임플란트** 센서 무선 연결          |
   |       **GIS**       | **지리 정보** 수집/분석 시스템                |
   | **애드혹 네트워크** | 고정 유선망 없이 **모바일 호스트**만으로 구성 |
   |       **LOD**       | 웹상에 **공개된 연계 데이터**                 |
   |       **SSO**       | **한 번 로그인**으로 모든 사이트 이용         |
   |  **스마트 그리드**  | **전력 + IT** 통합 시스템                     |
   |    **메타버스**     | 현실과 같은 **3차원 가상 세계**               |

### 2. 네트워크 구축

1. **네트워크 설치 구조란?**

   > **한 줄 정의:** 통신망을 구성하는 요소들을 **공간적으로 배치**하는 방법
   - 인터넷/통신 장비들을 “어디에, 어떻게 놓을 거냐”에 대한 설계도
     - 서버는 한 곳에 몰아둘지?
     - 각 사무실마다 장비를 둘지?
     - 중앙에서 관리할지, 지역별로 나눌지?

2. **네트워크 설치 구조 종류**

   |    구조    | 특징                 |
   | :--------: | -------------------- |
   |  **성형**  | 중앙 집중형, 별 모양 |
   |  **링형**  | 루프형, 원형         |
   | **버스형** | 한 회선에 여러 단말  |
   | **계층형** | 트리형, 분산형       |
   |  **망형**  | 메시형, 모두 연결    |

3. **성형 (Star, 중앙 집중형)**
   - **중앙 컴퓨터**를 중심으로 단말장치들이 연결되는 **중앙 집중식** 구조
   - 성형 특징

     | 특징           | 설명                      |
     | -------------- | ------------------------- |
     | 연결 방식      | **Point-to-Point**        |
     | 데이터 교환    | **중앙 컴퓨터** 통해 교환 |
     | 단말 추가/제거 | **쉬움**                  |
     | 교환 노드 수   | **가장 적음**             |

   - 성형 장단점

     | 장점                                 | 단점                              |
     | ------------------------------------ | --------------------------------- |
     | 단말 고장 시 **다른 단말 영향 없음** | **중앙 컴퓨터 고장** 시 전체 정지 |
     | 단말 추가/제거 **용이**              | 중앙 집중에 의존                  |

4. **링형 (Ring, 루프형)**
   - 컴퓨터와 단말장치들을 **이웃끼리 원형**으로 연결한 구조
   - 링형 특징

     | 특징      | 설명                       |
     | --------- | -------------------------- |
     | 연결 방식 | **Point-to-Point**         |
     | 제어 방식 | **분산 및 집중** 모두 가능 |
     | 전송 방향 | **단방향** 또는 **양방향** |
     | 중계기 수 | **많음**                   |

   - 링형 장단점

     | 장점                | 단점                                |
     | ------------------- | ----------------------------------- |
     | 분산/집중 제어 가능 | 단말 추가/제거 **어려움**           |
     | -                   | 기밀 보호 **어려움**                |
     | -                   | **전송 지연** 발생 가능             |
     | -                   | 단방향 시 하나 고장 → **전체 영향** |

5. **버스형 (Bus)**
   - **한 개의 통신 회선**에 여러 대의 단말장치가 연결된 구조
   - 버스형 특징

     | 특징           | 설명                 |
     | -------------- | -------------------- |
     | 구조           | **물리적 구조 간단** |
     | 단말 추가/제거 | **용이**             |
     | 회선 길이      | **제한 있음**        |

   - 버스형 장단점

     | 장점                            | 단점                 |
     | ------------------------------- | -------------------- |
     | 단말 고장 시 **전체 영향 없음** | **기밀 보장 어려움** |
     | **신뢰성 높음**                 | 회선 길이 제한       |
     | 구조 간단                       | -                    |

6. **계층형 (Tree, 분산형)**
   - 중앙 컴퓨터에서 중간 단말장치를 거쳐 **트리 구조**로 연결
   - 계층형 특징

     | 특징 | 설명                      |
     | ---- | ------------------------- |
     | 구조 | **트리 (나무)** 형태      |
     | 방식 | **분산 처리** 시스템 구성 |

7. **망형 (Mesh)**
   - 모든 컴퓨터와 단말장치를 **서로 연결**한 형태 (노드 연결성 높음)
   - 망형 특징

     | 특징   | 설명                   |
     | ------ | ---------------------- |
     | 연결성 | **가장 높음**          |
     | 경로   | **가장 김**            |
     | 용도   | **공중 데이터 통신망** |

   - 망형 장단점

     | 장점                            | 단점             |
     | ------------------------------- | ---------------- |
     | 장애 시 **다른 경로** 사용 가능 | 회선 수 **많음** |
     | 많은 통신량 처리 유리           | 비용 높음        |

   - 망형 회선 수 공식 (시험 필수!)

     ```
     노드 n개일 때:
     - 필요 회선 수 = n(n-1)/2
     - 노드당 포트 수 = n-1
     ```

8. **네트워크 구조 비교표**

   |    구조    |      장애 시 영향      | 추가/제거  | 특징                    |
   | :--------: | :--------------------: | :--------: | ----------------------- |
   |  **성형**  | 중앙 고장 시 전체 정지 |  **쉬움**  | 교환 노드 **가장 적음** |
   |  **링형**  |  단방향 시 전체 영향   | **어려움** | 중계기 많음             |
   | **버스형** |     전체 영향 없음     |  **쉬움**  | 신뢰성 높음             |
   |  **망형**  |     다른 경로 사용     |     -      | 경로 **가장 김**        |

9. **네트워크 분류 (LAN vs WAN)**

   |  구분  |        LAN         |        WAN        |
   | :----: | :----------------: | :---------------: |
   | 한글명 | **근거리** 통신망  | **광대역** 통신망 |
   |  범위  | 회사, 학교, 연구소 |    국가, 대륙     |
   |  속도  |      **빠름**      |       느림        |
   | 에러율 |      **낮음**      |       높음        |
   |  구조  |  **버스형, 링형**  |   LAN들을 연결    |
   |  목적  |   **자원 공유**    |    원거리 통신    |
   - VLAN (Virtual LAN)
     - LAN을 물리적 배치와 상관없이 **논리적으로 분리**하는 기술

     | 효과            |
     | --------------- |
     | **성능 향상**   |
     | **보안성 향상** |

10. **IEEE 802 표준 규격**
    - 유선·무선 LAN 통신을 방법을 정해놓은 국제 규칙 모음

      |    표준    | 내용                             |
      | :--------: | -------------------------------- |
      | **802.1**  | 전체 구성, OSI 관계, 통신망 관리 |
      | **802.2**  | **LLC** (논리 링크 제어) 계층    |
      | **802.3**  | **CSMA/CD** 방식                 |
      | **802.4**  | **토큰 버스** 방식               |
      | **802.5**  | **토큰 링** 방식                 |
      | **802.6**  | **MAN** (도시형 통신망)          |
      | **802.9**  | 종합 음성/데이터 네트워크        |
      | **802.11** | **무선 LAN**                     |

    - IEEE 802 암기
      - **802.3** = **CSMA/CD** = **이더넷**
      - **802.5** = **토큰 링**
      - **802.11** = **무선 LAN** = **Wi-Fi**

11. **IEEE 802.11 버전 (무선 LAN)**
    - Wi-Fi(무선 LAN) 통신 규칙을 정해놓은 표준

      |    버전     |  주파수  |    속도     | 특징                |
      | :---------: | :------: | :---------: | ------------------- |
      | **802.11**  |  2.4GHz  |    2Mbps    | **CSMA/CA**         |
      | **802.11a** | **5GHz** | **54Mbps**  | OFDM                |
      | **802.11b** |  2.4GHz  | **11Mbps**  | 초기 버전 개선      |
      | **802.11e** |    -     |      -      | **QoS** 지원        |
      | **802.11g** |  2.4GHz  | **54Mbps**  | -                   |
      | **802.11i** |    -     |      -      | **보안** (WPA/WPA2) |
      | **802.11n** | 2.4/5GHz | **600Mbps** | -                   |

12. **CSMA/CD vs CSMA/CA**
    - CSMA/CD
      - 유선 LAN에 사용하는 통신법
      - 동작 방식 : 채널이 비어 있는지 확인 > 데이터 전송 시작 > 충돌 발생 > 즉시 감지 (Detection) > 전송 중단 + 랜덤 시간 후 재전송
    - CSMA/CA
      - 무선 LAN에 사용하는 통신법
      - 동작 방식 : 채널이 비어 있는지 확인 > 비어 있어도 바로 안 보냄 > 랜덤 대기 시간(backoff) > 필요 시 RTS/CTS 요청 > 충돌을 미리 회피하고 전송

    | 구분 |         CSMA/CD         |           CSMA/CA            |
    | :--: | :---------------------: | :--------------------------: |
    | 의미 | Collision **Detection** |   Collision **Avoidance**    |
    | 한글 |      충돌 **검출**      |        충돌 **회피**         |
    | 용도 |      **유선 LAN**       |         **무선 LAN**         |
    | 표준 |        **802.3**        |          **802.11**          |
    | 방식 | 충돌 발생 후 **재송신** | 충돌 **예방** (대기 후 전송) |

### 3. 경로 제어 / 트래픽 제어

1. **경로 제어 (Routing)란?**

   > **한 줄 정의:** 송·수신 측 간의 전송 경로 중에서 **최적 패킷 교환 경로를 결정**하는 기능

2. **경로 제어 개요**

   | 항목 | 내용                            |
   | ---- | ------------------------------- |
   | 목적 | **최적 경로** 결정              |
   | 기준 | 최저 비용, 최단 시간            |
   | 참조 | **경로 제어표 (Routing Table)** |
   | 수행 | **라우터**                      |

3. **경로 제어 요소**
   - 성능 기준
   - 경로의 결정 시간과 장소
   - 정보 발생지
   - 경로 정보의 갱신 시간

4. **경로 제어 프로토콜**
   - 효율적인 경로 제어를 위해 네트워크 정보를 **생성, 교환, 제어**하는 프로토콜

     | 프로토콜 |    범위     | 특징                     |
     | :------: | :---------: | ------------------------ |
     | **IGP**  | **AS 내부** | 내부 게이트웨이 프로토콜 |
     | **EGP**  |  **AS 간**  | 외부 게이트웨이 프로토콜 |
     | **BGP**  |  **AS 간**  | EGP 보완                 |
     - AS (Autonomous System) = 자율 시스템
       - 하나의 조직(회사·기관)이 관리하는 네트워크 묶음
     - IGP (Interior Gateway Protocol)
       - 하나의 **자율 시스템(AS) 내부**의 라우팅에 사용되는 프로토콜
       - IGP 종류

         | 프로토콜 |     알고리즘     | 특징              |
         | :------: | :--------------: | ----------------- |
         | **RIP**  | **Bellman-Ford** | 거리 벡터, 소규모 |
         | **OSPF** |   **Dijkstra**   | 링크 상태, 대규모 |
         - RIP (Routing Information Protocol)
           - **거리 벡터** 라우팅 프로토콜, **Bellman-Ford** 알고리즘 사용
           - RIP 특징

             | 특징       | 설명                            |
             | ---------- | ------------------------------- |
             | 알고리즘   | **Bellman-Ford**                |
             | 방식       | **거리 벡터** (Distance Vector) |
             | 적합       | **소규모** 네트워크             |
             | 최대 홉 수 | **15** (초과 시 도달 불가)      |
             | 정보 갱신  | **30초**마다 모든 라우터에 알림 |
             | 이상 판단  | **180초** 내 미수신 시 이상     |

         - OSPF (Open Shortest Path First)
           - **링크 상태** 라우팅 프로토콜, **Dijkstra** 알고리즘 사용, **대규모** 네트워크
           - OSPF 특징

           | 특징      | 설명                       |
           | --------- | -------------------------- |
           | 알고리즘  | **Dijkstra** (다익스트라)  |
           | 방식      | **링크 상태** (Link State) |
           | 적합      | **대규모** 네트워크        |
           | 목적      | RIP **단점 해결**          |
           | 정보 갱신 | **변화된 정보만** 알림     |
           | 반영      | **실시간** 반영            |

     - EGP / BGP
       - EGP (Exterior Gateway Protocol)
         - **자율 시스템(AS) 간**의 라우팅, 게이트웨이 간 라우팅
       - BGP (Border Gateway Protocol)
         - AS 간 라우팅 프로토콜, **EGP 단점 보완**

         | 특징      | 설명                      |
         | --------- | ------------------------- |
         | 초기 연결 | **전체 경로 제어표** 교환 |
         | 이후      | **변화된 정보만** 교환    |

5. **트래픽 제어 (Traffic Control)**
   - 네트워크 보호와 성능 유지를 위해 **패킷의 흐름 또는 양을 조절**하는 기능
   - 트래픽 제어 종류

     |        종류         | 목적                            |
     | :-----------------: | ------------------------------- |
     |    **흐름 제어**    | 송·수신 측 간 패킷 양/속도 규제 |
     | **폭주(혼잡) 제어** | 네트워크 내 패킷 수 조절        |
     |  **교착상태 방지**  | 무한 대기 방지                  |
     - 흐름 제어 (Flow Control)
       - 송·수신 측 사이에 전송되는 **패킷의 양이나 속도를 규제**하는 기능
       - 흐름 제어 목적
         - **버퍼 오버플로 방지**
         - 처리 속도/버퍼 크기 차이 해결
       - 흐름 제어 기법

         |        기법         | 한 줄 정의                                  |
         | :-----------------: | ------------------------------------------- |
         |    **정지-대기**    | **ACK 받은 후** 다음 패킷 전송 (**하나씩**) |
         | **슬라이딩 윈도우** | **여러 개** 연속 전송, 윈도우 크기만큼      |
         - 정지-대기 (Stop-and-Wait)
           - 수신 측의 **확인 신호(ACK)를 받은 후** 다음 패킷을 전송하는 방식

           | 특징 | 설명                      |
           | ---- | ------------------------- |
           | 전송 | **한 번에 하나**의 패킷만 |
           | 효율 | 낮음                      |

         - 슬라이딩 윈도우 (Sliding Window)
           - 확인 신호 없이 **윈도우 크기만큼 연속 전송**하는 방식
           - 슬라이딩 윈도우 특징

             | 특징        | 설명                                  |
             | ----------- | ------------------------------------- |
             | 전송        | **여러 개** 연속 전송                 |
             | 효율        | **높음**                              |
             | 윈도우 크기 | ACK 없이 보낼 수 있는 **패킷 최대치** |
             - 윈도우 크기 변화

               |      응답      | 윈도우 크기 |
               | :------------: | :---------: |
               | **ACK** (긍정) |  **증가**   |
               | **NAK** (부정) |  **감소**   |

     - 폭주(혼잡) 제어 (Congestion Control)
       - **네트워크 내의 패킷 수를 조절**하여 오버플로를 방지하는 기능
       - 흐름 제어 vs 폭주 제어

         | 구분 |     흐름 제어      |       폭주 제어        |
         | :--: | :----------------: | :--------------------: |
         | 범위 | **송·수신 측 간**  |    **네트워크 내**     |
         | 목적 | 버퍼 오버플로 방지 | 네트워크 오버플로 방지 |

       - 폭주 제어 기법

         |     기법      | 한 줄 정의                          |
         | :-----------: | ----------------------------------- |
         | **느린 시작** | 윈도우 크기 **2배씩 지수적 증가**   |
         | **혼잡 회피** | 임계 값 도달 후 **1씩 선형적 증가** |
         - 느린 시작 vs 혼잡 회피

           |   구분    |     느린 시작      |      혼잡 회피      |
           | :-------: | :----------------: | :-----------------: |
           | 증가 방식 | **2배씩 (지수적)** |  **1씩 (선형적)**   |
           |   예시    |     1→2→4→8→16     |     16→17→18→19     |
           | 적용 시점 |        초기        | **임계 값 도달 후** |

     - 교착상태 (Dead Lock) 방지
       - 기억 공간이 꽉 차서 패킷들이 **무한정 기다리는 현상** 방지
       - 교착상태 방지 방법
         - 패킷이 **같은 목적지**를 갖지 않도록 할당
         - 교착상태 발생 시 **패킷 버퍼 폐기**

   - 블록체인 특징

     | 특징      | 설명                  |
     | --------- | --------------------- |
     | 네트워크  | **P2P**               |
     | 저장 방식 | **분산 저장**         |
     | 대상      | 온라인 금융 거래 정보 |

3. **분산 원장 기술 (DLT: Distributed Ledger Technology)**

   > **한 줄 정의:** **중앙 관리자 없이** P2P 망 참여자들에게 모든 거래 목록이 **분산 저장**되는 디지털 원장
   - DLT 특징

   | 특징        | 설명                         |
   | ----------- | ---------------------------- |
   | 중앙 관리자 | **없음**                     |
   | 저장        | **분산 저장**                |
   | 갱신        | 거래 발생 시 **지속적 갱신** |
   | 대표 사례   | **블록체인**                 |

4. **양자암호키 분배 (QKD: Quantum Key Distribution)**

   > **한 줄 정의:** **양자통신**을 위해 **비밀키를 분배**하여 관리하는 기술
   - QKD 특징

     | 특징 | 설명                             |
     | ---- | -------------------------------- |
     | 목적 | **비밀키** 안전하게 공유         |
     | 방식 | 양자암호키 분배 시스템 설치/운용 |

5. **프라이버시 강화 기술 (PET: Privacy Enhancing Technology)**
   - **개인정보 위험 관리** 기술, 암호화·익명화 등 **프라이버시 보호** 기술 통칭
   - PET 범위

     | 기술                          |
     | ----------------------------- |
     | **암호화**                    |
     | **익명화**                    |
     | 사용자 직접 개인정보 **통제** |

6. **비트로커 (BitLocker)**

   > **한 줄 정의:** **Windows 전용** 볼륨 **암호화** 기능
   - 비트로커 특징

   | 특징     | 설명                              |
   | -------- | --------------------------------- |
   | 지원     | **Windows 7**부터                 |
   | 모듈     | **TPM** (Trusted Platform Module) |
   | 알고리즘 | **AES-128**                       |

7. **개인정보 영향평가 제도 (PIA: Privacy Impact Assessment)**

   > **한 줄 정의:** 정보시스템이 **사생활에 미칠 영향**을 미리 **조사·분석·평가**하는 제도
   - PIA 적용 시점

     | 시점                             |
     | -------------------------------- |
     | 새로운 정보시스템 **도입** 시    |
     | 기존 정보시스템 **중요 변경** 시 |

8. **소프트웨어 에스크로 (Software Escrow, 임치)**

   > **한 줄 정의:** 소스 프로그램과 기술 정보를 **제3의 기관에 보관**하는 것
   - 소프트웨어 에스크로 목적

     |    대상    | 목적                                     |
     | :--------: | ---------------------------------------- |
     | **개발자** | **지식재산권 보호**                      |
     | **사용자** | 저렴한 비용으로 **안정적 사용/유지보수** |

9. **서비스형 블록체인 (BaaS: Blockchain as a Service)**

   > **한 줄 정의:** 블록체인 앱 개발 환경을 **클라우드 기반**으로 제공하는 서비스
   - BaaS 특징

     | 특징 | 설명                            |
     | ---- | ------------------------------- |
     | 기반 | **클라우드**                    |
     | 장점 | 노드 추가/제거 **용이**         |
     | 효과 | 블록체인 기술 **편리하게** 사용 |

10. **OWASP**

    > **한 줄 정의:** 웹 보안 **취약점을 연구**하는 **비영리 단체**, **10대 취약점** 발표
    - OWASP 특징

      | 특징      | 설명                                |
      | --------- | ----------------------------------- |
      | 성격      | **비영리 단체**                     |
      | 연구 대상 | 웹 정보 노출, 악성코드, 보안 취약점 |
      | 발표      | **10대 웹 취약점** (**3~4년**마다)  |

11. **TCP 래퍼 (TCP Wrapper)**

    > **한 줄 정의:** 외부 컴퓨터의 **접속 인가 여부를 점검**하여 접속을 허용/거부하는 **보안 도구**
    - TCP 래퍼 특징

      | 특징 | 설명                     |
      | ---- | ------------------------ |
      | 역할 | **방화벽** 역할 수행     |
      | 기능 | 접속 **허용/거부**       |
      | 조회 | 사용자 ID, **로그** 조회 |

12. **허니팟 (Honeypot)**

    > **한 줄 정의:** **비정상적인 접근을 탐지**하기 위해 설치해 둔 **미끼 시스템**
    - 허니팟 특징

      | 특징 | 설명                               |
      | ---- | ---------------------------------- |
      | 목적 | **침입자 속이기**                  |
      | 역할 | 실제 공격당하는 것처럼 보여줌      |
      | 효과 | **추적** 및 **공격기법 정보 수집** |

13. **DPI (Deep Packet Inspection)**

    > **한 줄 정의:** OSI **전 계층**의 프로토콜과 **패킷 내부 콘텐츠**를 파악하여 침입/해킹을 탐지하는 **패킷 분석 기술**
    - DPI 특징

      | 특징      | 설명                                  |
      | --------- | ------------------------------------- |
      | 분석 범위 | **OSI 7 Layer 전 계층**               |
      | 분석 대상 | 프로토콜, **패킷 내부 콘텐츠**        |
      | 기능      | 침입 탐지, 해킹 탐지, **트래픽 조정** |

### 6. HW 관련 신기술

1. **HW 관련 신기술 분류**

   |     분류      | 기술                     |
   | :-----------: | ------------------------ |
   | **고가용성**  | HACMP                    |
   | **출력 기술** | 3D Printing, 4D Printing |
   | **저장 기술** | RAID                     |
   |  **서비스**   | N-Screen                 |

2. **고가용성 솔루션 (HACMP)**

   > **한 줄 정의:** 장애 발생 시 **즉시 다른 시스템으로 대체** 가능한 환경을 구축하는 메커니즘
   - HACMP 특징

     | 특징 | 설명                                |
     | ---- | ----------------------------------- |
     | 목적 | **안정적인 서비스 운영**            |
     | 구성 | **공유 디스크** 중심 **클러스터링** |
     | 방식 | **이중화** (2개 서버 연결)          |
     | 효과 | 서버 **안정성** 향상                |

3. **3D Printing**

   > **한 줄 정의:** 대상을 **실제 물체**로 만들어내는 기술, **얇은 층을 적층**하여 형태 생성
   - 3D Printing 특징

     | 특징 | 설명                              |
     | ---- | --------------------------------- |
     | 방식 | 얇은 두께로 **한층한층 적층**     |
     | 결과 | 손으로 만질 수 있는 **실제 물체** |

   - 3D Printing 활용 분야

     | 분야          |
     | ------------- |
     | **건축**      |
     | **항공우주**  |
     | **전자**      |
     | **공구 제조** |
     | **자동차**    |
     | **디자인**    |
     | **의료**      |

4. **4D Printing**

   > **한 줄 정의:** 특정 환경 조건에서 **스스로 형태를 변화**시키는 **자가 조립(Self-Assembly)** 기술이 적용된 3D Printing
   - 4D Printing 특징

     | 특징      | 설명                                      |
     | --------- | ----------------------------------------- |
     | 핵심 기술 | **자가 조립 (Self-Assembly)**             |
     | 변화 조건 | **열, 진동, 습도, 중력** 등               |
     | 소재      | **스마트 소재** (형상기억합금, 나노 기술) |

   - 4D Printing 역사

     | 항목      | 내용                                         |
     | --------- | -------------------------------------------- |
     | 최초 공개 | **2013년 4월** TED 강연                      |
     | 발표자    | MIT **스카일러 티빗츠** 교수                 |
     | 연구소    | MIT **자가 조립 연구소** (Self-Assembly Lab) |

   - 4D Printing 제품 예시

   | 제품                                            |
   | ----------------------------------------------- |
   | **종이접기 로봇** (열을 가하면 접힘)            |
   | **키네메틱스 드레스** (접힌 상태 → 완전한 형태) |

5. **RAID (Redundant Array of Inexpensive/Independent Disk)**

   > **한 줄 정의:** 여러 개의 **하드디스크로 배열을 구성**하여 데이터를 **분산 저장**하고 **속도 향상** 및 **오류 복구**를 가능하게 하는 기술
   - RAID 특징

     | 특징   | 설명                                    |
     | ------ | --------------------------------------- |
     | 구성   | **여러 개의 하드디스크** 배열           |
     | 저장   | 데이터 블록 **분산 저장**               |
     | 장점   | 디스크 **속도 향상**                    |
     | 문제점 | 한 디스크 결함 → **전체 데이터 손상**   |
     | 해결   | **여분의 디스크** 추가 (오류 검출/복구) |

   - RAID 종류

     |    종류    | 특징                             |
     | :--------: | -------------------------------- |
     | **RAID 0** | 스트라이핑, 속도 향상, 복구 불가 |
     | **RAID 1** | **미러링**, 동일 데이터 복제     |
     | **RAID 2** | 해밍 코드 사용                   |
     | **RAID 3** | 패리티 디스크 1개 (바이트 단위)  |
     | **RAID 4** | 패리티 디스크 1개 (블록 단위)    |
     | **RAID 5** | **분산 패리티**, 가장 많이 사용  |

6. **N-Screen (앤 스크린)**

   > **한 줄 정의:** **N개의 서로 다른 단말기**에서 **동일한 콘텐츠**를 **끊김 없이** 이용할 수 있는 서비스
   - N-Screen 특징

     | 특징   | 설명                             |
     | ------ | -------------------------------- |
     | 핵심   | **동일 콘텐츠**, **여러 단말기** |
     | 연속성 | **끊김 없이** 이용               |
     | 단말기 | PC, TV, 휴대폰 등                |

   - N-Screen 예시
     - PC에서 보던 영상을 **TV에서 이어서** 시청
     - 휴대폰에서 보던 콘텐츠를 **태블릿에서 계속** 이용

### 7. Secure OS

1. **Secure OS란?**

   > **한 줄 정의:** 기존 OS의 **보안 취약점을 해소**하기 위해 **보안 기능을 갖춘 커널**을 이식하여 시스템 자원을 보호하는 **운영체제**

2. **Secure OS 개요**

   | 항목 | 내용                                   |
   | ---- | -------------------------------------- |
   | 목적 | **보안 취약점 해소**, 시스템 자원 보호 |
   | 핵심 | **보안 커널** 이식                     |
   | 기반 | **TCB**, **참조 모니터**               |

3. **보안 커널 (Security Kernel)**

   > **한 줄 정의:** **보안 기능을 갖춘 커널**, TCB 기반으로 **참조 모니터**의 개념을 구현
   - 보안 커널 보호 대상

     | 보호 대상              |
     | ---------------------- |
     | **메모리**             |
     | **보조기억장치**       |
     | 저장된 **데이터**      |
     | **하드웨어 장치**      |
     | **자료 구조**          |
     | **명령어**             |
     | 각종 **보호 메커니즘** |

4. 보호 방법 (복잡한 순서)

   | 순서 |    분리 방법    | 한 줄 정의                                      |
   | :--: | :-------------: | ----------------------------------------------- |
   |  1   | **암호적 분리** | 내부 정보를 **암호화**                          |
   |  2   | **논리적 분리** | 프로세스의 **논리적 구역** 지정, 구역 이탈 제한 |
   |  3   | **시간적 분리** | **동일 시간**에 **하나의 프로세스**만 수행      |
   |  4   | **물리적 분리** | 사용자별로 **특정 장비**만 사용                 |

5. **참조 모니터 (Reference Monitor)**

   > **한 줄 정의:** 보호대상 객체에 대한 **접근통제를 수행**하는 **추상머신**
   - 참조 모니터 특징

     | 특징 | 설명                              |
     | ---- | --------------------------------- |
     | 참조 | **SKDB** (보안 커널 데이터베이스) |
     | 역할 | 객체 **접근 허가 여부** 결정      |
     | 구현 | **보안 커널**이 실제 구현         |

   - 참조 모니터 / 보안 커널 3가지 특징

     |      특징      |     영문      | 설명                      |
     | :------------: | :-----------: | ------------------------- |
     |   **격리성**   |   Isolation   | **부정 조작 불가능**      |
     | **검증가능성** | Verifiability | 적절히 구현 **확인 가능** |
     |   **완전성**   | Completeness  | **우회 불가능**           |

6. **Secure OS 보안 기능**

   |         보안 기능         | 한 줄 정의                               |
   | :-----------------------: | ---------------------------------------- |
   |     **식별 및 인증**      | 접근 주체의 **고유한 식별/인증**         |
   | **임의적 접근통제 (DAC)** | **소속/개인**에 따라 접근 통제           |
   | **강제적 접근통제 (MAC)** | **보안 속성 규칙**에 따라 **강제** 통제  |
   |   **객체 재사용 보호**    | 메모리 기존 데이터 **초기화**            |
   |      **완전한 조정**      | 모든 접근 경로 **완전하게 통제**         |
   |       **신뢰 경로**       | 보안 작업을 위한 **안전한 경로** 제공    |
   |   **감사 및 감사기록**    | 보안 사건/작업 **기록(Log)** 및 **축소** |

7. **DAC vs MAC**

   |   구분    |           DAC (임의적)           |         MAC (강제적)         |
   | :-------: | :------------------------------: | :--------------------------: |
   |   영문    | **D**iscretionary Access Control | **M**andatory Access Control |
   | 다른 이름 |        **신분 기반** 정책        |      **규칙 기반** 정책      |
   |   기준    |        **소속 그룹/개인**        |      **보안 속성 규칙**      |
   | 통제 방식 |        부여된 권한에 따라        |        **강제적**으로        |

8. **Secure OS 보안 기능 상세**
   1. 식별 및 인증 : 각 접근 주체에 대한 **안전하고 고유한** 식별 및 인증
   2. 임의적 접근통제 (DAC) : **소속 그룹 또는 개인**에 따라 부여된 권한에 따라 접근 통제
   3. 강제적 접근통제 (MAC) : 접속 단말, 권한, 객체 특성 등 **여러 보안 속성 규칙**에 따라 **강제적** 통제
   4. 객체 재사용 보호 : 메모리에 **기존 데이터가 남아있지 않도록 초기화**
   5. 완전한 조정 : **우회 불가능**하도록 모든 접근 경로를 **완전하게 통제**
   6. 신뢰 경로 : 비밀번호 변경, 권한 설정 등 보안 작업을 위한 **안전한 경로** 제공
   7. 감사 및 감사기록 축소 : 모든 보안 관련 사건/작업을 **기록(Log)** 후 **보호**, 막대한 양의 기록을 **분석하고 축소**

9. **Secure OS 구성 요소 관계**

   ```
   ┌─────────────────────────────────────┐
   │           Secure OS                 │
   ├─────────────────────────────────────┤
   │  ┌─────────────────────────────┐   │
   │  │       보안 커널              │   │
   │  │  (참조 모니터 구현)          │   │
   │  │                             │   │
   │  │  ┌─────────────────────┐   │   │
   │  │  │   참조 모니터        │   │   │
   │  │  │  (접근통제 수행)     │   │   │
   │  │  └─────────────────────┘   │   │
   │  │           ↓                │   │
   │  │  ┌─────────────────────┐   │   │
   │  │  │   SKDB              │   │   │
   │  │  │  (보안 커널 DB)      │   │   │
   │  │  └─────────────────────┘   │   │
   │  └─────────────────────────────┘   │
   └─────────────────────────────────────┘
   ```

### 8. DB 관련 신기술

1. **DB 관련 신기술 분류**

   |      분류       | 기술                                 |
   | :-------------: | ------------------------------------ |
   | **데이터 유형** | 빅데이터, 브로드 데이터, 메타 데이터 |
   |  **분산 처리**  | 하둡, 맵리듀스, 타조                 |
   | **데이터 관리** | 디지털 아카이빙, 데이터 다이어트     |
   | **데이터 분석** | 데이터 마이닝, OLAP                  |

2. **빅데이터 (Big Data)**

   > **한 줄 정의:** 기존 관리/분석 체계로는 처리하기 어려운 **막대한 양의 정형/비정형 데이터** 집합
   - 빅데이터 특징

     | 특징 | 설명                                           |
     | ---- | ---------------------------------------------- |
     | 원인 | 스마트 단말, SNS, 사물 네트워크 확대           |
     | 목적 | **미래 예측**, 최적 대응, **새로운 가치 창출** |

   - 빅데이터 3V
     - **Volume** (규모)
     - **Velocity** (속도)
     - **Variety** (다양성)

3. **브로드 데이터 (Broad Data)**

   > **한 줄 정의:** 다양한 채널에서 소비자 상호 작용으로 생성된 **효율적이고 다양한** 마케팅 데이터
   - 빅데이터 vs 브로드 데이터

     | 구분 |    빅데이터     |    브로드 데이터    |
     | :--: | :-------------: | :-----------------: |
     | 핵심 | **대량**의 자료 |   **다양한** 정보   |
     | 예시 |   대용량 로그   | SNS 활동, 위치 정보 |

4. **메타 데이터 (Meta Data)**

   > **한 줄 정의:** 일련의 데이터를 **정의하고 설명**해 주는 **데이터에 대한 데이터**
   - 메타 데이터 예시

     |    분야    | 메타 데이터 예시              |
     | :--------: | ----------------------------- |
     | **컴퓨터** | **데이터 사전**, **스키마**   |
     |  **HTML**  | **메타 태그** 내용            |
     |  **방송**  | 촬영 일시, 장소, 작가, 출연자 |
     |  **음원**  | 작곡자, 가수명                |

   - 메타 데이터 용도
     - **빠른 검색**
     - 내용의 **간략화**, **체계화**

5. **디지털 아카이빙 (Digital Archiving)**

   > **한 줄 정의:** 디지털 정보 자원을 **장기적으로 보존**하기 위한 작업
   - 디지털 아카이빙 작업

     |        대상         | 작업                                       |
     | :-----------------: | ------------------------------------------ |
     | **아날로그** 콘텐츠 | 디지털 **변환** → **압축** → 저장          |
     |  **디지털** 콘텐츠  | **분류** → **메타 데이터** 생성 → **DB화** |

6. **하둡 (Hadoop)**

   > **한 줄 정의:** **오픈 소스** 기반의 **분산 컴퓨팅 플랫폼**, 대용량 데이터 **병렬 처리**
   - 하둡 특징

     | 특징 | 설명                                          |
     | ---- | --------------------------------------------- |
     | 기반 | **오픈 소스**, **자바**                       |
     | 구성 | 일반 PC급 컴퓨터로 **가상화된 대형 스토리지** |
     | 처리 | **병렬 처리**                                 |
     | 적용 | 구글, 야후 등                                 |

   - 하둡 관련 도구

     |       도구       | 역할                                  |
     | :--------------: | ------------------------------------- |
     | **스쿱 (Sqoop)** | 하둡 ↔ RDB 간 **대용량 데이터 전송** |

7. **맵리듀스 (MapReduce)**

   > **한 줄 정의:** **대용량 데이터 분산 처리**를 위한 프로그래밍 모델, **Google**이 개발
   - 맵리듀스 처리 과정

     | 단계 |    작업    | 설명                                          |
     | :--: | :--------: | --------------------------------------------- |
     |  1   |  **Map**   | 흩어진 데이터를 **연관성 있는 데이터로 분류** |
     |  2   | **Reduce** | **중복 제거** + 원하는 데이터 **추출**        |

8. **타조 (Tajo)**

   > **한 줄 정의:** **아파치 하둡** 기반 **분산 데이터 웨어하우스**, **한국** 주도 개발
   - 타조 특징

     | 특징 | 설명                                     |
     | ---- | ---------------------------------------- |
     | 기반 | **아파치 하둡 (Apache Hadoop)**          |
     | 개발 | **한국** 주도                            |
     | 특징 | **맵리듀스 사용 안 함**                  |
     | 방식 | **SQL** 사용하여 **HDFS** 파일 직접 읽기 |
     | 용도 | 대규모 데이터 처리, **실시간 상호 분석** |

9. **데이터 다이어트 (Data Diet)**

   > **한 줄 정의:** 데이터를 **삭제하지 않고** **압축**, **중복 배제**, 새로운 기준으로 **분리 저장**
   - 데이터 다이어트 특징

   | 특징   | 설명                                     |
   | ------ | ---------------------------------------- |
   | 핵심   | 삭제 ❌, **압축/중복 배제/분리 저장** ⭕ |
   | 목적   | 방대한 정보의 **효율적 관리**            |
   | 중요점 | 필요할 때 **제대로 찾아내는 체계**       |

10. **데이터 마이닝 (Data Mining)**

    > **한 줄 정의:** 데이터 웨어하우스에서 **유용하고 가능성 있는 정보를 발견**하기 위한 기법
    - 데이터 마이닝 특징

      | 특징 | 설명                                   |
      | ---- | -------------------------------------- |
      | 대상 | **데이터 웨어하우스**                  |
      | 방법 | 변수 간 **상호관계 규명** → **패턴화** |
      | 목적 | **효율적인 데이터 추출**               |

11. **OLAP (Online Analytical Processing)**

    > **한 줄 정의:** **다차원 데이터**로부터 통계적 요약 정보를 분석하여 **의사결정에 활용**
    - OLAP 특징

      | 특징        | 설명                                   |
      | ----------- | -------------------------------------- |
      | 데이터      | **다차원** 데이터                      |
      | 분석        | **통계적 요약** 정보                   |
      | 목적        | **의사결정** 활용                      |
      | 연관 시스템 | **데이터 웨어하우스**, **데이터 마트** |

    - OLAP 연산 (시험 필수!)

      |       연산        | 설명                       |
      | :---------------: | -------------------------- |
      |    **Roll-up**    | 요약 (하위 → 상위)         |
      |  **Drill-down**   | 상세 (상위 → 하위)         |
      | **Drill-through** | 다른 데이터 소스로 이동    |
      | **Drill-across**  | 다른 큐브로 이동           |
      |   **Pivoting**    | 축 회전                    |
      |    **Slicing**    | 한 차원 고정 (2차원 단면)  |
      |    **Dicing**     | 여러 차원 고정 (부분 큐브) |

### 9. 회복 / 병행제어

1. **회복 (Recovery)이란?**

   > **한 줄 정의:** 장애 발생 시 데이터베이스를 **손상되기 이전의 정상 상태로 복구**하는 작업

2. **장애의 유형**

   |     장애 유형     | 한 줄 정의                                          |
   | :---------------: | --------------------------------------------------- |
   | **트랜잭션 장애** | **입력 오류, 시스템 자원 과다** 등 내부 비정상 상황 |
   |  **시스템 장애**  | **하드웨어 오동작, 소프트웨어 손상, 교착상태** 등   |
   |  **미디어 장애**  | 디스크 블록 손상, 헤드 충돌 등 **물리적 손상**      |

3. 장애 유형 상세

   |     장애     | 특징                           |
   | :----------: | ------------------------------ |
   | **트랜잭션** | 프로그램 실행 **중지**         |
   |  **시스템**  | DB 손상 ❌, 연속 수행 **장애** |
   |  **미디어**  | DB **물리적 손상** (일부/전부) |

4. **회복 관리기 (Recovery Management)**

   > **한 줄 정의:** 트랜잭션 실패 시 모든 변화를 **취소(Undo)**하고 **원래 상태로 복구**하는 DBMS 구성 요소
   - 회복 관리기 특징

   | 특징 | 설명                              |
   | ---- | --------------------------------- |
   | 위치 | **DBMS** 구성 요소                |
   | 역할 | 실패 시 **Undo** → 원래 상태 복구 |
   | 이용 | **메모리 덤프**, **로그(Log)**    |

5. **회복 기법**

   |       기법        | 한 줄 정의                             |
   | :---------------: | -------------------------------------- |
   |   **연기 갱신**   | 완료 때까지 **갱신 연기**, Redo만      |
   |   **즉각 갱신**   | 완료 전이라도 **즉시 반영**, Redo+Undo |
   | **그림자 페이지** | **복사본** 보관 후 대체                |
   |    **검사점**     | **검사점부터 회복**, 시간 절약         |
   1. 연기 갱신 기법 (Deferred Update)
      - 트랜잭션 **성공적 완료 때까지** 실질적인 갱신을 **연기**하는 방법
      - 연기 갱신 특징

        | 특징        | 설명                              |
        | ----------- | --------------------------------- |
        | 갱신 시점   | **부분 완료** 시점에 실제 DB 기록 |
        | 수행 중     | **Log**에 갱신 내용 보관          |
        | Rollback 시 | **Undo 불필요** (DB에 영향 없음)  |
        | 가능 작업   | **Redo만** 가능                   |

   2. 즉각 갱신 기법 (Immediate Update)
      - 부분 완료 **전이라도 즉시** 실제 데이터베이스에 **반영**하는 방법
      - 즉각 갱신 특징

        | 특징      | 설명                      |
        | --------- | ------------------------- |
        | 갱신 시점 | **즉시** 실제 DB 반영     |
        | 수행 중   | **Log**에 갱신 내용 보관  |
        | 가능 작업 | **Redo + Undo** 모두 가능 |

   3. 그림자 페이지 대체 기법 (Shadow Paging)
      - 갱신 이전 DB를 **그림자 페이지(복사본)**로 보관하고 장애 시 **대체**하여 회복
      - 그림자 페이지 특징

        | 특징   | 설명                                 |
        | ------ | ------------------------------------ |
        | 단위   | **페이지 단위**                      |
        | 보관   | **그림자 페이지** (복사본)           |
        | 회복   | 실제 페이지에 그림자 페이지 **대체** |
        | 불필요 | **Log, Undo, Redo** 모두 불필요      |

   4. 검사점 기법 (Check Point)
      - **검사점(Check Point)**을 로그에 보관하고 장애 시 **검사점부터 회복**하여 시간 절약
      - 검사점 기법 특징

        | 특징      | 설명                        |
        | --------- | --------------------------- |
        | 보관 내용 | 갱신 내용, 시스템 상황 정보 |
        | 회복 범위 | 전체 ❌, **검사점부터**     |
        | 장점      | **회복 시간 절약**          |

6. **병행제어 (Concurrency Control)란?**

   > **한 줄 정의:** 여러 트랜잭션을 **동시 수행**할 때 DB **일관성을 파괴하지 않도록** 상호 작용을 **제어**하는 것
   - 병행제어의 목적

     | 목적                         |
     | ---------------------------- |
     | 데이터베이스 **공유 최대화** |
     | 시스템 **활용도 최대화**     |
     | 데이터베이스 **일관성 유지** |
     | 사용자 **응답 시간 최소화**  |

   - 병행제어 기법

     |         기법         | 한 줄 정의                           |
     | :------------------: | ------------------------------------ |
     |       **로킹**       | **Lock** 허락 후 액세스, 상호 배타적 |
     | **타임 스탬프 순서** | **시간표** 부여, 교착상태 없음       |
     |  **최적 병행수행**   | **Read Only** 트랜잭션, 낙관적 기법  |
     |    **다중 버전**     | **버전** 부여하여 관리               |
     1. 로킹 (Locking)
        - 트랜잭션이 **Lock을 요청**해서 **허락되어야만** 액세스할 수 있도록 하는 기법
        - 로킹 특징

          | 특징 | 설명                                  |
          | ---- | ------------------------------------- |
          | 방식 | **상호 배타적** 액세스                |
          | 절차 | **Lock 요청** → **허락** → **액세스** |

     2. 타임 스탬프 순서 (Time Stamp Ordering)
        - 트랜잭션에 **시간표(Time Stamp)**를 부여하여 **순서대로** 작업 수행하는 기법
        - 타임 스탬프 특징

          | 특징 | 설명                          |
          | ---- | ----------------------------- |
          | 방식 | 실행 전 **시간표** 부여       |
          | 순서 | 부여된 **시간 순서**대로 작업 |
          | 장점 | **교착상태 발생 안 함**       |

     3. 최적 병행수행 (검증 기법, 확인 기법, 낙관적 기법)
        - 대부분 **Read Only** 트랜잭션일 경우 충돌률이 낮아 병행제어 없이 실행하는 기법
        - 최적 병행수행 다른 이름
          - **검증 기법**
          - **확인 기법**
          - **낙관적 기법**
     4. 다중 버전 기법
        - 갱신될 때마다 **버전을 부여**하여 관리하는 기법
        - 타임 스탬프 vs 다중 버전

          |   구분    |  타임 스탬프   |     다중 버전     |
          | :-------: | :------------: | :---------------: |
          | 관리 대상 |    **시간**    |     **버전**      |
          |   방식    | 이용 시간 관리 | 갱신 시 버전 부여 |

7. 로킹 단위 (Locking Granularity)

   > **한 줄 정의:** 병행제어에서 한꺼번에 **로킹할 수 있는 객체의 크기**
   - 로킹 단위 종류

     | 단위             |
     | ---------------- |
     | **데이터베이스** |
     | **파일**         |
     | **레코드**       |
     | **필드**         |

   - 로킹 단위 크기에 따른 특징 (시험 필수!)

     |  구분   | 로킹 단위 큼 |      로킹 단위 작음      |
     | :-----: | :----------: | :----------------------: |
     | 로크 수 |   **적음**   |         **많음**         |
     |  관리   |   **쉬움**   | **복잡** (오버헤드 증가) |
     | 병행성  |   **낮음**   |         **높음**         |

### 10. 교착상태

1. **교착상태란?**
   > **한 줄 정의:** 둘 이상의 프로세스들이 자원을 점유한 상태에서 **서로 다른 프로세스의 자원을 요구하며 무한정 기다리는 현상**
2. **교착상태 개요**

   | 항목 | 내용                                       |
   | ---- | ------------------------------------------ |
   | 원인 | **상호 배제**에 의해 발생                  |
   | 상황 | 자원 점유 + 다른 자원 요구 + **무한 대기** |

3. 교착상태 비유
   - 자동차 = **프로세스**
   - 도로 = **자원**
   - 현재 위치 **점유** + 다른 도로 **대기** → 움직일 수 없음

4. **교착상태 발생 필요 충분 조건**
   - **4가지 조건 모두 충족** 시 교착상태 발생
   - **하나라도 미충족** 시 교착상태 발생 ❌

     |      조건       |       영문       | 한 줄 정의                            |
     | :-------------: | :--------------: | ------------------------------------- |
     |  **상호 배제**  | Mutual Exclusion | **한 번에 한 프로세스**만 자원 사용   |
     | **점유와 대기** |  Hold and Wait   | 자원 **점유** + 다른 자원 **대기**    |
     |   **비선점**    |  Non-preemption  | 사용 끝날 때까지 **강제 빼앗기 불가** |
     |  **환형 대기**  |  Circular Wait   | 프로세스들이 **원형으로 대기**        |

5. **교착상태 조건 상세**
   1. 상호 배제 (Mutual Exclusion)
      - 한 번에 **한 개의 프로세스만** 공유 자원 사용 가능

        | 특징          | 설명                        |
        | ------------- | --------------------------- |
        | 조건          | **한 프로세스**만 자원 사용 |
        | 다른 프로세스 | 사용 **불가**               |

   2. 점유와 대기 (Hold and Wait)
      - 자원을 **점유**하면서 다른 자원을 **대기**

        | 특징 | 설명                                    |
        | ---- | --------------------------------------- |
        | 점유 | 최소 **하나의 자원** 보유               |
        | 대기 | 다른 프로세스의 자원 **추가 점유 대기** |

   3. 비선점 (Non-preemption)
      - 다른 프로세스의 자원을 **강제로 빼앗을 수 없음**

        | 특징 | 설명                             |
        | ---- | -------------------------------- |
        | 조건 | 사용 **끝날 때까지** 빼앗기 불가 |
        | 방식 | **자발적 반납**만 가능           |

   4. 환형 대기 (Circular Wait)
      - 프로세스들이 **원형(순환)**으로 자원을 요구하며 대기

      | 특징 | 설명                                     |
      | ---- | ---------------------------------------- |
      | 구성 | 프로세스들이 **원형** 구조               |
      | 상태 | 자원 점유 + 앞/뒤 프로세스 자원 **요구** |

6. **교착상태 해결 방법**

   |   방법   |    영문    | 한 줄 정의                           |
   | :------: | :--------: | ------------------------------------ |
   | **예방** | Prevention | **사전에 조건 제거**하여 발생 방지   |
   | **회피** | Avoidance  | 발생 가능성 있어도 **적절히 피해감** |
   | **발견** | Detection  | 교착상태 **발생 여부 점검**          |
   | **회복** |  Recovery  | 교착상태 **프로세스 종료/자원 선점** |
   1. 예방 기법 (Prevention)
      - 교착상태 **4가지 조건 중 하나를 제거**(부정)하여 사전에 제어
      - 예방 기법 특징

        | 특징 | 설명                        |
        | ---- | --------------------------- |
        | 방법 | 4가지 조건 중 **하나 제거** |
        | 단점 | **자원 낭비 가장 심함**     |

   2. 회피 기법 (Avoidance)
      - 교착상태 발생 가능성을 배제하지 않고 **적절히 피해가는** 방법
      - 회피 기법 특징

        | 특징          | 설명                                     |
        | ------------- | ---------------------------------------- |
        | 방법          | 발생하면 **피해감**                      |
        | 대표 알고리즘 | **은행원 알고리즘 (Banker's Algorithm)** |

      - 은행원 알고리즘 (Banker's Algorithm)

        | 특징   | 설명                                         |
        | ------ | -------------------------------------------- |
        | 제안자 | **E. J. Dijkstra** (다익스트라)              |
        | 유래   | 은행에서 모든 고객 요구 충족하도록 현금 할당 |
        | 방식   | **안전 상태** 유지하며 자원 할당             |

   3. 발견 기법 (Detection)
      - 교착상태 **발생 여부를 점검**하여 프로세스와 자원을 발견
      - 발견 기법 도구

        | 도구                       |
        | -------------------------- |
        | 교착상태 **발견 알고리즘** |
        | **자원 할당 그래프**       |

   4. 회복 기법 (Recovery)
      - 교착상태 **프로세스를 종료**하거나 **자원을 선점**하여 회복
      - 회복 기법 방법

        |       방법        | 설명                            |
        | :---------------: | ------------------------------- |
        | **프로세스 종료** | 교착상태 프로세스 **강제 종료** |
        |   **자원 선점**   | 할당된 자원 **강제 빼앗기**     |
