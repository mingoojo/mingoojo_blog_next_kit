### 7. 프로세스의 개요

1. **프로세스(Process)란?**

   > **한 줄 정의:** 프로세서(CPU)에 의해 처리되는 **실행중인 프로그램**

2. **프로세스의 다양한 정의**

   | 정의                                               |
   | -------------------------------------------------- |
   | **PCB**를 가진 프로그램                            |
   | **실기억장치**에 저장된 프로그램                   |
   | 프로세서가 할당되는 실체, **디스패치** 가능한 단위 |
   | **프로시저**가 활동중인 것                         |
   | **비동기적 행위**를 일으키는 주체                  |
   | 운영체제가 관리하는 **실행 단위**                  |
   - 프로세스 = 작업(Job) = 태스크(Task)

3. **PCB (Process Control Block)**

   > **한 줄 정의:** 운영체제가 **프로세스에 대한 중요한 정보를 저장**해 놓는 곳
   - 다른 이름
     - **Task Control Block** (TCB)
     - **Job Control Block** (JCB)
   - PCB 특징
     - 프로세스 **생성** 시 → PCB **생성**
     - 프로세스 **완료** 시 → PCB **제거**
   - PCB 저장 정보

     | 저장 정보                | 설명                                  |
     | ------------------------ | ------------------------------------- |
     | **프로세스 현재 상태**   | 준비, 대기, 실행 등                   |
     | **포인터**               | 부모/자식 프로세스, 메모리, 자원 주소 |
     | **프로세스 고유 식별자** | 프로세스 구분 **고유 번호 (PID)**     |
     | **스케줄링 및 우선순위** | 스케줄링 정보, 실행 우선순위          |
     | **CPU 레지스터 정보**    | 누산기, 인덱스, PC 등                 |
     | **주기억장치 관리 정보** | 기준 레지스터, 페이지 테이블          |
     | **입·출력 상태 정보**    | 입출력장치, 개방된 파일 목록          |
     | **계정 정보**            | CPU 사용시간, 실제 사용 시간          |

4. **프로세스 상태 전이**
   - 프로세스가 시스템 내에 존재하는 동안 프로세스의 상태가 변하는 것을 의미
   - 프로세스 상태 5가지

     |         상태          | 한 줄 정의                        |
     | :-------------------: | --------------------------------- |
     |   **제출** (Submit)   | 사용자가 작업을 **시스템에 제출** |
     |    **접수** (Hold)    | 스풀 공간(디스크)에 **저장**      |
     |   **준비** (Ready)    | 프로세서 할당받기 위해 **대기**   |
     |    **실행** (Run)     | 프로세서 할당받아 **실행 중**     |
     | **대기** (Wait/Block) | **입·출력 처리** 완료까지 대기    |
     |    **종료** (Exit)    | 실행 끝, **할당 해제**            |

   - 주요 3가지 상태
     > **준비 → 실행 → 대기**

5. **프로세스 상태 전이도**

```
                   Dispatch
        ┌─────────────────────────────┐
        │                             ▼
    [준비] ◄──────────────────────── [실행]
      Ready                           Run
        ▲                             │
        │                             │ I/O 발생
        │         Wake Up             ▼
        └──────────────────────────[대기]
                                   Wait/Block
```

6. **상태 전이 관련 용어**

   |       용어        | 한 줄 정의                             |    전이     |
   | :---------------: | -------------------------------------- | :---------: |
   |   **Dispatch**    | 프로세서 할당받아 **실행** 상태로 전이 | 준비 → 실행 |
   | **Timer Run Out** | 할당 시간 **종료**                     | 실행 → 준비 |
   |     **Block**     | **입·출력** 처리 필요                  | 실행 → 대기 |
   |    **Wake Up**    | 입·출력 **완료**                       | 대기 → 준비 |

7. **기타 용어**

   |       용어        | 한 줄 정의                                              |
   | :---------------: | ------------------------------------------------------- |
   |   **Spooling**    | 입출력 데이터를 **디스크에 임시 저장** 후 한꺼번에 처리 |
   | **교통량 제어기** | 프로세스 상태 **조사와 통보** 담당                      |

8. **스레드 (Thread)**

   > **한 줄 정의:** 프로세스 내에서의 **작업 단위**, 시스템 자원을 할당받아 실행하는 **프로그램 단위**
   - 스레드 종류

     |      종류       | 한 줄 정의                      |
     | :-------------: | ------------------------------- |
     | **단일 스레드** | 프로세스 내 스레드 **1개**      |
     | **다중 스레드** | 프로세스 내 스레드 **2개 이상** |

   - 스레드 특징
     - **경량(Light Weight) 프로세스**라고도 함
     - 독립적인 **스케줄링의 최소 단위**
     - 동일 프로세스 환경에서 **독립적 다중 수행** 가능

9. **스레드 분류**

   |   구분    |      사용자 수준      |     커널 수준      |
   | :-------: | :-------------------: | :----------------: |
   |   운용    | **사용자 라이브러리** | **운영체제 커널**  |
   | 오버헤드  |       **적음**        |        많음        |
   |   속도    |       **빠름**        |        느림        |
   |   구현    |      **어려움**       |      **쉬움**      |
   | 커널 접근 |           -           | **동시 접근 가능** |
   - 스레드 사용 장점
     | 장점 |
     | ------------------------- |
     | **병행성** 증진 |
     | **처리율** 향상 |
     | **응답 시간** 단축 |
     | 기억장소 **낭비 감소** |
     | 프로세스 간 **통신 향상** |

10. **프로세스 vs 스레드**

| 구분 |       프로세스        |          스레드           |
| :--: | :-------------------: | :-----------------------: |
| 정의 | 실행중인 **프로그램** | 프로세스 내 **작업 단위** |
| 자원 |    **독립적** 할당    |         **공유**          |
| 통신 |        어려움         |        **효율적**         |
| 크기 |        무거움         |         **경량**          |

### 8. 스케줄링 (Scheduling)

1. **스케줄링이란?**

   > **한 줄 정의:** 프로세스가 실행될 때 필요한 **시스템 자원을 할당**하는 작업

2. **스케줄링 종류**

   | 스케줄링 | 다른 이름              | 한 줄 정의                        |
   | :------: | ---------------------- | --------------------------------- |
   | **장기** | 작업/상위 스케줄링     | 프로세스를 **준비상태 큐**로 보냄 |
   | **중기** | -                      | CPU 할당 결정, **부하 조절**      |
   | **단기** | 프로세서/하위 스케줄링 | CPU **할당 시기**와 프로세스 지정 |

3. **스케줄링 단계별 특징**
   1. 장기 스케줄링 (Job Scheduling)
      - 어떤 프로세스가 자원을 차지할지 결정하여 **준비상태 큐**로 보내는 작업
      - 디스크에 있는 작업(Job) 중 어떤 프로세스를 메모리에 올릴지 결정

      | 특징      | 설명                             |
      | --------- | -------------------------------- |
      | 다른 이름 | **작업 스케줄링**, 상위 스케줄링 |
      | 수행      | **작업 스케줄러**                |

   2. 중기 스케줄링
      - CPU 할당받을 프로세스 결정, 프로세스를 **일시 보류**하여 부하 조절
        = 메모리에 있는 프로세스 중 일부를 일시 보류(Suspend) 상태로 이동

      | 특징 | 설명                |
      | ---- | ------------------- |
      | 목적 | **부하 조절**       |
      | 방법 | 일시 보류 후 활성화 |

   3. 단기 스케줄링 (Processor Scheduling)
      - CPU **할당 시기**와 특정 프로세스를 지정하는 작업
      - Ready Queue 에 있는 프로세스 중 지금 CPU를 줄 프로세스 선택

      | 특징      | 설명                                 |
      | --------- | ------------------------------------ |
      | 다른 이름 | **프로세서 스케줄링**, 하위 스케줄링 |
      | 수행      | **프로세서 스케줄러**                |
      | 포함      | **문맥 교환**                        |

4. **문맥 교환 (Context Switching)**

   > **한 줄 정의:** CPU를 다른 프로세스에 할당하기 위해 **현재 상태 저장 → 새 상태 설정 → CPU 할당**하는 작업
   - 문맥 교환 과정

     ```
     1. 현재 프로세스 상태 정보 저장
     2. 새 프로세스 상태 정보 설정
     3. CPU 할당하여 실행
     ```

5. **비선점 vs 선점 스케줄링**

   |    구분     | 비선점 (Non-Preemptive) |   선점 (Preemptive)   |
   | :---------: | :---------------------: | :-------------------: |
   | CPU 빼앗기  |       **불가능**        |       **가능**        |
   |  CPU 사용   |      완료까지 사용      |     강제로 빼앗김     |
   |   공정성    |        **공정**         | 우선순위 높은 것 먼저 |
   |  응답 시간  |      **예측 용이**      |         빠름          |
   |  오버헤드   |          적음           |       **많음**        |
   | 적합 시스템 |      **일괄 처리**      |   **대화식 시분할**   |

6. **비선점 스케줄링**

   > **한 줄 정의:** 한 프로세스가 CPU를 할당받으면 종료되거나 대기 상태가 될 때까지 CPU를 계속 점유하는 방식
   - 비선점 특징

     | 특징     | 설명                           |
     | -------- | ------------------------------ |
     | CPU 사용 | 프로세스 **완료**까지 사용     |
     | 장점     | 요구를 **공정하게** 처리       |
     | 장점     | 응답 시간 **예측 용이**        |
     | 적합     | **일괄 처리** 방식             |
     | 단점     | 중요한 작업이 긴 작업을 기다림 |

   - 비선점 스케줄링 알고리즘 (시험 필수!)

     |   알고리즘   | 전체 이름                   |
     | :----------: | --------------------------- |
     |   **FCFS**   | First Come First Served     |
     |   **SJF**    | Shortest Job First          |
     |   **HRN**    | Highest Response-ratio Next |
     | **우선순위** | Priority                    |
     |  **기한부**  | Deadline                    |

7. **선점 스케줄링**

   > **한 줄 정의:** 우선순위가 높은 프로세스가 오면 실행 중인 프로세스를 중단하고 CPU를 할당하는 방식
   - 선점 특징

     | 특징       | 설명                                |
     | ---------- | ----------------------------------- |
     | CPU 빼앗기 | **우선순위 높은** 프로세스가 빼앗음 |
     | 장점       | 우선순위 높은 것 **빠르게** 처리    |
     | 적합       | **대화식 시분할** 시스템            |
     | 단점       | 많은 **오버헤드** 발생              |
     | 필요       | **인터럽트용 타이머 클록**          |

   - 선점 스케줄링 알고리즘 (시험 필수!)

     |       알고리즘       | 전체 이름                  |
     | :------------------: | -------------------------- |
     |        **RR**        | Round Robin                |
     |       **SRT**        | Shortest Remaining Time    |
     |  **선점 우선순위**   | Preemptive Priority        |
     |    **다단계 큐**     | Multi-Level Queue          |
     | **다단계 피드백 큐** | Multi-Level Feedback Queue |

### 9. 주요 스케줄링 알고리즘

1. **스케줄링 알고리즘 분류**

   |     비선점      |       선점       |
   | :-------------: | :--------------: |
   | **FCFS (FIFO)** | RR (Round Robin) |
   |     **SJF**     |     **SRT**      |
   |     **HRN**     |  선점 우선순위   |
   |    우선순위     |    다단계 큐     |
   |     기한부      | 다단계 피드백 큐 |

2. **비선점 스케줄링 알고리즘**
   1. FCFS (First Come First Service)

      > **한 줄 정의:** 준비상태 큐에 **도착한 순서**대로 CPU를 할당하는 기법
      - 다른 이름
        - **FIFO** (First In First Out)
        - 선입선출
        - FCFS 특징

        | 특징 | 설명                                |
        | ---- | ----------------------------------- |
        | 방식 | **도착 순서**대로 처리              |
        | 구분 | **비선점**                          |
        | 장점 | 가장 **간단**, **공평성** 유지      |
        | 단점 | 짧은 작업이 긴 작업 기다림          |
        | 단점 | 중요한 작업이 덜 중요한 작업 기다림 |

      - FCFS 예시

        ```
        도착 순서: P1 → P2 → P3
        실행 순서: P1 → P2 → P3 (도착 순서 그대로!)
        ```

   2. **SJF (Shortest Job First)**

      > **한 줄 정의:** **실행 시간이 가장 짧은** 프로세스에게 먼저 CPU를 할당하는 기법
      - 다른 이름
        - 단기 작업 우선
        - SJF 특징

        | 특징 | 설명                                     |
        | ---- | ---------------------------------------- |
        | 방식 | **실행 시간 짧은** 것 먼저               |
        | 구분 | **비선점**                               |
        | 장점 | **평균 대기 시간 최소** (최적 알고리즘)  |
        | 단점 | 실행 시간 긴 프로세스 **무한 연기** 가능 |

      - SJF 예시

        ```
        프로세스: P1(5초), P2(2초), P3(8초), P4(1초)
        실행 순서: P4(1초) → P2(2초) → P1(5초) → P3(8초)
                 (짧은 순서대로!)
        ```

      - SJF 핵심
        - **평균 대기 시간 최소** = **최적** 알고리즘
        - 단점: **무한 연기** (기아 현상)

   3. **HRN (Highest Response-ratio Next)**

      > **한 줄 정의:** **대기 시간과 서비스 시간을 고려**하여 우선순위를 계산하는 기법 (SJF 보완)
      - HRN 특징

        | 특징 | 설명                               |
        | ---- | ---------------------------------- |
        | 목적 | **SJF 단점 보완**                  |
        | 구분 | **비선점**                         |
        | 방식 | **우선순위 계산**하여 높은 것 먼저 |

      - HRN 우선순위 계산식 (시험 필수!)

        ```
        우선순위 = (대기 시간 + 서비스 시간) / 서비스 시간
        ```

      - 우선순위가 높아지는 경우

        | 조건                           |  우선순위  |
        | ------------------------------ | :--------: |
        | 서비스(실행) 시간 **짧을수록** | **높음** ↑ |
        | 대기 시간 **길수록**           | **높음** ↑ |

      - HRN 예시

        ```
        P1: 대기 5초, 서비스 2초 → (5+2)/2 = 3.5
        P2: 대기 3초, 서비스 6초 → (3+6)/6 = 1.5
        P3: 대기 8초, 서비스 4초 → (8+4)/4 = 3.0

        우선순위: P1(3.5) > P3(3.0) > P2(1.5)
        실행 순서: P1 → P3 → P2
        ```

      - HRN 핵심
        - SJF의 **무한 연기** 문제 해결
        - 대기 시간 고려 → **오래 기다린 것도 기회** 부여

   4. **비선점 알고리즘 비교**

      | 알고리즘 | 기준                  | 특징                      |
      | :------: | --------------------- | ------------------------- |
      | **FCFS** | **도착 순서**         | 가장 간단, 공평           |
      | **SJF**  | **실행 시간** 짧은 것 | 평균 대기 최소, 무한 연기 |
      | **HRN**  | **계산식**            | SJF 보완, 대기 시간 고려  |

3. **선점 스케줄링 알고리즘**
   1. RR (Round Robin)

      > **한 줄 정의:** 각 프로세스에 **동일한 시간 할당량**을 부여하여 순환 처리

      | 특징 | 설명                                  |
      | ---- | ------------------------------------- |
      | 방식 | **시간 할당량**(Time Slice) 만큼 순환 |
      | 구분 | **선점**                              |
      | 적합 | **시분할** 시스템                     |

   2. SRT (Shortest Remaining Time)

      > **한 줄 정의:** **남은 실행 시간이 가장 짧은** 프로세스에게 CPU 할당 (SJF의 선점 버전)

      | 특징 | 설명                       |
      | ---- | -------------------------- |
      | 방식 | **남은 시간** 짧은 것 먼저 |
      | 구분 | **선점**                   |
      | 관계 | **SJF의 선점 버전**        |

   3. SJF vs SRT 비교

      |   구분    |      SJF      |      SRT      |
      | :-------: | :-----------: | :-----------: |
      | 선점 여부 |  **비선점**   |   **선점**    |
      |   기준    | **실행 시간** | **남은 시간** |
      |  빼앗김   |     없음      |   **있음**    |

### 12. 인터넷

1. 인터넷이란?

   > **한 줄 정의:** **TCP/IP 프로토콜** 기반으로 전 세계 컴퓨터와 네트워크가 연결된 **광범위한 통신망**

2. **인터넷 특징**

   | 특징     | 설명                                |
   | -------- | ----------------------------------- |
   | 기원     | 미 국방성 **ARPANET**               |
   | 운영체제 | **유닉스** 기반                     |
   | 프로토콜 | **TCP/IP**                          |
   | 주소     | 모든 컴퓨터는 **고유 IP 주소** 보유 |
   - 네트워크 연결 장치
     - **브리지 (Bridge)**
     - **라우터 (Router)**
     - **게이트웨이 (Gateway)**
   - 백본 (Backbone)
     - 인터넷의 **주가 되는 기간망** (중추적 역할)

3. **IP 주소 (Internet Protocol Address)**

   > **한 줄 정의:** 인터넷에 연결된 모든 컴퓨터를 **구분하기 위한 고유 주소**
   - IPv4 구성
     - **8비트 × 4부분 = 총 32비트**
     - 예: 192.168.0.1

4. **IP 주소 클래스**

   | 클래스 |  시작 범위  | 용도              |    호스트 수    |
   | :----: | :---------: | ----------------- | :-------------: |
   | **A**  |  **0~127**  | 국가, 대형 통신망 | 2²⁴ (약 1677만) |
   | **B**  | **128~191** | 중대형 통신망     |  2¹⁶ (65,536)   |
   | **C**  | **192~223** | 소규모 통신망     |    2⁸ (256)     |
   | **D**  | **224~239** | **멀티캐스트**용  |        -        |
   | **E**  |    240~     | 실험용 (공용 X)   |        -        |
   - 클래스 암기

     | 클래스 | 시작 |    규모    |
     | :----: | :--: | :--------: |
     |   A    |  0   |  **대형**  |
     |   B    | 128  |  **중형**  |
     |   C    | 192  |  **소형**  |
     |   D    | 224  | 멀티캐스트 |

5. **서브네팅 (Subnetting)**

   > **한 줄 정의:** 할당된 네트워크 주소를 **여러 개의 작은 네트워크로 나누어** 사용
   - 서브넷 마스크 (Subnet Mask)
     - IP 주소 중 **네트워크 주소와 호스트 주소를 구분**하기 위한 비트
   - IPv4 vs IPv6 비교
     | 구분 | IPv4 | IPv6 |
     | :-----: | :-----------: | :----------------: |
     | 비트 수 | **32비트** | **128비트** |
     | 구성 | 8비트 × 4부분 | **16비트 × 8부분** |
     | 표기 | 점(.) 구분 | **콜론(:)** 구분 |
     | 표현 | 10진수 | **16진수** |
     | 예시 | 192.168.0.1 | 2001:0db8:... |

6. **⭐IPv6 특징**

   > **한 줄 정의:** IPv4 **주소 부족 문제 해결**을 위해 개발된 128비트 주소 체계
   - IPv6 장점

     | 장점               | 설명                        |
     | ------------------ | --------------------------- |
     | **주소 부족 해결** | 128비트 → 거의 무한대       |
     | **빠른 전송**      | IPv4보다 전송 속도 빠름     |
     | **보안 강화**      | 인증성, 기밀성, 무결성 지원 |
     | **품질 보장**      | Traffic Class, Flow Label   |
     | **자동 구성**      | 네트워크 환경 **자동 구성** |
     | **확장성**         | 패킷 크기 제한 없음         |

7. **IPv4 → IPv6 전환 전략**

   |     전략      | 한 줄 정의                                        |
   | :-----------: | ------------------------------------------------- |
   | **듀얼 스택** | IPv4와 IPv6 **모두 처리**하도록 두 개 스택 구성   |
   |  **터널링**   | IPv4 망에 **터널**을 만들어 IPv6 패킷 통과        |
   |   **변환**    | 헤더 변환, 전송 계층 릴레이, 응용 계층 게이트웨이 |

8. **IPv6 주소 체계**

   |      유형      | 한 줄 정의                            |  통신 방식  |
   | :------------: | ------------------------------------- | :---------: |
   | **유니캐스트** | 단일 송신자 ↔ **단일** 수신자        | **1 대 1**  |
   | **멀티캐스트** | 단일 송신자 ↔ **다중** 수신자        | **1 대 다** |
   | **애니캐스트** | 단일 송신자 ↔ **가장 가까운** 수신자 | **1 대 1**  |
   - 캐스트 암기
     - **유니** = 하나 = **1:1**
     - **멀티** = 여러 개 = **1:다**
     - **애니** = 아무나 = 가장 **가까운 1개**

9. **도메인 네임 (Domain Name)**

   > **한 줄 정의:** 숫자 IP 주소를 **사람이 이해하기 쉬운 문자**로 표현한 것
   - 도메인 구조

     ```
     www.example.co.kr
     │    │      │  │
     │    │      │  └─ 국가 (상위)
     │    │      └──── 기관 종류
     │    └─────────── 기관 이름
     └──────────────── 호스트 이름 (하위)
     ```

     - **왼쪽 → 오른쪽** = **하위 → 상위** 도메인

   - DNS (Domain Name System)
     - 문자 도메인 네임을 **IP 주소로 변환**하는 시스템

10. **IP 주소 요약**

    | 항목   |  IPv4  |  IPv6   |
    | ------ | :----: | :-----: |
    | 비트   |   32   |   128   |
    | 부분   |   4    |    8    |
    | 단위   | 8비트  | 16비트  |
    | 구분자 | 점(.)  | 콜론(:) |
    | 표기   | 10진수 | 16진수  |

### 13. OSI 참조 모델 (정보처리기사)

1. **OSI 참조 모델이란?**

   > **한 줄 정의:** **ISO**에서 제안한 다른 시스템 간 원활한 통신을 위한 **7계층** 통신 규약

2. **OSI 참조 모델 개요**

   | 항목      | 내용                              |
   | --------- | --------------------------------- |
   | 제정      | **ISO** (국제표준화기구)          |
   | 목적      | 서로 다른 시스템 간 **상호 접속** |
   | 구성      | **7계층**                         |
   | 하위 계층 | 1~3 계층                          |
   | 상위 계층 | 4~7 계층                          |

3. **OSI 7계층 구조**

   ```
   ┌─────────────────────────────┐
   │   7. 응용 계층 (Application)  │  ← 상위
   ├─────────────────────────────┤     계층
   │   6. 표현 계층 (Presentation) │
   ├─────────────────────────────┤
   │   5. 세션 계층 (Session)      │
   ├─────────────────────────────┤
   │   4. 전송 계층 (Transport)    │  ← 인터페이스
   ├─────────────────────────────┤
   │   3. 네트워크 계층 (Network)   │  ← 하위
   ├─────────────────────────────┤     계층
   │   2. 데이터 링크 계층 (Data Link)│
   ├─────────────────────────────┤
   │   1. 물리 계층 (Physical)     │
   └─────────────────────────────┘
   ```

4. **계층별 데이터 단위 (PDU)**

   |        계층         |   PDU (데이터 단위)    |
   | :-----------------: | :--------------------: |
   |      물리 계층      |     **비트 (Bit)**     |
   |  데이터 링크 계층   |   **프레임 (Frame)**   |
   |    네트워크 계층    |   **패킷 (Packet)**    |
   |      전송 계층      | **세그먼트 (Segment)** |
   | 세션/표현/응용 계층 |  **메시지 (Message)**  |

5. **1계층: 물리 계층 (Physical Layer)**

   > **한 줄 정의:** 두 장치 간의 **실제 접속과 절단** 등 기계적, 전기적, 기능적 특성 정의

   | 항목 | 내용                                  |
   | ---- | ------------------------------------- |
   | 역할 | 물리적 전송 매체, 전송 신호 방식 정의 |
   | PDU  | **비트 (Bit)**                        |
   | 표준 | **RS-232C**, X.21                     |
   | 장비 | **리피터**, **허브**                  |

6. **2계층: 데이터 링크 계층 (Data Link Layer)**

   > **한 줄 정의:** 인접한 시스템 간 **신뢰성 있는 정보 전송**, 연결 설정/유지/종료

   | 항목 | 내용                                           |
   | ---- | ---------------------------------------------- |
   | 역할 | 흐름 제어, 오류 제어, 순서 제어, 프레임 동기화 |
   | PDU  | **프레임 (Frame)**                             |
   | 표준 | **HDLC**, LAPB, LLC, **MAC**, LAPD, **PPP**    |
   | 장비 | **랜카드**, **브리지**, **스위치**             |
   - 데이터 링크 계층 기능

     | 기능          | 설명                  |
     | ------------- | --------------------- |
     | **흐름 제어** | 송수신 속도 차이 해결 |
     | **오류 제어** | 오류 검출과 회복      |
     | **순서 제어** | 프레임 순서적 전송    |
     | **동기화**    | 프레임 시작/끝 구분   |

7. **3계층: 네트워크 계층 (Network Layer)**

   > **한 줄 정의:** 네트워크 연결 관리, 데이터의 **교환 및 중계**, **경로 설정(Routing)**

   | 항목 | 내용                                     |
   | ---- | ---------------------------------------- |
   | 역할 | 경로 설정, 데이터 교환/중계, 트래픽 제어 |
   | PDU  | **패킷 (Packet)**                        |
   | 표준 | **X.25**, **IP**                         |
   | 장비 | **라우터**                               |
   - 네트워크 계층 기능

     | 기능                    | 설명           |
     | ----------------------- | -------------- |
     | **경로 설정 (Routing)** | 최적 경로 선택 |
     | **데이터 교환/중계**    | 패킷 전달      |
     | **트래픽 제어**         | 혼잡 관리      |

8. **4계층: 전송 계층 (Transport Layer)**

   > **한 줄 정의:** **종단 시스템(End-to-End)** 간 투명한 데이터 전송, 상하위 계층 **인터페이스**

   | 항목 | 내용                                            |
   | ---- | ----------------------------------------------- |
   | 역할 | 종단 간 연결, 주소 설정, 다중화, 오류/흐름 제어 |
   | PDU  | **세그먼트 (Segment)**                          |
   | 표준 | **TCP**, **UDP**                                |
   | 장비 | **게이트웨이**                                  |
   - 전송 계층 특징
     - 하위 3계층과 상위 3계층의 **인터페이스** 담당
     - **End-to-End** 전송

9. **5계층: 세션 계층 (Session Layer)**

   > **한 줄 정의:** 송수신 측 간의 **관련성 유지**와 **대화 제어**

   | 항목 | 내용                                       |
   | ---- | ------------------------------------------ |
   | 역할 | 대화 구성, **동기 제어**, 데이터 교환 관리 |
   | PDU  | **메시지 (Message)**                       |
   | 특징 | **토큰** 사용, **동기점** 설정             |
   - 세션 계층 용어

     | 용어       | 설명                                         |
     | ---------- | -------------------------------------------- |
     | **토큰**   | 대화 관리를 위해 사용                        |
     | **동기점** | 오류 회복을 위한 체크점 (소동기점, 대동기점) |

10. **6계층: 표현 계층 (Presentation Layer)**

    > **한 줄 정의:** 데이터 **형태 변환** (코드 변환, 암호화, 압축)

    | 항목 | 내용                               |
    | ---- | ---------------------------------- |
    | 역할 | 코드 변환, 암호화, 압축, 구문 검색 |
    | PDU  | **메시지 (Message)**               |
    - 표현 계층 기능

      | 기능               |
      | ------------------ |
      | **코드 변환**      |
      | **데이터 암호화**  |
      | **데이터 압축**    |
      | **구문 검색**      |
      | **정보 형식 변환** |

11. **7계층: 응용 계층 (Application Layer)**

    > **한 줄 정의:** 사용자가 OSI 환경에 **접근할 수 있도록 서비스 제공**

    | 항목 | 내용                                           |
    | ---- | ---------------------------------------------- |
    | 역할 | 정보 교환, 전자 사서함, 파일 전송, 가상 터미널 |
    | PDU  | **메시지 (Message)**                           |

12. **OSI 7계층 요약표**

    | 계층  |    이름     |   PDU    |         주요 프로토콜/장비         | 핵심 기능           |
    | :---: | :---------: | :------: | :--------------------------------: | ------------------- |
    | **7** |    응용     |  메시지  |             HTTP, FTP              | 사용자 서비스       |
    | **6** |    표현     |  메시지  |             JPEG, MPEG             | 데이터 변환, 암호화 |
    | **5** |    세션     |  메시지  |                 -                  | 대화 제어, 동기     |
    | **4** |    전송     | 세그먼트 |     **TCP, UDP** / 게이트웨이      | End-to-End          |
    | **3** |  네트워크   |   패킷   |     **IP, X.25** / **라우터**      | **경로 설정**       |
    | **2** | 데이터 링크 |  프레임  | **HDLC, MAC** / **브리지, 스위치** | 흐름/오류 제어      |
    | **1** |    물리     |   비트   |     RS-232C / **리피터, 허브**     | 물리적 연결         |

### 14. 네트워크 관련 장비

1. 네트워크 장비 개요

   > 네트워크 장비는 **OSI 계층별**로 동작하는 장비가 다름

2. **OSI 계층별 네트워크 장비**

   |        OSI 계층         | 장비                   |
   | :---------------------: | ---------------------- |
   |    **1계층 (물리)**     | **리피터**, **허브**   |
   | **2계층 (데이터 링크)** | **브리지**, **스위치** |
   |  **3계층 (네트워크)**   | **라우터**             |
   | **4~7계층 (전송~응용)** | **게이트웨이**         |

3. **NIC (Network Interface Card)**

   > **한 줄 정의:** 컴퓨터와 네트워크를 연결하여 **정보 형태를 변경**하는 장치
   - 다른 이름
     - **이더넷 카드**
     - **LAN 카드**
     - **네트워크 어댑터**

4. **허브 (Hub)**

   > **한 줄 정의:** 가까운 거리의 컴퓨터들을 연결하고 **신호 증폭**(리피터 역할 포함)하는 장치

   | 특징     | 설명                      |
   | -------- | ------------------------- |
   | OSI 계층 | **1계층 (물리)**          |
   | 역할     | 회선 통합 관리, 신호 증폭 |
   - 허브 종류

     |      종류       | 한 줄 정의                                      |
     | :-------------: | ----------------------------------------------- |
     |  **더미 허브**  | 단순 연결만 제공, 대역폭 **나누어** 사용        |
     | **스위칭 허브** | 데이터 흐름 제어, **최대 대역폭** 사용 (지능형) |

   💡 최근 대부분 **스위칭 허브** 사용

5. **리피터 (Repeater)**

   > **한 줄 정의:** **왜곡되거나 약해진 신호**를 원래 형태로 **재생**하여 다시 전송하는 장치

   | 특징     | 설명                          |
   | -------- | ----------------------------- |
   | OSI 계층 | **1계층 (물리)**              |
   | 역할     | 신호 재생, **전송 거리 연장** |
   | 용도     | 배선 자유도 향상              |

6. **브리지 (Bridge)**

   > **한 줄 정의:** **LAN과 LAN**을 연결하거나 **세그먼트**를 연결하는 장치

   | 특징      | 설명                          |
   | --------- | ----------------------------- |
   | OSI 계층  | **2계층 (데이터 링크)**       |
   | 다른 이름 | **MAC 브리지**                |
   | 장점      | 트래픽 병목 감소, 보안성 향상 |
   - 브리지 전송 회선 수 공식

     ```
       브리지 n개일 때 전송 가능 회선 수 = n(n-1)/2
     ```

7. **스위치 (Switch)**

   > **한 줄 정의:** LAN과 LAN을 연결하여 **더 큰 LAN**을 만드는 장치 (브리지 발전형)

   | 특징      | 설명                                   |
   | --------- | -------------------------------------- |
   | OSI 계층  | **2계층 (데이터 링크)**                |
   | 처리 방식 | **하드웨어** 기반 → **전송 속도 빠름** |
   | 포트      | 수십~수백 개 포트 제공                 |
   | 속도 제어 | 포트마다 **다른 전송 속도** 지원       |

8. **라우터 (Router)**

   > **한 줄 정의:** LAN과 LAN 연결 + 데이터 전송의 **최적 경로 선택 (Routing)**하는 장치

   | 특징      | 설명                              |
   | --------- | --------------------------------- |
   | OSI 계층  | **3계층 (네트워크)**              |
   | 핵심 기능 | **경로 설정 (Routing)**           |
   | 연결 범위 | LAN-LAN, **LAN-WAN**              |
   | 저장      | **라우팅 테이블** (Routing Table) |
   - 라우터 기능

     | 기능              | 설명                  |
     | ----------------- | --------------------- |
     | **경로 설정**     | 최적 경로 선택        |
     | **프로토콜 변환** | 다른 네트워크 간 연결 |
     | **라우팅 테이블** | 경로 정보 저장        |

9. 브라우터 (Brouter)
   - **브리지 + 라우터** 기능을 모두 수행하는 장치

     | 기능            | 용도                   |
     | --------------- | ---------------------- |
     | **브리지** 기능 | **내부** 네트워크 분리 |
     | **라우터** 기능 | **외부** 네트워크 연결 |

10. **게이트웨이 (Gateway)**

    > **한 줄 정의:** **전 계층(1~7계층)**의 프로토콜이 다른 네트워크 연결, **프로토콜 변환**

    | 특징      | 설명                         |
    | --------- | ---------------------------- |
    | OSI 계층  | **전 계층 (1~7계층)**        |
    | 핵심 기능 | **프로토콜 변환**            |
    | 역할      | 다른 네트워크와의 **출입구** |
    - 게이트웨이 기능

      | 기능                 |
      | -------------------- |
      | **데이터 형식 변환** |
      | **주소 변환**        |
      | **프로토콜 변환**    |

11. **전처리기 (FEP: Front End Processor)**

    > **한 줄 정의:** 메시지 조립/분해, 전송 검사 등을 **미리 수행**하여 컴퓨터 **부담 감소**

    | 특징 | 설명                                        |
    | ---- | ------------------------------------------- |
    | 위치 | 호스트 컴퓨터와 단말장치 사이               |
    | 역할 | 통신 회선 제어, 메시지 조립/분해, 전송 검사 |

12. 네트워크 장비 비교표 (시험 필수!)

    |      장비      |      OSI 계층       | 핵심 기능                   |
    | :------------: | :-----------------: | --------------------------- |
    |   **리피터**   |    1계층 (물리)     | 신호 **재생**, 거리 연장    |
    |    **허브**    |    1계층 (물리)     | 연결, 신호 **증폭**         |
    |   **브리지**   | 2계층 (데이터 링크) | LAN 연결, **세그먼트** 연결 |
    |   **스위치**   | 2계층 (데이터 링크) | LAN 연결, **속도 빠름**     |
    |   **라우터**   |  3계층 (네트워크)   | **경로 설정 (Routing)**     |
    | **게이트웨이** |    전 계층 (1~7)    | **프로토콜 변환**           |

### 15. TCP/IP (정보처리기사)

1. **TCP/IP란?**

   > **한 줄 정의:** 인터넷에 연결된 **서로 다른 기종의 컴퓨터**들이 데이터를 주고받을 수 있도록 하는 **표준 프로토콜**

2. **TCP/IP 개요**

   | 항목    | 내용                     |
   | ------- | ------------------------ |
   | 개발    | **ARPA** (1960년대 말)   |
   | 사용    | **ARPANET** (1972)       |
   | 기본 OS | **UNIX** 기본 프로토콜   |
   | 현재    | 인터넷 **범용 프로토콜** |

3. **TCP vs IP 비교**

   |   구분   |         TCP         |          IP          |
   | :------: | :-----------------: | :------------------: |
   | OSI 계층 |  **4계층 (전송)**   | **3계층 (네트워크)** |
   |  서비스  |     **연결형**      |     **비연결형**     |
   |   특징   |   **신뢰성** 있음   |     신뢰성 없음      |
   |   방식   |      가상회선       |    **데이터그램**    |
   |   기능   | 순서/오류/흐름 제어 | 주소 지정, 경로 선택 |

4. **TCP (Transmission Control Protocol)**

   > **한 줄 정의:** **신뢰성** 있는 **연결형** 서비스를 제공하는 전송 계층 프로토콜
   - TCP 특징

     | 특징      | 설명                                        |
     | --------- | ------------------------------------------- |
     | OSI 계층  | **4계층 (전송)**                            |
     | 서비스    | **연결형**                                  |
     | 연결 방식 | **양방향 (Full Duplex)**, 가상회선          |
     | 전달 방식 | **스트림** 위주 (패킷 단위)                 |
     | 제어 기능 | **순서 제어**, **오류 제어**, **흐름 제어** |
     | 헤더 크기 | 20~60Byte (최대 100Byte)                    |

   - TCP 헤더 정보
     - Source/Destination Port Number
     - Sequence Number
     - Acknowledgment Number
     - Checksum

5. **IP (Internet Protocol)**

   > **한 줄 정의:** **비연결형** 서비스를 제공하며 **주소 지정**과 **경로 선택**을 수행하는 네트워크 계층 프로토콜
   - IP 특징

     | 특징      | 설명                                         |
     | --------- | -------------------------------------------- |
     | OSI 계층  | **3계층 (네트워크)**                         |
     | 서비스    | **비연결형**                                 |
     | 방식      | **데이터그램** 기반                          |
     | 원칙      | **Best Effort** (최선형)                     |
     | 기능      | 패킷 분해/조립, **주소 지정**, **경로 선택** |
     | 헤더 크기 | 20~60Byte                                    |
     | 신뢰성    | **보장 안 됨**                               |

6. **TCP/IP 4계층 구조**

   |       TCP/IP        |        OSI        | 주요 프로토콜                         |
   | :-----------------: | :---------------: | ------------------------------------- |
   |      **응용**       | 응용, 표현, 세션  | FTP, SMTP, TELNET, SNMP, DNS, HTTP    |
   |      **전송**       |       전송        | **TCP**, **UDP**                      |
   |     **인터넷**      |     네트워크      | **IP**, ICMP, IGMP, **ARP**, **RARP** |
   | **네트워크 액세스** | 데이터 링크, 물리 | Ethernet, HDLC, X.25                  |

7. **응용 계층 프로토콜**

   |  프로토콜  | 한 줄 정의                   |
   | :--------: | ---------------------------- |
   |  **FTP**   | **파일 전송** 프로토콜       |
   |  **SMTP**  | **전자우편 전송** 프로토콜   |
   | **TELNET** | **원격 접속** (가상 터미널)  |
   |  **SNMP**  | **네트워크 관리** 프로토콜   |
   |  **DNS**   | **도메인 → IP** 변환         |
   |  **HTTP**  | **HTML 문서** 송수신 (WWW)   |
   |  **MQTT**  | **IoT** 환경 메시징 프로토콜 |

8. **전송 계층 프로토콜**
   - TCP vs UDP 비교

     |   구분    |     TCP     |          UDP          |
     | :-------: | :---------: | :-------------------: |
     | 연결 방식 | **연결형**  |     **비연결형**      |
     |  신뢰성   |  **높음**   |         낮음          |
     |   속도    |    느림     |       **빠름**        |
     | 순서 제어 |  **있음**   |         없음          |
     | 오류 제어 |  **있음**   |         없음          |
     | 흐름 제어 |  **있음**   |         없음          |
     | 오버헤드  |    많음     |       **적음**        |
     |   용도    | 신뢰성 중요 | **실시간, 속도 중요** |

   - UDP (User Datagram Protocol)
     - **비연결형** 서비스, **빠른 속도**, **실시간 전송**에 유리한 프로토콜
   - RTCP (Real-Time Control Protocol)
     - RTP 패킷의 **전송 품질 제어**를 위한 프로토콜

9. **인터넷 계층 프로토콜**

   | 프로토콜 | 한 줄 정의                         |
   | :------: | ---------------------------------- |
   |  **IP**  | 주소 지정, 경로 설정 (비연결형)    |
   | **ICMP** | **오류 처리**, 전송 경로 변경 제어 |
   | **IGMP** | **멀티캐스트** 그룹 관리           |
   | **ARP**  | **IP → MAC** (논리 → 물리)         |
   | **RARP** | **MAC → IP** (물리 → 논리)         |
   - ARP vs RARP 암기

     | 프로토콜 |       변환 방향        |
     | :------: | :--------------------: |
     | **ARP**  |      **IP → MAC**      |
     | **RARP** | **MAC → IP** (Reverse) |

10. **네트워크 액세스 계층 프로토콜**

    |   프로토콜   | 한 줄 정의                        |
    | :----------: | --------------------------------- |
    | **Ethernet** | **CSMA/CD** 방식 LAN (IEEE 802.3) |
    | **IEEE 802** | LAN 표준 프로토콜                 |
    |   **HDLC**   | **비트** 위주 데이터 링크 제어    |
    |   **X.25**   | 패킷 교환망 인터페이스            |
    | **RS-232C**  | 공중전화망 인터페이스             |
