Git은 전 세계 개발자들이 가장 많이 사용하는 **분산 버전 관리 시스템(Distributed VCS)** 이다.  
간단히 말해, 파일의 **변경 이력**을 저장하고 필요할 때마다 특정 시점으로 돌아가거나 여러 사람이 동시에 작업할 수 있도록 도와준다.  

이 글에서는 Git을 처음 접하는 사람들을 위해 **설치 → 사용자 정보 설정 → 저장소 생성 → 기본 명령어** 순서로 정리한다.  
맥북(macOS)을 기준으로 설명하지만, 윈도우/리눅스 환경에서도 거의 동일하게 적용된다.

### 1. Git 설치하기

먼저 PC에 Git을 설치해야 한다.  
맥북(macOS)을 사용한다면 **Homebrew** 패키지 관리자를 통해 설치하는 방법이 가장 간단하다.

```bash
brew install git
```

설치가 되었는지 확인하려면 다음과 같은 명령어를 사용하면 된다.

```bash
git --version
```

정상적으로 설치되었다면 깃 버전이 잘 출력될 것이다.<br/>
참고: Windows 사용자는 [Git 공식 홈페이지](https://git-scm.com/downloads)에서 설치 파일을 다운로드할 수 있다.

--- 

### 2. 사용자 정보 설정

Git은 커밋(Commit)을 할 때 누가 작업했는지 기록한다.
따라서 처음 사용할 때 이름과 이메일을 등록해야 한다.

```bash
git config --global user.name "사용자이름"
git config --global user.email "이메일주소"
```

여기서 --global 옵션은 한 번만 설정하면 모든 프로젝트에서 공통으로 적용된다는 의미이다.
특정 프로젝트에서만 다른 사용자 정보를 쓰고 싶다면 --global 대신 프로젝트 폴더 안에서 git config user.name "다른이름" 처럼 설정할 수 있다.

설정이 잘되었는지 확인하려면 다음과 같이 할 수 있다.

```bash
git config --list
```

출력 예시

```bash
user.name=mingoo
user.email=mingoo@example.com
```

### 3. 저장소(repository) 만들기

1. **새로운 저장소 생성**

Git을 사용하는 가장 기본적인 방법은 저장소(repository) 를 만드는 것이다.
저장소는 Git이 파일의 변경 이력을 추적할 수 있는 “작업 공간”이라고 이해하면 된다.


```bash
git init
```

현재 디렉토리에 `.git` 파일이 생성되고, 깃이 이 폴더를 추적한다.
터미널에서 다음과 같은 방법으로 `.git`파일이 생성되있는지 확인할 수 있다.

```bash
ls -al
```


2. **기존 프로젝트 가져오기(clone)**

```bash
git clone [저장소URL]
```

특정 레퍼지토리의 url을 활용해, 깃을 복제해온다.

### 4. 자주쓰는 기본 명령어

Git을 본격적으로 사용할 때 가장 많이 쓰는 명령어들을 정리한다.

1. **상태 확인**

```bash
git status
```

현재 작업 디렉토리 상태를 확인한다.
추적되지 않은 파일, 변경된 파일, 커밋 대기 중인 파일 등을 한눈에 볼 수 있다.


2. **변경 파일 추적하기**

```bash
git add 파일명
```

특정 파일을 커밋에 포함시키고 싶을 때 사용한다.
작업한 모든 파일을 한 번에 추가하려면

```bash
git add .
```

3. **커밋 만들기**

```bash
git commit -m "작업 내용 메시지"
```

커밋은 스냅샷(snapshot) 을 찍는 것과 같다.
이 시점까지의 변경 내역이 저장되며, 필요할 때 언제든 해당 시점으로 돌아갈 수 있다.


4. **커밋 내역확인**

```bash
git log
```

작성한 커밋의 내역을 시간순으로 확인할 수 있다.
옵션을 붙이면 더 깔끔하게 볼 수 있다.

```bash
git log --oneline --graph
```

5. **원격 저장소 연결**

```bash
git remote add origin [저장소URL]
```

6. **원격 저장소에 업로드 (push)**

```bash
git push origin main
```

### ✅ 마무리 정리

여기까지 Git의 기초 세팅과 자주 쓰는 기본 명령어를 정리했다.
- Git 설치 (brew install git)
- 사용자 정보 등록 (git config)
- 저장소 만들기 (git init / git clone)
- 작업 흐름: git add → git commit → git push

다음 글에서는 브랜치(Branch)와 협업을 위한 Git Workflow 를 다룬다.
본격적인 협업을 하려면 branch, merge, pull request 등을 알아야 한다 🚀
