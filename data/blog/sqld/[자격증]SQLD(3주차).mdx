### 1. 데이터베이스

    1.  **데이터베이스**
        - **데이터** : 컴퓨터로 어떤 일을 처리할 때 그 처리 대상이 되는 것
        - **데이터베이스**
          - 데이터를 일정한 체계에 따라 통합하여 디스크나 메모리에 저장한 것
          - 데이터베이스의 이점 : 독립성, 중복저장의 최소화, 통합 처리, 자체적 관리
          - 데이터베이스를 구축하고 관리할 수 있는 기능을 제공하는 시스템 소프트웨어를 DBMS(DataBase Management System)이라 부른다

    2.  **관계형 데이터베이스**
        - RDB라고 불리며, 행과 열의 테이블 형태를 띄며, 테이블 간의 관계를 통해 데이터를 연결, 관리하는 데이터베이스이다.
        - SQL(Structured Query Languege)이라는 공통의 질의언어를 사용하여 에이터의 검색, 삽입, 수정, 삭제등을 쉽게 수행한다.

    3.  **테이블**
        - 관계형 데이터베이스에서 사용하는 데이터 모델을 2차원 구조의 행과 열로 표현한 매트릭스 형태.
        - 엔터티는 논리적 모델링에서 사용하는 용어이고, 물리적 모델링에서는 이를 테이블이라고 부름. (릴레이션이라고 부르기도 함)

    4.  **SQL**

    - Structured Query Languege의 약자로, 구조화된 질의 언어이다.
    - 데이터베이스에 정해진 문법으로 명령을 주고 그에 대한 결과를 얻기 위한 명령어
    - 데이터베이스를 제어하고 안의 데이터를 조작할 수 있는 언어수단

            <br/>
          1. DDL(Data Definition Languege, 데이터 정의 언어)
            - 데이터의 구조, 즉 <InternalLink link="스키마schema" name="스키마" file="sqld"/>를 정의하는 명령어이다.
            - ERD(Entity Relationship Diagram)으로 그려진 데이터 모델은 DDL로 상호 변환이 가능하다.
            - | 명령어   | 설명                                             |
              | -------- | ------------------------------------------------ |
              | CREATE   | 테이블, 뷰, 인덱스를 생성                        |
              | ALTER    | 기존 객체 구조 변경(칼럼 추가 및 자료형 변경 등) |
              | DROP     | 객체 삭제                                        |
              | TRUNCATE | 테이블의 데이터 전체 삭제(구조 유지)             |
            - ```sql
                  -- 테이블 생성
                  CREATE TABLE EMPLOYEE (
                  EMP_ID INT PRIMARY KEY,
                  NAME VARCHAR(50),
                  DEPT_ID INT,
                  SALARY DECIMAL(10,2)
                  );

                  -- 컬럼 추가
                  ALTER TABLE EMPLOYEE ADD HIRE_DATE DATE;

                  -- 테이블 삭제
                  DROP TABLE EMPLOYEE;

                  -- 데이터만 비우기 (테이블 구조 유지)
                  TRUNCATE TABLE EMPLOYEE;
              ```

          2. DML(Data Manipulation Languege, 데이터 조작 언어)
            - 테이블에 데이터를 조회, 입력, 수정, 삭제하는 명령어이다.
            - Commit전에 롤백이 가능하다.
            - | 명령어 | 설명        |
              | ------ | ----------- |
              | SELECT | 데이터 조회 |
              | INSERT | 데이터 삽입 |
              | UPDATE | 데이터 수정 |
              | DELETE | 데이터 삭제 |
            - ```sql
                  -- 데이터 삽입
                  INSERT INTO EMPLOYEE (EMP_ID, NAME, DEPT_ID, SALARY)
                  VALUES (1, 'Jo Mingu', 101, 5000);

                  -- 데이터 조회
                  SELECT * FROM EMPLOYEE WHERE DEPT_ID = 101;

                  -- 데이터 수정
                  UPDATE EMPLOYEE
                  SET SALARY = SALARY * 1.1
                  WHERE DEPT_ID = 101;

                  -- 데이터 삭제
                  DELETE FROM EMPLOYEE WHERE EMP_ID = 1;
              ```

          3. DCL(Data Control Languege, 데이터 제어 언어)
            - 사용자 접근권한과 같이 보안과 제어를 다루는 명령어이다.
            - | 명령어 | 설명                 |
              | ------ | -------------------- |
              | GRANT  | 사용자에게 권한 부여 |
              | REVOKE | 권한회수             |
            - ```sql
                  -- 사용자에게 SELECT, INSERT 권한 부여
                  GRANT SELECT, INSERT ON EMPLOYEE TO HR_USER;

                  -- 권한 회수
                  REVOKE INSERT ON EMPLOYEE FROM HR_USER;
              ```

          4. TCL(Transaction Control Languege, 트랜젝션 제어 언어)
            - <InternalLink link="스키마schema" name="스키마" file="sqld"/>에 대한 제어를 다루는 명령어이다., DCL의 일부로 보기도 한다.
            - | 명령어    | 설명                    |
              | --------- | ----------------------- |
              | COMMIT    | 변경 내용 확정(저장)    |
              | ROLLBACK  | 이전 상태로 복구(최소)  |
              | SAVEPOINT | 트랜잭션 중간 지점 설정 |
            - ```sql
                  -- 데이터 삽입
                  INSERT INTO EMPLOYEE VALUES (2, 'Minji', 102, 4000);

                  -- 저장 지점 설정
                  SAVEPOINT point1;

                  UPDATE EMPLOYEE SET SALARY = 3500 WHERE EMP_ID = 2;

                  -- 롤백 (point1 이후의 변경 취소)
                  ROLLBACK TO point1;

                  -- 최종 확정
                  COMMIT;
              ```

              - **DELETE VS TRUNCATE VS DROP (데이터 삭제 명령어의 차이점)**
                - | 구분            | DELETE                            | TRUNCATE              | DROP                 |
                  |---------------|-----------------------------------|-----------------------|----------------------|
                  | **언어 종류**         | DML (데이터 조작어)                     | DDL (데이터 정의어)         | DDL (데이터 정의어)        |
                  | **삭제 대상**         | 테이블의 데이터(행)                       | 테이블의 모든 데이터(행)        | 테이블 자체 (데이터 + 구조 모두) |
                  | **스키마(구조)**       | 유지됨                               | 유지됨                   | 삭제됨                  |
                  | **ROLLBACK (복구)** | 가능 (트랜잭션 로그 남김)                   | 불가능                   | 불가능                  |
                  | **저장공간 릴리즈**      | X (즉시 해제 안됨)                      | O (즉시 해제됨)            | O (완전 해제)            |
                  | **WHERE 절 사용**    | 가능                                | 불가능                   | 불가능                  |
                  | **트랜잭션 로그**       | 각 행마다 기록                          | 최소 기록                 | 없음                   |
                  | **속도**            | 느림                                | 빠름                    | 매우 빠름                |
                  | **사용 예시**         | DELETE FROM table WHERE age > 30; | TRUNCATE TABLE table; | DROP TABLE table;    |
                  | **주요 특징 요약**      | 데이터만 삭제 + 되돌리기 가능                 | 모든 데이터 삭제 + 되돌리기 불가   | 테이블 자체 삭제 + 되돌리기 불가  |

### 2. SELECT 문

    1.  **SELECT**
        - 데이터를 조회하는 명령어. WHERE절을 통해 특정 조건의 열만 조회할 수 있다.
        - 테이블의 대한 별명(Alias)도 부여 할 수 있다.
        - | SELECT 칼럼1 FROM TABLE1                                                                                                                |
          | --------------------------------------------------------------------------------------------------------------------------------------- |
          | 테이블 1로부터 칼럼 1을 조회한다. 칼럼이 유일할 때는 칼럼명만 붙혀도 되나, 여러 테이블에서 조회할때는 테이블명.칼럼명으로 작성해야한다. |
          |                                                                                                                                         |
        - | SELECT A.칼럼1, B.칼럼2 FROM TABLE1 A, TABLE2 B WHERE A.칼럼1 = B.칼럼1                                       |
          | ------------------------------------------------------------------------------------------------------------- |
          | 이처럼 WHERE절을 통해 조건을 추가할 수 있다. A.칼럼1 = B.칼럼1인 칼럼들만 모아서 A.칼럼1, B.칼럼2를 조회한다. |
          |
        - | SELECT 칼럼1 AS ID FROM TABLE1                                                               |
          | -------------------------------------------------------------------------------------------- |
          | AS절을 활용하면 칼럼1이라는 이름을 ID값이라는 별명을 부여해서 사용가능하다. (AS는 생갹 가능) |
          |                                                                                              |
        - WHERE와 JOIN의 차이 완벽 정리
          - `WHERE절`과 `JOIN절`의 결과가 같게 나오는 경우가 있다.
          - ```sql
            -- (1) WHERE 방식
            SELECT A.ID, B.NAME
            FROM TABLE1 A, TABLE2 B
            WHERE A.ID = B.ID;

            -- (2) JOIN 방식
            SELECT A.ID, B.NAME
            FROM TABLE1 A
            JOIN TABLE2 B ON A.ID = B.ID;
            ```

          - 둘의 결과는 같지만 내용은 다르다.
            - WHERE 방식 (구식 문법)
              - 이건 SQL-89 (Old SQL) 스타일로, 두 테이블을 카티전 곱(Cartesian Product - 모든 경우의 수를 구하는 것) 으로 합친 뒤 WHERE 절에서 조건으로 필터링하는 방식이다.
              - TABLE1 × TABLE2 → 모든 조합을 만든다. >> WHERE A.ID = B.ID → 그 중 조건을 만족하는 행만 남긴다.
            - JOIN 방식 (현대 문법)
              - 이건 ANSI SQL-92 표준 문법이다. JOIN 조건을 명시적으로 써서 테이블을 결합한다.
              - 이 방식은 가독성이 높고, LEFT JOIN, RIGHT JOIN, FULL JOIN 같은 외부 조인(OUTER JOIN) 도 지원한다. - 나중에 조인문에서 좀더 볼 예정

    2.  **산술 연산자**
        - 더하기, 곱하기와 같은 산술적인 연산을 수행하는 연산자
        - NULL의 산술 연산 결과는 항상 NULL이다.
        - | 연산자 | 설명                           | 우선순위 |
          | ------ | ------------------------------ | -------- |
          | **()** | 괄호 (가장 먼저 계산)          | 1        |
          | **\*** | 곱셈(Multiplication)           | 2        |
          | **/**  | 나눗셈(Division, 실수 결과)    | 2        |
          | **%**  | 나머지(Modulus, 나눈 나머지값) | 2        |
          | **+**  | 덧셈(Addition)                 | 3        |
          | **-**  | 뺄셈(Subtraction)              | 3        |
        - Oracle에서는 나머지 연산자가 MOD함수로 사용된다.
        - | SELECT C1, C2, C1+C2 AS C3, (C1+C2)/2 C4 FROM TABLE1                          |
          | ----------------------------------------------------------------------------- |
          | 테이블 1에서 C1, C2항목 대한 연산을 진행한뒤 각각 C3, C4로 명칭을 붙혀서 조회 |
          |                                                                               |

    3.  **합성 연산자**
        - || 문자열은 연결할 때 사용하는 연산자다.
        - 문자열값 또는 문자열형의 칼럼에 대해 사용 가능하다.
        - | SELECT C1 \|\| '' \|\| C2 AS C3 FROM TABLE1                                      |
          | -------------------------------------------------------------------------------- |
          | 테이블1에서 C1, C2칼럼을 가운데에 공백을 넣고 합성하여 C3라는 명칭을 붙혀서 조회 |
          |                                                                                  |

### 3. SELECT 문

    - SQL에서 함수란 입력된 값에 대해서 특정 연산을 수행한 후 그 결과를 반환하는 일련의 코드 덩어리
    - DBMS별로 만들어져있는 내장함수와 직접 SQL문으로 작성한 사용자 졍의 함수로 나뉜다.
      1. 내장함수
        - 단일행 함수 : 하나의 행에 연산을 수행한 후에 결과를 반환하는 함수
        - 다중행 함수 : 여러행에 대하여 연상을 수행하여 결과값을 반환하는 함수(SUM, AVG 등)
      2. 사용자 지정 함수

    1.  **문자함수**
        - 문자열을 대상으로 한 연산을 수행하는 함수
        1. LOWER : 입력된 문자열을 소문자로 변환하여 반환한다.
          - | SELECT LOWER(NAME) AS NAME FROM MEMBER                                                                                   |
            | ------------------------------------------------------------------------------------------------------------------------ |
            | 네임 값을 소문자로 변환하여 반환한다. <br/> AS [별명] 구문을 사용하지 않으면 lower라는 속성명으로 출력된다(postgres기준) |
            |                                                                                                                          |

        2. UPPER : 입력된 문자열을 대문자로 변환하여 반환한다.
          - | SELECT UPPER(NAME) AS NAME FROM MEMBER                                                                                   |
            | ------------------------------------------------------------------------------------------------------------------------ |
            | 네임 값을 대문자로 변환하여 반환한다. <br/> AS [별명] 구문을 사용하지 않으면 upper라는 속성명으로 출력된다(postgres기준) |
            |                                                                                                                          |

        3. CHR : 입렫된 아스키코드(ASCII) 대응하는 문자를 반환한다.
          - | SELECT CHR(97) FROM DUAL(오라클에서만 FROM DUAL을 사용하고, 다른 DBMS에서는 `SELECT CHR(97)`만 사용한다.) |
            | --------------------------------------------------------------------------------------------------------- |
            | 아스키 코드 97번에 해당하는 **a**값이 반환된다.                                                           |
            |                                                                                                           |

        4. TRIM : 문자열의 양끝단에서 공백 또는 지정된 문자열을 제거하고 반환한다.
          - | TRIM([[arg1][arg2]FROM]arg3)                                    | 설명                                                                                               |
            | --------------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
            | arg1 : LEADING / TRAILING / BOTH 생략될 경우 기본값은 BOTH이다. | LEADING : 문자열의 앞쪽(왼쪽) 에서만 제거<br/>[TRIM(LEADING 'a' FROM 'aaabca') → 'bca'] <br/>      |
            |                                                                 | TRAILING : 문자열의 뒤쪽(오른쪽) 에서만 제거<br/>[TRIM(TRAILING 'a' FROM 'aaabca') → 'aaabc']<br/> |
            |                                                                 | BOTH (기본값) : 문자열의 양쪽 끝 에서 제거<br/>[TRIM(BOTH 'a' FROM 'aaabca') → 'bca']<br/>         |
            | arg2 : 제거할 특정 문자 또는 문자열. 생략되면 공백을 제거한다.  |                                                                                                    |
            | arg3 : 문자열값, 문자열형의 칼럼                                |                                                                                                    |
            |                                                                 |                                                                                                    |
        5. LTRIM : 문자열의 왼쪽 끝에서 공백 또는 지정된 문자열 제거하고 반환한다.
          - | LTRIM([arg1[, arg2])                                           |
            | -------------------------------------------------------------- |
            | arg1 : 문자열 값, 문자열형의 칼럼                              |
            | arg2 : 제거할 특정 문자 또는 문자열. 생략되면 공백을 제거한다. |
            |                                                                |
        6. RTRIM : 문자열의 오른쪽 끝에서 공백 또는 지정된 문자열 제거하고 반환한다.
          - | RTRIM([arg1[, arg2])                                           |
            | -------------------------------------------------------------- |
            | arg1 : 문자열 값, 문자열형의 칼럼                              |
            | arg2 : 제거할 특정 문자 또는 문자열. 생략되면 공백을 제거한다. |
            |                                                                |
        7. SUBSTR : 문자열의 부분 문자열을 추출하여 반환한다.
          - | SELECT SUBSTR('Good Morning', 1, 4)                          |     |
            | ------------------------------------------------------------ | --- |
            | Good Morning에서 1~4까지의 문자열을 추출한다. > **Good반환** |     |
            |                                                              |     |
        8. LENGTH : 입력된 문자의 길이를 반환한다.
          - | SELECT LENGTH('Good Morning')                   |     |
            | ----------------------------------------------- | --- |
            | Good Morning의 길이 값을 반환한다. > **12반환** |     |
            |                                                 |     |
        9. REPLACE : 입력된 문자열에서 특정 문자열을 찾아 다른 문자열로 대체하여 반환한다.
          - | SELECT REPLACE('Good Morning', 'Morning', 'Afternoon')                          |     |
            | ------------------------------------------------------------------------------- | --- |
            | Good Morning에서 Morning을 Afternoon으로 변환하여 반환 > **Good Afternoon반환** |     |
            |                                                                                 |     |

    2.  **숫자함수**
        1. ABS : 절댓값을 반환한다.
          - | SELECT ABS(-2.3)         |     |
            | ------------------------ | --- |
            | 절대값인 2.3을 반환한다. |     |
            |                          |     |
        2. MOD : 첫 번째 인자를 두번째 인자로 나눈 나머지를 계산하여 반환
          - | SELECT MOD(9,2)                   |     |
            | --------------------------------- | --- |
            | 9를 2로 나눈 나머지 값인 1을 반환 |     |
            |                                   |     |
        3. ROUND : 수를 반올림하여 그 결과를 반환한다.
          - | SELECT ROUND(2.67,1)                                                                                                       |     |
            | -------------------------------------------------------------------------------------------------------------------------- | --- |
            | 2.67의 소수점 첫번째 자리까지의 자릿수 까지 표현. 2.7이 반환(두번째 인자**1**이 없다면 정수형이 되게 반환된다. > 3이 반환) |     |
            |                                                                                                                            |     |
        4. TRUNC : 수를 반올림하지 않고 특정 자릿수에서 버림한 후 결과 값을 반환한다.
          - | SELECT TRUNC(2.67,1)                                                                                                       |     |
            | -------------------------------------------------------------------------------------------------------------------------- | --- |
            | 2.67의 소수점 첫번째 자리까지의 자릿수 까지 표현. 2.6이 반환(두번째 인자**1**이 없다면 정수형이 되게 반환된다. > 2이 반환) |     |
            |                                                                                                                            |     |
        5. SIGN : 입력된 값이 양수면 1, 음수면 -1, 0이면 0을 반환한다.
          - | SELECT SIGN(2.67)          |     |
            | -------------------------- | --- |
            | 2.67이 양수이므로 1을 반환 |     |
            |                            |     |
        6. CEIL : 입력된 값보다 크거나 같은 최소의 정수를 반환(FLOOR의 반대?)
          - | SELECT CEIL(2.67)              |     |
            | ------------------------------ | --- |
            | 2.67이상의 최소정수인 3을 반환 |     |
            |                                |     |
        7. FLOOR : 입력된 값보다 작거나 같은 최소의 정수를 반환
          - | SELECT FLOOR(2.67)             |     |
            | ------------------------------ | --- |
            | 2.67이하의 최소정수인 2을 반환 |     |
            |                                |     |

    3.  **날짜함수**
        1. SYSDATE : 오늘의 날짜를 날짜형으로 반환한다.
          - | SELECT SYSDATE AS TODAY |     |
            | ----------------------- | --- |
            | 오늘의 날짜 반환        |     |
            |                         |     |
        2. EXTRACT : 날짜로부터 년, 월, 일을 추출해서 반환한다.
          - | SELECT SYSDATE AS TODAY, EXTRACT(MONTH FROM SYSDATE) AS MON, EXTRACT(DAY FROM SYSDATE) AS DAY |     |
            | --------------------------------------------------------------------------------------------- | --- |
            | 오늘의 날짜, 오늘에 해당하는 달, 오늘에 해당하는 날짜를 반환                                  |     |
            |                                                                                               |     |

    4.  **변환함수**
        - 테이블의 칼럼이 같는 데이터의 타입을 변환하는 함수
        1. TO_NUMBER : 데이터 타입을 숫자형으로 변환한다.
          - | SELECT TO_NUMBER('1001')        |     |
            | ------------------------------- | --- |
            | '1001'을 숫자로 변환 > 1001반환 |     |

        2. TO_CHAR : 데이터 타입을 숫자형으로 변환한다.
          - | SELECT TO_CHAR(1001)            |     |
            | ------------------------------- | --- |
            | 1001을 문자로 변환 > '1001'반환 |     |
          - | SELECT TO_CHAR(SYSDATE, 'YYYYMMDD')                                                             |     |
            | ----------------------------------------------------------------------------------------------- | --- |
            | 오늘 날짜를 'YYYYMMDD'의 형태로 반환 이 arg2는 날짜를 문자로 변환할때 포맷을 설정하는 인자이다. |     |

        3. TO_DATE : 데이터 타입을 날짜 형식으로 변환하여 반환한다.
          - | SELECT TO_DATE(20241011, 'YYYYMMDD')                                                |     |
            | ----------------------------------------------------------------------------------- | --- |
            | 20241011을 YYYYMMDD로 받아 들여 날짜 값으로 반환 > 2024-10-11 00:00:00.000이 반환됨 |     |

    5.  **NULL관련 함수**
        1. NVL
          - 첫번째 인자가 Null이 아니면 첫번째 인자를 그대로 반환하고 Null이면 두번째 인자를 반환한다. 첫번째, 두번째 인자의 데이터 타입은 같아야한다.
          - | SELECT NVL(COL1, 0) FROM TABLE1 |     |
            | ------------------------------- | --- |
            | COL1의 값이 null이면 0을 반환   |     |
        2. NULLIF
          - 입력된 두 인자가 같으면 Null을 반환하고 다르면 첫 번째 인자를 반환한다.
          - | SELECT NVL(COL1, 100) FROM TABLE1 |     |
            | --------------------------------- | --- |
            | COL1 = 100이면 Null을 반환        |     |
        3. COALESCE
          - 입력된 인자를 순서대로 평가하여 Null이 아닌 첫번째 인자를 반환한다. 전부 Null이면 Null을 반환한다.
          - | SELECT COALESCE(COL1, COL2, COL3, COL4, COL5) FROM TABLE1                   |     |
            | --------------------------------------------------------------------------- | --- |
            | COL1, COL2, COL3, COL4, COL5을 순서대로 읽어서 NULL이 아닌 첫번째 값을 반환 |     |

    6.  **CASE문**
        - CASE문은 칼럼이 특정 값을 가지면 이를 대체하는 새 값을 반환하는 연산을 정의한다.
        - 조건에 따라 서로 다른 결과를 반환하기 위한 제어 구조이며 if-else문과 같은 역할을 한다.
        - 예시 1(Simple CASE)
          - ```sql
                SELECT name,
                (CASE grade
                    WHEN 'A' THEN '우수'
                    WHEN 'B' THEN '보통'
                    ELSE '미흡'
                END AS grade_text)
            FROM student
            ```
          - 각행별 grade값을 보고 A이면 우수, B이면 보통, 아니면 미흡이라는 값을 같는 grade_text속성(열)을 추가
        - 예시 2(Searched CASE)
          - ```sql
                SELECT
                name,
                CASE
                    WHEN score >= 90 THEN 'A'
                    WHEN score >= 80 THEN 'B'
                    WHEN score >= 70 THEN 'C'
                    ELSE 'F'
                END AS grade
            FROM student
            ```
          - 각행별 score값을 보고 값이 해당되는 부분에 값을 grade속성(열)을 반환하고 종료

### 4. WHERE 절

    - 특정 조건을 충족하는 열만을 불러내기 위한 조건절
    - SELECT, DELETE, UPDATE에서 사용 가능하다. INSERT에서 사용불가.
    - | SELECT * FROM EMP WHERE DEPTNO = 30; |     |
      | --------------------------------- | --- |
      | DEPTNO = 30행의 모든 값을 반환       |     |

    1.  **비교 연산자**
        - 두값의 크기를 비교하거나, 복합적인 비교를 실행하는 연산자
        - 단일행 비교연산자, 다중행 비교 연산자로 나뉜다.
        1. 단일행 비교연산자
            - 비교 대상이 단일행인 경우 두 값이 같은지 비교하거나, 첫번째 값이 두번째 값보다 큰지 작은지 비교하는 연산자.
            - | 연산자     | 설명                                    |
                |---------|---------------------------------------|
                | = | 첫 번째 값이 두 번째 값과 같으면 참(True) 을 반환 |
                | > | 첫 번째 값이 두 번째 값보다 크면 참(True) 을 반환 |
                | \< | 첫 번째 값이 두 번째 값보다 작으면 참(True) 을 반환 |
                | >= | 첫 번째 값이 두 번째 값보다 크거나 같으면 참(True) 을 반환 |
                | \<= | 첫 번째 값이 두 번째 값보다 작거나 같으면 참(True) 을 반환 |
                | IS NULL | NULL이면 참(True)을 반환 |
        2. 다중행 비교연산자
            - 비교 대상이 다중행인 경우 복합적인 비교를 수행하는 연산자
            - | 연산자    | 설명                                                                                                                        |
              |--------|---------------------------------------------------------------------------------------------------------------------------|
              | IN     | 리스트 중 동일한 값이 하나라도 있으면 참(True) 을 반환함.예시:WHERE C1 IN (10, 20, 30) → WHERE C1=10 OR C1=20 OR C1=30                           |
              | EXISTS | 서브쿼리의 결과가 한 건이라도 있으면 참(True) 을 반환함.(EXISTS는 서브쿼리와 함께 사용됨)예시:WHERE EXISTS (SELECT * FROM TBL) → TBL에 해당하는 행이 존재하면 참        |
              | ALL    | 리스트 각 원소와의 비교에서 모두 참이어야 참(True) 을 반환함.단일형 비교연산자와 결합하여 사용함.예시:WHERE C1 < ALL(10, 20, 30) → WHERE C1\<10 AND C1\<20 AND C1\<30 |
              | ANY    | 리스트의 각 원소와 비교하여 하나라도 참이면 참(True) 을 반환함.단일형 비교연산자와 결합하여 사용함.예시:WHERE C1 < ANY(10, 20, 30) → WHERE C1\<10 OR C1\<20 OR C1\<30  |
    2.  **부정 비교 연산자**
        - 비교연산자의 반대의미. 값이 다른지를 체크하는 의미 > !(Not의 의미를 가짐)를 사용해서 표현
        1. 단일행 부정 비교연산자
            - | 연산자     | 설명                                    |
                |---------|---------------------------------------|
                | != | 첫 번째 값이 두 번째 값과 다르면 참(True) 을 반환 |
                | ^= | 첫 번째 값이 두 번째 값과 다르면 참(True) 을 반환 |
                | \<> | 첫 번째 값이 두 번째 값과 다르면 참(True) 을 반환 |
                | IS NOT NULL | NULL이 아니면 참(True)을 반환 |
        2. 다중행 부정 비교연산자
            - | 연산자     | 설명                                    |
                |---------|---------------------------------------|
                | NOT IN | 리스트중에 동일한 값이 하나도 없으면 참을 반환 |
                | NOT EXISTS | 서브쿼리값이 하나도 없으면 참을 반환 |
    3.  **SQL 연산자**
        - SQL 문법에서 값을 비교하거나 조작할 때 사용하는 기호
        - | 연산자     | 설명                                    |
            |---------|---------------------------------------|
            | BETWEEN A AND B | A보다 크고 B보다 작으면 참을 반환 |
            | LIKE | 패턴 문자열과 매칭 되면 참을 반환 |
        - ```sql
              //SALARY가 3000 이상 5000 이하인 직원만 조회 SALARY >= 3000 AND SALARY <= 5000
              SELECT *
              FROM EMP WHERE SALARY BETWEEN 3000 AND 5000;
            ```
        - ```sql
              //이름에 ‘AN’이 포함된 회원 (‘JANE’, ‘ANDY’)
              SELECT *
              FROM MEMBER WHERE NAME LIKE '%AN%';
        - LIKE 패턴정리
            - | 패턴                   | 의미                | 예시 매칭         | 비고             |
              |----------------------|-------------------|---------------|----------------|
              | A%'                  | A로 시작             | A, AB, APPLE  | 길이 무관(0자 이상)   |
              | %A'                  | A로 끝              | A, BA, KOREA  | 〃              |
              | %A%'                 | A 포함              | JANE, KAY     | 〃              |
              | A_'                  | A로 시작 + 정확히 1글자 더 | AB, Ax        | 전체 2글자         |
              | _A%'                 | 두 번째 글자가 A        | BA, cAfe      | 첫 글자 아무거나      |
              | A__'                 | A + 정확히 2글자 더     | ABC, A한글      | _ 갯수 = 글자 수    |
              | 100\_%' ESCAPE '\'   | 문자 그대로 ‘100_’로 시작 | 100_test      | _와 %를 이스케이프    |
              | (SQL Server) '[AB]%' | A 또는 B로 시작        | Apple, Banana | DBMS 종속(표준 아님) |

    4.  **부정 SQL 연산자**
        - SQL 문법에서 값을 비교하거나 조작할 때 사용하는 기호(부정의 값을 사용)
        - | 연산자     | 설명                                    |
            |---------|---------------------------------------|
            | NOT BETWEEN A AND B | A보다 작거나 B보다 크면 참(True)를 반환 |
            | NOT LIKE | 패턴 문자열과 매칭 되지 않으면 참(True)을 반환 |
        - ```sql
              //SALARY가 3000 미만 5000 초과인 직원만 조회 SALARY < 3000 OR SALARY > 5000
              SELECT *
              FROM EMP WHERE SALARY NOT BETWEEN 3000 AND 5000;
            ```
        - ```sql
              //이름에 ‘AN’이 포함되지 않은 회원 ('ERICA', 'DAVID')
              SELECT *
              FROM MEMBER WHERE NAME NOT LIKE '%AN%';
            ```
    5.  **논리 연산자**
        - 논리 연산자는 SQL에서 여러 조건을 연결하거나, 조건의 참/거짓을 제어하기 위해 사용하는 연산자
        - | 연산자     | 설명                                    |
            |---------|---------------------------------------|
            | AND | AND로 연결된 모든 조건이 참이어야 참을 반환 |
            | OR | OR로 연결된 조건중 하나가 참이면 참 반환 |
            | NOT | 뒤에 값의 참/거짓을 반대로 반환 |

### 5. GROUP BY, HAVING 절

    1. **GROUP BY**
        - 데이터를 그룹화 하기 위한 절 구문
        - 부하가 높은 연산이라 WHERE절로 선 필터링을 진행하고 진행하길 권장한다.
        - | SELECT DEPTNO, SUM(SAL) AS SALS FROM EMP FROM BY GROUP BY DEPTNO                  |     |
            | --------------------------------------------------------------------------- | --- |
            | DEPTNO로 그룹을 지어 그룹별로 SAL의 합값과 DEPTNO를 반환 |     |
    2. **집계함수**
        - GROUP BY를 통해 그룹핑을 한 다음 해당 그룹에 대한 연산(갯수, 합, 평균 등)을 진행하는 함수
        - GROUP BY 없이 집계함수를 쓰면, 그 함수는 전체 테이블(모든 행)을 하나의 그룹처럼 보고 계산한다.
        - | 연산자     | 설명                                    |
          |---------|---------------------------------------|
          | COUNT | 입력된 값의 갯수 반환 > Null제외 |
          | SUM | 입력된 값의 합을 반환 |
          | AVG | 입력된 값의 평균을 반환 |
          | MIN | 입력된 값의 최솟값을 반환 |
          | MAX | 최댓값 |
    3. **HAVING 절**
        - 필터링 조건을 명시하는 구문
        - GROUP BY와 같이 사용될때 그룹화 연산이 끝난 그룹에 대해 필터링이 진행된다
        - ```sql
          SELECT DEPT_ID, COUNT(*) AS CNT, AVG(SALARY) AS AVG_SAL
          FROM EMP                              -- EMP테이블에서
          WHERE JOB = 'ENGINEER'                 -- ① 엔지니어만 남기고
          GROUP BY DEPT_ID                      -- ② 부서별로 묶어서
          HAVING AVG(SALARY) > 4000;            -- ③ 평균 급여가 4000 넘는 부서만 선택
          ```

### 6. ORDER BY 절

    1. **ORDER BY**
      - 정렬 수행하는 절 구문
      - 관계형 데이터베이스의 튜플의 무순서성때문에 조회된 데이터를 순서에 맞게 출력할때 사용된다.
      - SELECT문에서 가장 나중에 실행된다.
      - ```sql
          -- 급여가 높은 사람부터 낮은 사람 순으로 출력
          SELECT 컬럼명
          FROM 테이블명
          ORDER BY 컬럼명 [ASC|DESC]
        ```
      - ```sql
           -- 부서번호(DEPT_ID)를 오름차순으로 정렬하고,같은 부서 내에서는 급여(SALARY)를 내림차순으로 정렬
          SELECT *
          FROM EMP
          ORDER BY DEPT_ID ASC, SALARY DESC;
        ```
      - ```sql
           -- ORDER BY절이 가장 나중에 실행되므로 AS를 통해 만든 alias(AVG_SAL)를 조건으로 사용할 수 있다.
          SELECT DEPT_ID, AVG(SALARY) AS AVG_SAL
          FROM EMP
          GROUP BY DEPT_ID
          ORDER BY AVG_SAL DESC;
        ```
      - 오름차순/내림차순/높은순/낮은순
        - | 한국어 표현    | SQL 키워드  | 숫자   | 문자열       | 날짜     |
          |-----------|----------|------|-----------|--------|
          | 오름차순, 낮은순 | ASC(기본값) | 작은→큰 | A→Z / ㄱ→ㅎ | 오래된→최근 |
          | 내림차순, 높은순 | DESC     | 큰→작은 | Z→A / ㅎ→ㄱ | 최근→오래된 |
