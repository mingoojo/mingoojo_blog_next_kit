## manifest.json이 뭔가

[MDN 공식문서 — Web app manifests](https://developer.mozilla.org/en-US/docs/Web/Manifest)

`manifest.json`은 **웹 앱의 메타데이터를 정의하는 JSON 파일**이다.
브라우저에게 "이 사이트는 이런 이름이고, 이런 아이콘이고, 이런 방식으로 실행해달라"고 알려주는 역할을 한다.

이 파일 하나로 웹사이트를 **앱처럼 설치하고 실행**할 수 있게 된다.
이걸 PWA(Progressive Web App)라고 한다.

HTML에서 아래 태그로 연결한다.

```html
<link rel="manifest" href="/manifest.json" />
```

---

## Next.js가 아니어도 쓸 수 있나

**완전히 프레임워크와 무관한 웹 표준이다.**

```
React (CRA, Vite)  ✅
Vue, Svelte        ✅
Next.js            ✅
순수 HTML          ✅
어떤 프레임워크든  ✅
```

중요한 건 프레임워크가 아니라
**HTML에 `<link rel="manifest">` 태그가 있고, 파일이 올바른 경로에 있느냐**다.

파일명도 `manifest.json`이 관례지만 고정은 아니다.
`<link>` 태그의 `href`와 실제 파일 경로만 일치하면 어떤 이름이든 된다.

```html
<!-- 이것도 되고 -->
<link rel="manifest" href="/manifest.json" />

<!-- 이것도 되고 -->
<link rel="manifest" href="/app.webmanifest" />

<!-- 이것도 됨 -->
<link rel="manifest" href="/static/manifest.json" />
```

`.webmanifest` 확장자가 공식 표준이긴 하지만 `.json`도 전혀 문제없다.

## 설정할 수 있는 값들

### name / short_name

```json
{
  "name": "CrossFit Open 2026 카드 생성기",
  "short_name": "CF Open"
}
```

- `name`: 설치 화면, 스플래시 화면에 표시되는 전체 이름
- `short_name`: 홈 화면 아이콘 아래 표시되는 짧은 이름 (12자 이내 권장)

공간이 부족할 때 브라우저가 `short_name`을 우선 사용한다.

---

### start_url

```json
{
  "start_url": "/crossfit"
}
```

앱 아이콘을 눌렀을 때 **처음 열리는 URL**이다.
보통 앱의 메인 페이지 경로를 넣는다.

---

### display

```json
{
  "display": "standalone"
}
```

**앱이 실행될 때 브라우저 UI를 어떻게 보여줄지** 결정하는 가장 중요한 값이다.

| 값           | 설명                                                             |
| ------------ | ---------------------------------------------------------------- |
| `standalone` | 주소창, 탭바 없이 앱처럼 실행. 상태바는 유지. **가장 많이 쓰임** |
| `fullscreen` | 상태바까지 없애고 완전 전체화면                                  |
| `minimal-ui` | 최소한의 브라우저 UI(뒤로가기 버튼 정도)만 유지                  |
| `browser`    | 일반 브라우저 그대로. manifest 효과 없음                         |

PWA의 핵심이 이 값이다.
`standalone`으로 설정하면 홈 화면에 추가 후 실행 시 **네이티브 앱과 구분이 거의 안 된다.**

---

### background_color

```json
{
  "background_color": "#0a0a0a"
}
```

**스플래시 화면의 배경색**이다.
앱 아이콘을 눌렀을 때 앱이 로드되기 전 잠깐 보이는 화면이다.

앱의 메인 배경색과 맞춰두면 로딩 시 자연스럽게 전환된다.

---

### theme_color

```json
{
  "theme_color": "#ff4500"
}
```

**브라우저 상단 상태바 색상**이다.
Android에서 크롬으로 접속했을 때 주소창 배경색이 이 값으로 바뀐다.
`standalone` 모드에서는 앱 상단 색상이 된다.

HTML `<meta name="theme-color">` 태그와 같은 역할을 한다.

---

### orientation

```json
{
  "orientation": "portrait"
}
```

**화면 방향을 고정**할 수 있다.

| 값          | 설명                      |
| ----------- | ------------------------- |
| `portrait`  | 세로 고정                 |
| `landscape` | 가로 고정                 |
| `any`       | 기기 방향 따라감 (기본값) |

인스타 스토리 카드 생성기처럼 세로 기반 앱이라면 `portrait`으로 고정하는 게 적합하다.

---

### icons

```json
{
  "icons": [
    {
      "src": "/icons/icon-192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/icons/icon-512.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "any maskable"
    }
  ]
}
```

**홈 화면 아이콘 및 스플래시 화면에 사용되는 이미지**다.

- `192x192`: Android 홈 화면 아이콘 기본 크기
- `512x512`: 스플래시 화면 및 고해상도 디스플레이용
- `purpose: "maskable"`: Android에서 아이콘 배경을 채워 원형/사각형 등 원하는 형태로 맞춰줌

두 가지 크기를 모두 제공하는 게 기본이다.

---

### 그 외 값들

```json
{
  "description": "CrossFit Open 기록 카드 생성기",

  "categories": ["fitness", "utilities"],

  "screenshots": [
    {
      "src": "/screenshots/preview.png",
      "sizes": "1080x1920",
      "type": "image/png",
      "form_factor": "narrow"
    }
  ]
}
```

- `description`: 앱 설명. 설치 화면에서 표시될 수 있다.
- `categories`: 앱 카테고리 분류.
- `screenshots`: 앱 설치 프롬프트에 표시되는 미리보기 이미지. `form_factor: "narrow"`는 모바일, `"wide"`는 데스크탑용이다.

## iOS vs Android 차이

manifest.json의 동작이 플랫폼마다 조금 다르다.

| 기능                  | Android (Chrome)             | iOS (Safari)              |
| --------------------- | ---------------------------- | ------------------------- |
| 홈 화면 추가          | 자동 설치 프롬프트 지원      | 수동으로 "홈 화면에 추가" |
| `display: standalone` | ✅                           | ✅                        |
| `theme_color`         | ✅ 상태바 색상 적용          | ⚠️ 제한적                 |
| `icons`               | manifest.json 아이콘 사용    | `apple-touch-icon` 우선   |
| 스플래시 화면         | manifest.json 기반 자동 생성 | 별도 메타 태그 필요       |

iOS에서 아이콘과 스플래시를 제대로 지원하려면 추가 메타 태그가 필요하다.

```html
<!-- iOS 아이콘 -->
<link rel="apple-touch-icon" href="/icons/icon-192.png" />

<!-- iOS 상태바 스타일 -->
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent" />
<meta name="apple-mobile-web-app-title" content="CF Open" />
```

## 마치며

manifest.json을 한 줄로 요약하면 이렇다.

> **브라우저에게 "이 웹사이트를 앱처럼 다뤄달라"고 요청하는 파일이다.**

```
핵심 설정 3가지

1. display: "standalone"
   → 브라우저 UI 없애고 앱처럼 실행

2. icons
   → 홈 화면 아이콘 지정

3. theme_color + background_color
   → 앱 색상 테마 통일
```

특정 프레임워크에 종속된 기능이 아니라 **웹 표준**이기 때문에,
어떤 스택을 쓰든 HTML에 `<link rel="manifest">` 하나만 추가하면 적용된다.

작은 설정 하나로 웹사이트가 앱처럼 설치되고 실행되는 경험을 만들 수 있다.
