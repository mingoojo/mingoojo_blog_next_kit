블로그를 만들고 Vercel에 배포했는데 들어오는 사람이 전혀 없었다.
분석 도구를 켜봐도 트래픽은 0에 가까웠고, 구글에 검색해도 내 블로그는 나오지 않았다.

일단 구글에 내 블로그의 존재를 알리는게 가장 중요한 일이었다.
그걸 위해 구글 서치 콘솔에 내 블로그를 등록하기로 했다.

이 글은 블로그 SEO를 단계별로 개선해나가는 과정을 기록한다.
1편은 그 첫 번째 단계인 **Google Search Console 등록**이다.

## Google Search Console이란

구글 서치 콘솔은 **구글이 내 블로그를 어떻게 인식하고 있는지 확인할 수 있는 무료 도구**다.
없어도 구글 검색 결과에 노출되긴 하지만, 있으면 훨씬 빠르고 정확하게 관리할 수 있다.

```text
구글 서치 콘솔이 해주는 것

├── 크롤링 현황   → 구글이 내 페이지를 언제, 어떻게 수집했는지
├── 색인 상태     → 어떤 페이지가 검색 결과에 등록됐는지
├── 오류 확인     → 색인이 안 된 페이지와 그 이유
├── URL 검사      → 수정한 페이지가 제대로 반영됐는지 검증
└── 실적 보고서   → 노출수, 클릭수, 검색 순위, 유입 키워드
```

### 왜 개인 블로그에 필수인가

블로그를 배포해도 구글은 그 존재를 모른다.
구글 봇이 알아서 발견하기까지 몇 달씩 걸릴 수 있고, 외부 링크가 없는 개인 블로그는 더 오래 걸린다.

```text
서치 콘솔 없이                  서치 콘솔 등록 후

구글 봇이 언제           →      sitemap 제출로
발견할지 모름                   빠르게 인덱싱 요청 가능

어떤 글이 검색에         →      실적 보고서로
노출되는지 모름                 키워드별 노출수 확인 가능

페이지 오류 발생해도     →      오류 즉시 감지하고
모르고 지나침                   수정 후 재검증 요청 가능
```

## 1단계 — sitemap 만들기

서치 콘솔에 등록하기 전에 먼저 sitemap을 만들어야 한다.

### sitemap이란

sitemap은 **구글 봇이 읽기 좋은 XML 형태의 내 블로그 전체 페이지 목록**이다.

```text
sitemap 없을 때

구글 봇이 내 블로그 발견
    ↓
메인 페이지만 읽음
    ↓
링크 타고 다른 페이지 찾아다님 (언제 될지 모름)
    ↓
못 찾은 글은 인덱싱 안 됨
```

```text
sitemap 있을 때

구글 봇이 sitemap.xml 읽음
    ↓
"페이지가 53개네, 전부 크롤링할게"
    ↓
모든 글 빠르게 인덱싱
```

sitemap은 **검색 결과에 뜰 수 있는 자격을 빠르게 얻는 것**이다.
검색 순위는 그 이후 글의 품질과 키워드로 결정된다.

### next-sitemap 설치

```bash
npm install next-sitemap
```

### 설정 파일 생성

프로젝트 루트에 `next-sitemap.config.js`를 만든다.

```js
// next-sitemap.config.js
module.exports = {
  siteUrl: 'https://mingoojo-blog-next-kit.vercel.app',
  generateRobotsTxt: true,
}
```

### package.json 수정

```json
"scripts": {
  "postbuild": "next-sitemap"
}
```

`postbuild`는 `next build` 완료 후 자동으로 실행된다.
새 글을 올릴 때마다 sitemap이 자동으로 업데이트된다.

### 배포 후 확인

배포 후 `/sitemap.xml`로 접속하면 아래와 같이 나온다.

```xml
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://mingoojo-blog-next-kit.vercel.app/</loc>
    <lastmod>2026-03-04</lastmod>
  </url>
  <url>
    <loc>https://mingoojo-blog-next-kit.vercel.app/blog</loc>
    <lastmod>2026-03-04</lastmod>
  </url>
  <url>
    <loc>https://mingoojo-blog-next-kit.vercel.app/projects</loc>
    <lastmod>2026-03-04</lastmod>
  </url>
  ...
</urlset>
```

## 2단계 — Google Search Console 등록

<EnvLink href="https://search.google.com/u/1/search-console/welcome" text="Google Search Console" />

### 소유권 인증이란

서치 콘솔에 블로그를 등록하려면 **내가 이 사이트의 실제 소유자임을 구글에 증명**해야 한다.
구글 입장에서는 아무나 남의 사이트를 등록할 수 없으니 당연한 절차다.

소유권 인증 방식은 여러 가지가 있는데, 그 중 두 가지가 주로 쓰인다.

```text
방식 1. HTML 메타태그
→ <meta name="google-site-verification" content="코드" /> 를 <head>에 삽입
→ 구글이 이 태그를 확인하면 인증 완료
→ 코드를 배포해야 하므로 소스 코드 접근이 필요

방식 2. DNS TXT 레코드
→ 도메인 DNS 설정에 구글이 준 값을 TXT 레코드로 추가
→ 구글이 DNS를 조회해서 해당 값이 있으면 인증 완료
→ 코드 배포 없이 DNS 설정만으로 가능
```

### DNS가 뭔데

DNS(Domain Name System)는 **도메인 주소를 IP 주소로 변환해주는 시스템**이다.

사람은 `mingoojo.com` 같은 주소를 쓰지만, 실제 서버는 `1.2.3.4` 같은 숫자 IP로 통신한다.
DNS는 그 둘을 연결해주는 전화번호부 같은 역할이다.

```text
사용자가 mingoojo.com 입력
    ↓
DNS 서버가 조회
    ↓
"이 도메인의 IP는 1.2.3.4야"
    ↓
해당 서버로 연결
```

### DNS 레코드의 종류

DNS에는 여러 종류의 레코드가 있다.

```text
레코드 종류

├── A 레코드     → 도메인을 IP 주소로 연결
│                  예) mingoojo.com → 1.2.3.4
│
├── CNAME 레코드 → 도메인을 다른 도메인으로 연결 (별칭)
│                  예) www.mingoojo.com → mingoojo.com
│
├── MX 레코드    → 이메일 서버 설정
│                  예) @mingoojo.com 메일을 어디로 받을지
│
└── TXT 레코드   → 도메인에 텍스트 정보를 붙여두는 용도
                   예) 구글 소유권 인증, SPF 이메일 인증 등
```

### DNS TXT 레코드로 소유권 인증이 되는 이유

핵심은 **DNS 레코드는 도메인 소유자만 수정할 수 있다**는 점이다.

```text
구글 서치 콘솔이 하는 일

1. 구글이 인증 코드를 발급
   예) google-site-verification=abc123xyz

2. "이 값을 DNS TXT 레코드에 추가해"

3. 등록자가 도메인 DNS 설정에 TXT 레코드 추가
   (도메인 소유자만 할 수 있는 작업)

4. 구글이 DNS를 조회해서 해당 TXT 레코드가 있는지 확인

5. 있으면 → "이 사람이 도메인 소유자가 맞구나" → 인증 완료
   없으면 → 인증 실패
```

즉, DNS 레코드를 수정했다는 사실 자체가 소유권의 증거가 된다.

### Next.js에서 HTML 메타태그로 인증하기

이 프로젝트는 Vercel에 배포하는 구조라 HTML 메타태그 방식을 사용했다.
서치 콘솔에서 발급받은 코드를 `layout.tsx`의 `<head>`에 추가한다.

소유권 인증 후 왼쪽 메뉴의 Sitemaps에서 sitemap을 제출한다.

```text

→ 왼쪽 메뉴 "Sitemaps"
→ sitemap URL 입력란에 "sitemap.xml" 입력
→ 제출
```


## 마치며

```text
소유권 인증 완료  ✅
sitemap 제출 완료 ✅
구글 봇 크롤링 중 ✅
```

여기까지 완료하면 구글이 내 블로그를 인식하고 크롤링을 시작한다.
검색 결과에 반영되기까지는 시간이 걸린다.

```text
구글 인덱싱 반영  : 1일 ~ 2주
검색 결과 노출    : 1개월 ~ 3개월
검색 순위 상승    : 3개월 ~ 6개월
```

다음 글에서는 실제 노출수와 클릭수 데이터를 확인하고,
추가적인 SEO 최적화 방법을 다룰 예정이다.
