## 질문당 0.5달러, 그래서 사내 LLM을 로컬에 직접 세웠다

사내 문서를 물어보면 답해주는 RAG 챗봇을 만들려고 했다. 처음엔 당연히 Claude API로 붙였다. 잘 됐다. 그런데 **비용이 문제였다.**

<br />
### 1. 배경 — API 자동화가 비용에서 막혔다
 
질문 한 번에 **0.3~0.5달러**가 나갔다. 데모로 몇십 번 눌러본 것만으로 체감이 됐다. "사내 전 직원이 쓰는 도구"를 상상하니 숫자가 무서워졌다.
 
왜 이렇게 비쌌나. 단순 챗봇이 아니라 **RAG**라서다.
 
<Callout type="info" label="왜 RAG는 호출당 비용이 큰가">
  RAG는 질문할 때마다 "관련 문서 조각"을 프롬프트에 **같이 욱여넣는다.** 즉 매 호출의 입력 토큰이 (질문 + 문서 여러 조각)이라 크다. 대화가 길어지거나 문서를 많이 끼울수록 입력 토큰이 계속 불어난다. 출력보다 **입력 토큰이 비용을 끌어올리는** 구조다.
</Callout>
선택지는 둘이었다.
 
| 방향          | 장점                          | 단점                                  |
| ------------- | ----------------------------- | ------------------------------------- |
| Claude API    | 품질 최고, 세팅 간단          | **호출당 과금** — 사내 전체로 확산 시 부담 |
| 로컬 LLM      | **호출 비용 0**, 사내망에서 폐쇄 운영 | 품질·속도 한계, 직접 세팅 필요         |
 
사내 문서는 외부로 안 나가는 게 좋다는 보안 측면까지 더하면, **로컬 LLM**이 합리적이었다.
 
<br />
### 2. 왜 Ollama인가
 
로컬 LLM을 돌리는 방법은 여럿이다(llama.cpp 직접, LM Studio, vLLM …). 그중 **Ollama**를 골랐다.
 
- **설치 한 번 → 모델 한 줄로 다운로드**. `ollama pull qwen2.5:7b` 하면 끝.
- **로컬 HTTP API를 그냥 띄워준다** (`localhost:11434`). 내 코드는 Claude API 부르듯 똑같이 `fetch`로 호출하면 된다.
- 그래서 **나중에 Claude API로 다시 갈아탈 때 코드 구조가 안 바뀐다.** 엔드포인트만 교체하면 된다.
<Callout type="tip" label="설계 의도">
  "로컬로 원리 익히고 → 최종본은 필요하면 API로" 전략을 잡았다. Ollama는 OpenAI 호환에 가까운 단순 HTTP라, **추론 백엔드를 갈아끼울 수 있는 구조**로 처음부터 짜기 좋다.
</Callout>
<br />
### 3. 모델 선택 — 생성 1개 + 임베딩 1개
 
RAG에는 모델이 **두 개** 필요하다. 답을 만드는 생성 모델과, 문서를 벡터로 바꾸는 임베딩 모델.
 
| 용도   | 모델          | 고른 이유                                              |
| ------ | ------------- | ------------------------------------------------------ |
| 생성   | `qwen2.5:7b`  | 한국어 품질이 준수하고 7B라 사무용 PC에서도 돈다       |
| 임베딩 | `bge-m3`      | **다국어 특화** — 한국어 문서 검색 정확도가 영어 위주 모델보다 좋다 |
 
<Callout type="warning" label="함정: 임베딩은 한국어를 따로 챙겨야 한다">
  로컬 RAG 글에서 흔히 추천되는 `nomic-embed-text`는 영어 중심이다. **사내 문서가 한국어면** 검색 단계에서 엉뚱한 조각을 물어온다. 생성 모델만 한국어 되는 걸 고르고 임베딩을 놓치면, "답은 한국어인데 근거를 못 찾는" 챗봇이 된다. 그래서 임베딩을 다국어 모델(`bge-m3`)로 맞췄다.
</Callout>
<br />
### 4. 하드웨어 현실 — GPU가 없다
 
서버로 쓸 PC는 사무용 데스크톱이었다.
 
| 항목   | 사양                                    |
| ------ | --------------------------------------- |
| CPU    | Intel i7-12700 (12코어/20스레드)        |
| RAM    | 16GB (가용 ~6.6GB를 추론에 사용)        |
| GPU    | **없음** — Intel UHD 770 내장그래픽뿐   |
 
외장 GPU가 없으니 추론은 **CPU로** 돈다. Ollama 로그가 솔직하게 말해줬다.
 
```text
msg="dropping integrated GPU; to enable, set OLLAMA_IGPU_ENABLE=1" ... Intel(R) UHD Graphics 770
msg="inference compute" id=cpu ... total="15.7 GiB" available="6.6 GiB"
```
 
<Callout type="tip" label="현실 직시">
  GPU 없는 CPU 추론은 **느리다**(초당 몇 단어 수준). 하지만 RAG 챗봇은 한 번에 한 명이 묻는 도구라 시연·소규모엔 충분하다. 모델 크기는 가용 메모리(~6.6GB) 때문에 **7B 양자화(Q4)가 상한선**. 14B는 무리. "내 PC에 맞는 모델"을 아는 게 첫 단추였다.
</Callout>
<br />
### 5. 설치와 첫 실행
 
설치는 허무할 만큼 쉽다.
 
```bash
# 1) ollama.com 에서 설치파일 받아 실행 (백그라운드 서버 자동 기동: localhost:11434)
 
# 2) 모델 2개 받기
ollama pull qwen2.5:7b   # 생성 (약 4.7GB)
ollama pull bge-m3       # 임베딩 (약 1.1GB)
 
# 3) 진짜 도는지 테스트
ollama run qwen2.5:7b
>>> 정밀측위 RTK가 뭐야?
```
 
답이 한국어로 나오면 성공. 임베딩도 확인한다.
 
```bash
curl http://localhost:11434/api/embeddings \
  -d '{"model":"bge-m3","prompt":"테스트"}'
# → {"embedding":[0.01, -0.23, ...]}  숫자 배열이 좌르륵 나오면 OK
```
 
여기까지는 키도 인터넷도 필요 없다. **호출 비용 0.**
 
<br />
### 6. 사내망에 추론 서버로 열기 — 삽질 구간
 
내 PC만 쓸 거면 끝이지만, 목표는 "**이 PC를 사내 LLM 서버로** 두고 다른 PC가 호출"이었다. 여기서 세 번 데었다.
 
#### 함정 ① 0.0.0.0으로 바꿔도 안 열린다
 
Ollama는 기본적으로 `127.0.0.1`(자기 자신)만 듣는다. `OLLAMA_HOST=0.0.0.0`을 줘야 모든 인터페이스로 연다. 그런데 환경 변수만 추가하고 끝낸 뒤 확인하니:
 
```text
> netstat -an | findstr 11434
TCP    127.0.0.1:11434    0.0.0.0:0    LISTENING   ← 여전히 localhost
```
 
트레이 앱이 떠 있는 상태라 변수가 안 먹은 거였다. **완전히 종료 후** 직접 띄우니 그제야 바뀌었다.
 
```bash
# 트레이에서 Ollama 완전 종료 후
set OLLAMA_HOST=0.0.0.0
ollama serve
# 로그: OLLAMA_HOST:http://0.0.0.0:11434  /  Listening on [::]:11434  ← 이제 열림
```
 
#### 함정 ② 방화벽 명령이 cmd에서 안 먹는다
 
포트 11434를 방화벽에서 열어야 하는데, 흔히 보이는 PowerShell 명령을 cmd에 그대로 쳤다가 실패했다.
 
<CodeCompare>
  <CodePanel type="bad" title="cmd에서 실패 — PowerShell 전용">
    ```text
    New-NetFirewallRule -DisplayName "Ollama" ...
    → 'New-NetFirewallRule'은(는) 내부 또는 외부
      명령... 이 아닙니다.
    ```
  </CodePanel>
  <CodePanel type="good" title="cmd에선 netsh로 (관리자 권한)">
    ```text
    netsh advfirewall firewall add rule ^
      name="Ollama" dir=in action=allow ^
      protocol=TCP localport=11434
    ```
  </CodePanel>
</CodeCompare>
#### 함정 ③ ping이 안 된다고 연결이 안 된 게 아니다
 
연결 확인하려고 다른 PC(Mac)에서 `ping`을 쳤더니 전부 timeout. "망이 다른가?" 싶었지만 아니었다.
 
```text
$ ping 192.168.0.10
Request timeout for icmp_seq 0   ← 5번 다 실패
```
 
<Callout type="error" label="치명적 착각">
  **윈도우 방화벽은 기본적으로 ping(ICMP)을 막는다.** ping 실패는 "연결 불가"가 아니다. 진짜 봐야 할 건 ping이 아니라 **그 포트(11434)가 열렸는가**다. ping으로 판단했으면 멀쩡한 설정을 의심하며 시간을 날렸을 것이다.
</Callout>
포트로 직접 찔러보니 깔끔하게 열려 있었다.
 
```bash
$ curl --max-time 5 http://192.168.0.10:11434/api/tags
HTTP/1.1 200 OK
{"models":[{"name":"bge-m3:latest",...},{"name":"qwen2.5:7b",...}]}
```
 
이제 **이 데스크톱이 사내 LLM 서버**가 되고, 다른 PC들은 `192.168.0.10:11434`로 요청한다.
 
<Callout type="warning" label="보안 경계">
  `0.0.0.0`은 **사내망 누구나 인증 없이** 접근 가능하다는 뜻이다. 신뢰된 내부망이면 괜찮지만, **공유기 포트포워딩으로 인터넷에 노출하면 절대 안 된다.**
</Callout>

### 7. 기초 RAG 프로젝트 — 외부 라이브러리 0개
 
추론 서버가 섰으니 RAG로 감싼다. 원리를 눈으로 보려고 **프레임워크 없이** Node 내장 `fetch`만으로 짰다. RAG는 시점이 다른 두 단계일 뿐이다.
 
```text
[색인 — 미리 한 번]
docs/ 문서 → 조각내기(chunk) → 임베딩(bge-m3) → index.json 저장
 
[질의 — 물을 때마다]
질문 → 임베딩 → index에서 코사인 유사도 top-k 검색
     → 그 조각만 근거로 qwen2.5가 답변 + 출처
```
 
핵심은 세 함수뿐이다 — 임베딩 호출, 코사인 유사도, 청킹.
 
```js
// 텍스트 → 벡터
async function embed(text) {
  const res = await fetch(`${OLLAMA_BASE}/api/embeddings`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ model: 'bge-m3', prompt: text }),
  });
  return (await res.json()).embedding;
}
 
// 두 벡터의 의미 유사도 (1에 가까울수록 비슷)
function cosine(a, b) {
  let dot = 0, na = 0, nb = 0;
  for (let i = 0; i < a.length; i++) {
    dot += a[i] * b[i]; na += a[i] * a[i]; nb += b[i] * b[i];
  }
  return dot / (Math.sqrt(na) * Math.sqrt(nb) + 1e-8);
}
```
 
질의 단계에서 가장 중요한 건 **프롬프트로 환각을 막는 것**이다.
 
```js
const prompt = `너는 사내 문서 도우미야. 아래 [문서]의 내용만 근거로 답해.
문서에 없는 내용은 지어내지 말고 "문서에서 찾을 수 없습니다"라고 답해.
답변 끝에 사용한 근거를 [번호]로 표기해.
 
[문서]
${context}      // ← 유사도 상위 조각들
 
[질문]
${question}`;
```
 
<Callout type="tip" label="신뢰의 핵심 두 가지">
  ① **출처 표기** — 답이 어느 문서에서 왔는지 [번호]로 보여주면, 사용자가 검증할 수 있다. ② **"없으면 모른다고 답하라"** — RAG의 가장 큰 리스크인 "그럴듯한 거짓말"을 프롬프트 레벨에서 1차 차단한다.
</Callout>
색인 한 번, 질문 한 번이면 끝이다.
 
```bash
npm run ingest                       # docs/ → index.json
npm run ask "KPOP은 어떤 기술로 만들어졌어?"
# === 답변 === Spring Boot 3.3 / Java 17 기반 27개 멀티모듈로 ... [1]
# === 참고한 문서 === [1] 회사소개.md (유사도 0.71)
```

### 8. 결과 — 비용은 0, 코드는 그대로
 
| 항목         | Claude API            | 로컬 (Ollama)            |
| ------------ | --------------------- | ------------------------ |
| 호출당 비용  | **0.3~0.5달러**       | **0달러**                |
| 데이터       | 외부 전송             | **사내망 폐쇄**          |
| 품질/속도    | 높음 / 빠름           | 보통 / 느림(CPU)         |
| 코드 구조    | `fetch` 호출          | **동일** (엔드포인트만 다름) |
 
추론 백엔드만 갈아끼우면 되도록 짰으니, **"비용이 중요하면 로컬, 품질이 중요하면 API"**를 상황에 따라 고를 수 있게 됐다.

### 9. 회고
 
#### 얻은 것
 
- 비용 때문에 막힌 자동화를 **로컬 LLM 서버**로 풀었다. 호출 비용 0 + 사내 데이터 폐쇄 운영.
- 추론 백엔드를 **갈아끼울 수 있는 구조**(Ollama ↔ Claude API)로 처음부터 설계.
- 프레임워크 없이 RAG를 직접 짜며 **임베딩·벡터검색·청킹·환각 방지**의 원리를 손에 익혔다.
#### 삽질 포인트
 
- **임베딩 모델의 언어**를 놓치지 마라. 한국어 문서엔 다국어 임베딩(`bge-m3`)이 필수다.
- **내 PC의 메모리가 모델 크기 상한**을 정한다. GPU 없으면 CPU 추론 → 7B Q4가 현실선.
- `OLLAMA_HOST=0.0.0.0`은 **앱 완전 재시작** 후에야 적용된다. `netstat`로 확인.
- 방화벽: `New-NetFirewallRule`은 PowerShell 전용. cmd에선 `netsh`.
- **ping 실패 ≠ 연결 불가.** 윈도우는 ICMP를 막는다. 포트로 직접 확인하라.
#### 마치며
 
"API가 비싸다"는 제약이 오히려 **추론을 갈아끼울 수 있는 구조**와 **로컬 인프라를 직접 세우는 경험**을 남겼다. 비용·보안·품질은 트레이드오프고, 정답은 상황마다 다르다. 중요한 건 **그 사이를 언제든 오갈 수 있게 만들어 두는 것**이라는 걸 이번에 배웠다.
